    3. Feature Allocations, Traits, and Completely Random Measures¤

    Chapter 3 definition and scope¤

    A feature allocation assigns each item a finite subset of an unbounded feature collection. Unlike a partition, one item may possess several features and two features may occur together. A trait allocation generalizes binary possession to nonnegative counts or other marks. The prior on allocations does not specify how those features generate observations; a likelihood is still required.

    Chapter 3 prerequisites and learning objectives¤

    This chapter assumes binary latent-factor models and Chapter 1's random-measure distinction. By the end, you should be able to:

    1. O1 — Distinguish partition, feature-allocation, and trait-allocation row semantics.
    2. O2 — Derive IBP predictive counts and the probability of an unlabeled finite feature allocation.
    3. O3 — Connect beta-process atoms to Bernoulli-process observations without treating inclusion weights as mixture weights.
    4. O4 — Prove local finiteness from the weighted Lévy-integrability condition.
    5. O5 — Diagnose column relabeling, duplicate histories, decoder nonidentifiability, and observation-model failure.
    6. O6 — Transfer the CRM grammar from binary features to count-valued traits or dependent feature measures.

    From one label to many features¤

    For a partition matrix, every row contains exactly one 1. For a feature matrix \(Z=(z_{ik})\), a row may contain any finite number:

    \[ z_{ik}=1 \quad\Longleftrightarrow\quad \text{item }i\text{ uses latent feature }k. \]

    Columns are unlabeled: permuting them does not change the feature allocation. Repeated columns require multiplicities in the probability law because two distinct latent features can have the same observed usage pattern.

    The Indian buffet process is an exchangeable distribution over equivalence classes of binary matrices with finitely many rows and an unbounded number of columns. In its one-parameter sequential construction:

    1. item 1 selects \(K_1\sim\operatorname{Poisson}(\alpha)\) features;
    2. item \(i\ge2\) selects each previously used feature \(k\) with probability \(m_k/i\), where \(m_k\) is its previous usage count;
    3. item \(i\) selects \(K_i^{+}\sim\operatorname{Poisson}(\alpha/i)\) new features.

    The construction is sequential, but the induced row law is exchangeable after columns are treated as unlabeled.

    Expected local and global complexity¤

    The expected number of features used by any row is \(\alpha\). The expected total number of distinct active features after \(N\) rows is

    \[ \mathbb E[K_N]=\alpha H_N =\alpha\sum_{i=1}^{N}\frac1i. \]

    These are prior expectations. Posterior feature counts also depend on the likelihood, feature effects, noise, and hyperparameter inference. “The IBP discovers dimensionality” is meaningful only relative to that complete model and posterior query.

    Worked two-row allocation¤

    Let \(\alpha=2\) and condition on the first item having instantiated exactly two features. For item 2:

    • each existing feature is selected independently with probability \(1/2\);
    • the number of new features is \(\operatorname{Poisson}(1)\).

    The probability that item 2 selects the first existing feature, rejects the second, and creates exactly one new feature is

    \[ \frac12\cdot\frac12\cdot e^{-1} =\frac{1}{4e}. \]

    Its conditional expected number of selected existing features is \(1\), and its expected number of new features is also \(1\). Unconditionally, every row has expected feature count \(\alpha=2\), while the expected number of distinct features after two rows is

    \[ 2H_2=3. \]

    Per-row sparsity and global dictionary growth are different quantities.

    The exchangeable probability of an unlabeled feature allocation¤

    The sequential buffet description hides two combinatorial facts: feature columns are unlabeled, and several features may have the same binary usage pattern. For an \(N\)-row binary matrix with \(K_+\) nonzero columns, let \(m_k\) be the number of ones in column \(k\). Index the \(2^N-1\) possible nonzero column histories by \(h\), and let \(K_h\) be the multiplicity of history \(h\). The one-parameter IBP assigns the equivalence class \([Z]\) the probability below. This is an exchangeable feature probability function (EFPF): it depends on feature multiplicities through a symmetric function. EFPF models form an important subclass, but exchangeability alone does not force every feature allocation to admit an EFPF. The representation boundary is characterized by Broderick, Pitman, and Jordan.

    Define the history-multiplicity and column-mass factors

    \[ \begin{aligned} \mathcal D_N(Z) &=\prod_{h=1}^{2^N-1}K_h!,\\ \mathcal M_N(Z) &=\prod_{k=1}^{K_+} \frac{(N-m_k)!(m_k-1)!}{N!}. \end{aligned} \]

    Then the one-parameter IBP probability is

    \[ \Pr([Z]) =\alpha^{K_+}e^{-\alpha H_N} \frac{\mathcal M_N(Z)}{\mathcal D_N(Z)}. \]

    The multiplicity denominator prevents repeated identical columns from being counted as distinct labelings. The expression depends on row-exchangeable statistics, so the arrival order used to derive it does not become temporal semantics.

    For a finite check, take \(\alpha=2\), \(N=2\), and the unlabeled feature matrix

    \[ Z= \begin{bmatrix} 1&1&0\\ 1&0&1 \end{bmatrix}. \]

    Its three column histories are distinct and have counts \(m=(2,1,1)\). Since \(H_2=3/2\), the displayed probability is

    \[ \Pr([Z]) =2^3e^{-3}\left(\frac12\right)^3 =e^{-3}. \]

    The sequential calculation agrees. The first item creates exactly two features with probability \(2e^{-2}\). The second selects exactly one of them with probability \(1/2\) and creates exactly one new feature with probability \(e^{-1}\). Their product is again \(e^{-3}\). If the two initial features had been treated as fixed labels, the factor of two in “select exactly one” would be lost.

    The beta–Bernoulli process construction¤

    A beta process draw is an unnormalized atomic measure

    \[ B=\sum_{k=1}^{\infty}\omega_k\delta_{\phi_k}, \qquad 0<\omega_k<1. \]

    Conditional on \(B\), each item draws a Bernoulli process

    \[ \begin{aligned} Z_i &=\sum_{k=1}^{\infty}z_{ik}\delta_{\phi_k},\\ z_{ik}\mid B &\overset{\mathrm{ind}}\sim \operatorname{Bernoulli}(\omega_k). \end{aligned} \]

    Thibaux and Jordan (2007) identify the beta process as the de Finetti mixing measure underlying the IBP, in the same structural sense that a DP underlies its exchangeable partition sequence. The analogy has a limit: DP atom masses sum to one and select one component per draw, whereas beta-process atom weights are inclusion probabilities and an item can select many atoms.

    For a diffuse finite base measure \(B_0\) and concentration \(c\), one common beta-process Lévy intensity is

    \[ \nu(d\omega,d\phi) =c\,\omega^{-1}(1-\omega)^{c-1} \,d\omega\,B_0(d\phi). \]

    Parameterizations vary, so \(c\), \(B_0(\Theta)\), and any discount parameters must be stated rather than inferred from the name “beta process.” Under the convention above, \(\mathbb E[B]=B_0\) and \(B_0(\Theta)\) controls the expected feature count per Bernoulli-process draw.

    Why infinitely many atoms can yield a finite row¤

    The local-finiteness claim follows from Poisson thinning. If the global atoms form a Poisson random measure with intensity \(\nu(d\omega,d\phi)\) and a Bernoulli-process draw retains atom \((\omega,\phi)\) with probability \(\omega\), then its active atoms form a thinned Poisson random measure. Hence

    \[ \begin{aligned} K_i &\sim\operatorname{Poisson}(\Lambda),\\ \Lambda &=\int_{\Theta}\int_0^1 \omega\,\nu(d\omega,d\phi). \end{aligned} \]

    provided \(\Lambda<\infty\). The global measure may nevertheless have infinitely many atoms when \(\int\nu(d\omega,d\phi)=\infty\). For the beta-process intensity above,

    \[ \begin{aligned} \Lambda &=B_0(\Theta) \int_0^1 c(1-\omega)^{c-1}\,d\omega\\ &=B_0(\Theta). \end{aligned} \]

    Under the one-parameter IBP convention \(B_0(\Theta)=\alpha\), this recovers \(K_i\sim\operatorname{Poisson}(\alpha)\) and \(\mathbb E[K_i]=\alpha\). Thus “infinite dictionary, finite row” is a consequence of a weighted Lévy-integrability condition, not of truncating the model.

    A likelihood turns features into a data model¤

    One linear-Gaussian latent-feature model is

    \[ x_i=z_i A+\varepsilon_i, \qquad \varepsilon_i\sim\mathcal N(0,\sigma^2I), \]

    where row \(z_i\) selects feature-effect rows of \(A\). Other likelihoods can use noisy-OR composition, neural decoders, count superposition, or temporal dynamics. The IBP supplies \(p(Z)\); it does not choose \(p(x\mid Z,A)\).

    This separation exposes failure modes:

    • column permutation is always nonidentifiable;
    • duplicate or nearly collinear feature effects can yield similar likelihoods;
    • a flexible decoder may ignore \(Z\) or encode several observations with one uninterpretable feature;
    • correlated true causes conflict with the conditional-independence assumptions of the simplest beta–Bernoulli construction;
    • missing-not-at-random features require an observation/selection model.

    From binary features to count-valued traits¤

    CRMs provide a reusable grammar. Let

    \[ \mu=\sum_{k=1}^{\infty}J_k\delta_{\phi_k} \]

    and draw item-specific counts \(x_{ik}\sim h(\,cdot\,;J_k)\). Choices include gamma–Poisson and beta–negative-binomial constructions. Each row still activates finitely many traits under appropriate Lévy-integrability conditions, while counts allow repeated or graded association.

    The observation can then be generated from the random trait measure

    \[ X_i=\sum_k x_{ik}\delta_{\phi_k}. \]

    This is not the same as a normalized mixture: total mass, number of active traits, and trait counts all carry information. Normalizing \(X_i\) can erase a scientifically relevant exposure or abundance variable.

    Temporal and grouped feature measures¤

    Hierarchical beta processes share a feature dictionary across groups while allowing group-specific inclusion probabilities. Dependent beta processes let probabilities evolve with time or covariates. The model must specify whether features persist, appear and disappear, or change only their effects. A dynamic prior over inclusion probabilities is not equivalent to independently applying an IBP at each time.

    The Wright–Fisher IBP, for example, constructs time-varying beta-process weights through Wright–Fisher diffusions. Its stationarity and transition assumptions are part of that model; “dynamic IBP” is not one universal process.

    Independent practice and assessment map: features and traits¤

    1. P1.1 — Predict before computing. In the two-row IBP example with \(\alpha=2\), predict whether the second row's expected total feature count differs from the first row's. Compute its expected old and new counts separately and reconcile them with expected global dictionary growth.
    2. P1.2 — Unlabeled allocation probability. Reproduce the \(N=2\) matrix probability \(e^{-3}\) from both the EFPF and the sequential buffet construction. Identify precisely where a fixed-label calculation loses a factor of two.
    3. P1.3 — Representation boundary. Explain why row exchangeability does not alone imply that an allocation probability is an EFPF. State which symmetry the EFPF adds and why the IBP satisfies it.
    4. P1.4 — Lévy thinning. Starting from \(\nu(d\omega,d\phi)=c\omega^{-1}(1-\omega)^{c-1}d\omega B_0(d\phi)\), derive the active-row Poisson mean and state which two integrals can respectively be finite and infinite.
    5. P1.5 — Likelihood and identifiability audit. In a linear-Gaussian feature model, construct two parameterizations related by a column permutation and describe a second ambiguity caused by duplicated or collinear feature effects. Give label-invariant evidence for synthetic recovery.
    6. P1.6 — Transfer. Replace Bernoulli inclusion by Poisson trait counts. Declare the base CRM, count kernel, local-finiteness condition, observation likelihood, permutation/mark equivalence, posterior query, and a diagnostic that would be invalidated by normalizing away total exposure.

    Complete solutions to P1
    1. It does not differ: both rows have expected count \(\alpha=2\). Conditional on two existing features, row 2 selects \(2(1/2)=1\) old feature on average and creates \(\alpha/2=1\) new one. The global dictionary grows from expected \(2\) to \(\alpha H_2=3\) because old selections do not add distinct features whereas new selections do.
    2. The three distinct column histories have masses \((2,1,1)\), so \(\mathcal D_2=1\) and \(\mathcal M_2=(1/2)^3\). Thus \(2^3e^{-3}(1/2)^3=e^{-3}\). Sequentially, the first row creates two features with probability \(2e^{-2}\), the second chooses exactly one with probability \(1/2\), and it creates one new feature with probability \(e^{-1}\); the product is \(e^{-3}\). Treating the two initial columns as fixed and asking for one named column yields \(1/4\) instead of the unlabeled “exactly one” probability \(1/2\).
    3. Row exchangeability only removes dependence on item order. An EFPF additionally makes the probability a symmetric function of feature multiplicities, so feature labels/order cannot matter in that representation. The IBP probability depends on nonzero-column masses and repeated-history multiplicities, hence has this stronger form; general exchangeable feature allocations need not.
    4. Bernoulli thinning multiplies the intensity by \(\omega\), giving \(\Lambda=B_0(\Theta)\int_0^1c(1-\omega)^{c-1}d\omega=B_0(\Theta)\). Thus \(K_i\sim\operatorname{Poisson}(B_0(\Theta))\). The weighted integral \(\int\omega\,d\nu\) is finite while the unweighted \(\int d\nu\) may diverge at zero, permitting a finite row and infinite global dictionary.
    5. For any permutation matrix \(P\), \((Z,A)\) and \((ZP,P^\top A)\) generate the same mean \(ZA\). Two equal or collinear rows of \(A\) can also exchange, split, or duplicate usage with similar likelihood. In simulations, match columns under a permutation-invariant assignment and report precision/recall or loss with posterior uncertainty rather than raw indices.
    6. One valid construction draws \(\mu=\sum_kJ_k\delta_{\phi_k}\) from a declared CRM and \(x_{ik}\sim\operatorname{Poisson}(J_k)\), requiring the corresponding thinned/marked Lévy integral for finitely many nonzero counts. It then declares how \(X_i=\sum_kx_{ik}\delta_{\phi_k}\) generates observations, treats atom relabeling and identical marks consistently, and targets a count, abundance, or predictive functional. If total exposure is scientific, normalizing \(X_i\) destroys it and makes total-count calibration an invalid downstream diagnostic.
    Retrieval questions
    1. Why can a partition not represent two simultaneous latent causes?
    2. What does a beta-process atom weight mean?
    3. Why is \(\alpha H_N\) not a posterior guarantee?
    4. What changes when binary feature indicators become counts?
    5. Why does the IBP probability contain \(\prod_h K_h!\)?
    6. Which integral permits infinitely many global beta-process atoms but only finitely many active features in one row?
    Complete solutions
    1. A partition assigns each item to exactly one block; a feature allocation permits multiple memberships.
    2. Under the beta–Bernoulli construction it is a conditional inclusion probability, not a normalized mixture weight.
    3. It is a prior expectation under the stated IBP; likelihood, data, and hyperparameter posterior determine posterior complexity.
    4. The generated object becomes a trait/count measure, total mass can matter, and a count likelihood replaces Bernoulli inclusion.
    5. Columns are unlabeled, and \(K_h\) columns can share the same usage history. Dividing by each multiplicity factorial removes duplicate labelings of one feature-allocation equivalence class.
    6. The Bernoulli-thinned intensity must satisfy \(\int_\Theta\int_0^1\omega\,\nu(d\omega,d\phi)<\infty\). The unweighted intensity may still be infinite.

    Chapter 3 synthesis¤

    Partitions express alternatives; feature and trait allocations express composition. The EFPF makes feature-label symmetry explicit, and Lévy thinning explains how an unbounded dictionary yields finite per-item activity. Beta and other completely random measures supply that dictionary, while the likelihood assigns observational meaning. Chapter 4 shares these random objects across groups, covariates, and neural components.
