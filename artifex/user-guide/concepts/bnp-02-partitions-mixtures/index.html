    2. Exchangeable Partitions and Random-Measure Mixtures¤

    Chapter 2 definition and scope¤

    A partition places every item in exactly one unlabeled block. A random-measure mixture generates such a partition by letting observations share atom-valued component parameters. The partition prior, component likelihood, posterior partition, observation density, and scientific interpretation of a cluster are separate layers.

    Chapter 2 prerequisites and learning objectives¤

    This chapter assumes Chapter 1 and finite mixture models. By the end, you should be able to:

    1. O1 — Derive the DP exchangeable partition probability function and account for partition multiplicities.
    2. O2 — Separate the random measure, sample partition, observation likelihood, and posterior scientific claim.
    3. O3 — Compute prior occupied-cluster growth and likelihood-weighted posterior reassignment.
    4. O4 — Compare Dirichlet- and Pitman–Yor-process predictive rules and their growth implications.
    5. O5 — Use label-invariant posterior summaries and decision losses.
    6. O6 — Distinguish density estimation, observed-sample partitioning, and finite component-count recovery.

    Two equivalent marginalizations of the same model expose different structure. The bottom-row warning is the main interpretive boundary: a mixture component is part of a density representation before it is a claimed population.

    From random measure to mixture density¤

    A Dirichlet-process mixture (DPM) is

    \[ G\sim\operatorname{DP}(\alpha,H), \qquad \theta_i\mid G\sim G, \qquad x_i\mid\theta_i\sim f(x_i\mid\theta_i). \]

    Using the stick representation,

    \[ p(x\mid G)=\sum_{k=1}^{\infty}\pi_k f(x\mid\phi_k). \]

    The almost-surely discrete \(G\) supplies tied component parameters; the kernel \(f\) determines the observation distribution around each atom. Changing \(f\) can change smoothness, tail behavior, component overlap, identifiability, and posterior cluster count without changing the DP prior.

    The partition law¤

    Assume \(H\) is diffuse. Integrating out \(G\) gives the predictive allocation rule

    \[ \Pr(c_{n+1}=k\mid\Pi_n)=\frac{n_k}{\alpha+n}, \qquad \Pr(c_{n+1}=\text{new}\mid\Pi_n)=\frac{\alpha}{\alpha+n}, \]

    where \(n_k\) is the size of occupied block \(k\). Multiplying these sequential factors yields the exchangeable partition probability function (EPPF)

    \[ \Pr(\Pi_n=\{B_1,\ldots,B_K\}) =\frac{\alpha^K\Gamma(\alpha)}{\Gamma(\alpha+n)} \prod_{k=1}^{K}\Gamma(|B_k|). \]

    The right-hand side depends only on block sizes, not arrival order or arbitrary cluster labels. It assigns probability to a particular unlabeled partition. Summing over all partitions with the same size profile introduces combinatorial multiplicities.

    Worked partition distribution for three items¤

    Set \(\alpha=1\). For \(n=3\):

    • the one-block partition \(\{123\}\) has probability \(1/3\);
    • each of \(\{12|3\}\), \(\{13|2\}\), and \(\{23|1\}\) has probability \(1/6\), hence the size profile \((2,1)\) has total probability \(1/2\);
    • the all-singleton partition \(\{1|2|3\}\) has probability \(1/6\).

    These probabilities sum to one. The expected number of occupied blocks is therefore

    \[ \mathbb E[K_3] =1\cdot\frac13+2\cdot\frac12+3\cdot\frac16 =\frac{11}{6}. \]

    More generally,

    \[ \mathbb E[K_n] =\sum_{i=1}^{n}\frac{\alpha}{\alpha+i-1}. \]

    For fixed \(\alpha\), this grows logarithmically. The value depends on both \(n\) and \(\alpha\); there is no dataset-independent conversion from concentration to “the number of clusters.”

    Posterior assignment includes the likelihood¤

    The predictive rule above is a prior over partitions before seeing the next \(x\). In a conjugate mixture, a collapsed reassignment for item \(i\) has weights

    \[ \Pr(c_i=k\mid c_{-i},x_{1:n}) \propto \begin{cases} n_{-i,k}\,m_k(x_i), & k\text{ occupied},\\[3pt] \alpha\,m_0(x_i), & k\text{ new}, \end{cases} \]

    where \(m_k\) is the posterior predictive under existing block \(k\) and \(m_0\) is the base-prior predictive. The count term and data-fit term play different roles.

    Worked beta–Bernoulli reassignment¤

    Let component success probabilities have prior \(\phi\sim\operatorname{Beta}(1,1)\). One occupied block contains two successes. Consider assigning a new failure \(x=0\) with \(\alpha=1\).

    The existing-block failure predictive is \(1/4\), so its unnormalized weight is

    \[ 2\times\frac14=\frac12. \]

    The new-block failure predictive is \(1/2\), so its unnormalized weight is

    \[ 1\times\frac12=\frac12. \]

    The posterior assignment probabilities are equal. The partition prior alone would favor joining the existing block by \(2/3\) to \(1/3\); the observed failure supplies exactly enough contrary likelihood evidence to balance the weights. “Rich get richer” is therefore an incomplete description of posterior clustering.

    Label switching and the inferential object¤

    Mixture labels are arbitrary. Relabeling every component leaves the likelihood unchanged, and a posterior sampler may move between symmetric labelings. Label-specific posterior means can therefore be meaningless. Useful label-invariant summaries include:

    • the posterior co-clustering matrix \(S_{ij}=\Pr(c_i=c_j\mid x)\);
    • a representative partition minimizing posterior expected variation of information or another declared loss;
    • posterior distributions of functionals such as density, cluster size profile, or predictive probability;
    • application-defined summaries attached to identifiable component features.

    An adjusted Rand index is appropriate in a simulation with known labels, not as a universal unsupervised score. A visually stable partition is not evidence that latent groups are scientifically real.

    Pitman–Yor and other partition growth laws¤

    A Pitman–Yor process introduces discount \(0\le d<1\) and strength \(\vartheta>-d\):

    \[ \Pr(c_{n+1}=k\mid\Pi_n) =\frac{n_k-d}{\vartheta+n}, \qquad \Pr(c_{n+1}=\text{new}\mid\Pi_n) =\frac{\vartheta+dK_n}{\vartheta+n}. \]

    The DP is the case \(d=0\) with \(\vartheta=\alpha\). For \(0<d<1\), the occupied-block count typically grows on the order of \(n^d\) under the prior rather than \(\log n\), and the induced frequencies have different tail behavior. This is a model choice, not a free robustness improvement. The discount changes small-cluster prevalence and unseen-species prediction and must be checked against the intended query.

    Normalized generalized-gamma and broader Poisson–Kingman models provide other random-mass and partition laws. Their predictive probabilities may require auxiliary variables or special functions. Sharing the phrase “infinite mixture” does not make their partition distributions or posterior algorithms equivalent.

    Density estimation is not component-count recovery¤

    Three targets are commonly collapsed:

    1. the observation density \(p(x)\);
    2. the partition of the observed sample;
    3. a finite number of data-generating populations or mixture components.

    Under stated regularity and support assumptions, Dirichlet mixtures of normals can have consistent density posteriors (Ghosal, Ghosh, and Ramamoorthi, 1999). Yet Miller and Harrison (2014) show that broad Dirichlet- and Pitman–Yor-process mixtures are inconsistent for the number of components when data come from many ordinary finite mixtures. Both statements can be true because they concern different functionals.

    If the scientific estimand is a finite component count, use a model that places a prior on that count and check its identifiability and consistency assumptions. A mixture of finite mixtures places \(K\sim p_K\) before finite mixture weights and can retain restaurant-like computation without asserting that a DPM occupied-block count estimates \(K\).

    Failure case: a narrow kernel manufactures clusters

    A DPM with a likelihood kernel much narrower than the true within-population distribution may use several nearby components to approximate one broad density feature. Better held-out density can then coexist with an increasing occupied-cluster count. Interpreting every occupied component as a distinct cell type, speaker, disease subtype, or physical regime changes a density approximation into an unsupported ontology.

    Connections to adjacent families¤

    • A finite mixture and a DPM share latent-mixture grammar; the mixing-measure prior and partition growth law differ.
    • A VAE can use a random-measure prior, but an amortized encoder does not turn every VAE into a BNP model.
    • An autoregressive clustering network may parameterize a posterior over partitions; it is an inference composition unless its generated partition law is declared as the model.
    • A random graph may use a BNP partition for communities, while node-count, graph-exchangeability, and edge-law semantics still belong to the graph family.

    Independent practice and assessment map: partitions and mixtures¤

    1. P1.1 — Predict before computing. In the beta–Bernoulli reassignment example, predict whether changing the candidate observation from failure to success makes the existing block more or less likely. Compute both unnormalized weights and the normalized probability of joining it.
    2. P1.2 — EPPF normalization. For a DP with \(alpha=1\) and \(n=3\), derive the probability of each unlabeled partition from the EPPF, group them by block-size profile, verify normalization, and compute \(\mathbb E[K_3]\).
    3. P1.3 — Pitman–Yor comparison. With \(n=5\), block sizes \((3,2)\), discount \(d=1/2\), and strength \(\vartheta=1\), compute the two existing-block probabilities and the new-block probability. Compare them with a DP having \(\alpha=1\).
    4. P1.4 — Label-invariant decision. Given posterior co-clustering probabilities \(S_{12}=0.9\), \(S_{13}=0.2\), and \(S_{23}=0.3\), explain why three marginal label modes do not define a coherent partition and name a valid route to a point decision.
    5. P1.5 — Estimand boundary. Construct a case in which two narrow mixture components approximate one broad population well. State which evidence could validate density prediction and which additional assumptions/evidence would be needed to claim two populations.
    6. P1.6 — Transfer. Design a species-discovery comparison between a DP and Pitman–Yor model. Hold the likelihood and data split fixed; declare the unseen-species query, prior-predictive growth check, rare-frequency calibration, computational budget, and a condition under which neither occupied count should be interpreted as a finite true count.

    Complete solutions to P1
    1. The existing block has two successes, so its next-success predictive is \(3/4\) and its weight is \(2(3/4)=3/2\). A new block has predictive \(1/2\) and weight \(1(1/2)=1/2\). The normalized existing-block probability is \((3/2)/(3/2+1/2)=3/4\), larger than for the failure because the likelihood now reinforces the count advantage.
    2. The EPPF gives \(1/3\) for \(\{123\}\), \(1/6\) for each of the three \((2,1)\) partitions, and \(1/6\) for \(\{1|2|3\}\). The profile totals are \(1/3,1/2,1/6\), which sum to one, and \(\mathbb E[K_3]=1/3+1+1/2=11/6\).
    3. The probabilities are \((3-1/2)/6=5/12\), \((2-1/2)/6=1/4\), and \((1+(1/2)2)/6=1/3\); they sum to one. The DP probabilities are \(3/6=1/2\), \(2/6=1/3\), and \(1/6\). Discounting shifts mass from both occupied blocks toward a new block and changes the relative small-block behavior.
    4. Marginal label names are arbitrary and can select mutually inconsistent events. Minimize posterior expected variation of information, Binder loss, or an application-specific loss over valid partitions using posterior co-clustering information; report uncertainty rather than treating the point action as truth.
    5. A broad unimodal population fitted with an underspread kernel may need two nearby components to reproduce its density. Held-out proper scores and posterior predictive tail/shape checks can support the density claim. Two populations additionally require an identifiable population model and independent scientific evidence such as external markers or interventions; occupied components alone do not suffice.
    6. Fit both priors with the same sampling model, preprocessing, held-out units, tuning allowance, and likelihood. Compare prior and posterior discovery curves, held-out new-type counts, rare-frequency calibration, proper predictive scores, and compute. If the data-generating population is not an identifiable finite mixture with a model for its count, neither \(K_n\) is an estimator of one finite truth.
    Retrieval questions
    1. Why is the CRP predictive rule exchangeable despite its sequential presentation?
    2. What term is missing from a posterior assignment rule that uses only \(n_k\)?
    3. Can consistent density estimation certify the true number of mixture components?
    4. What changes when a DP is replaced by a Pitman–Yor prior with \(d>0\)?
    Complete solutions
    1. Multiplying the predictive factors yields an EPPF depending only on the final unlabeled block sizes.
    2. The existing- or new-component posterior predictive likelihood of the observation.
    3. No. Density recovery, sample partitioning, and component-count recovery are different functionals with different consistency conditions.
    4. Existing blocks are discounted, new-block probability depends on \(K_n\), and prior occupied-block growth changes from logarithmic to a power-law order.

    Chapter 2 synthesis¤

    Random-measure mixtures induce exchangeable partitions, but data enter assignments through the likelihood. Partition inference is label-invariant, and occupied blocks need not be scientific populations. Chapter 3 replaces the one-block-per-item constraint by sparse membership in multiple latent features.
