    Newly unlocked in the default route

    No immediate successor names this page as its final required prerequisite. Continue through the book navigation or the chapter's cross-links.

    A point process generates a configuration: a random number of events, their locations in time or space, and optionally a mark attached to each event. A random finite set generates a finite set-valued state. These descriptions overlap for finite simple configurations, but each emphasizes questions that a fixed-length vector or an ordinary token sequence hides: How many points occur? What is the probability of no point in a region? Which reference measure defines the density? Does one event excite, inhibit, or repel another? How should a sampler stop?

    This volume develops those questions from first principles. It begins with counting measures and finite-set densities, derives temporal likelihoods from survival, separates spatial first-order intensity from interaction, and then studies inference, simulation, neural parameterizations, residual analysis, and current research. Architectures and samplers enter only after the generated object is explicit.

    Curriculum position

    Read Probability and Generative Modeling for measures, densities, and support; Sampling and Continuous Time for Poisson processes, Markov chains, ODEs, and SDEs; and Autoregressive Models for conditional factorization. Random functions remain in Stochastic-Process and Function-Space Generators, while random partitions and measures appear in Bayesian Nonparametric Random-Measure Models.

    1. Configurations, Counting Measures, and Set Densities¤

    Chapter 1 prerequisites and learning objectives¤

    This chapter assumes probability measures, densities with respect to a base measure, and conditional probability.

    Numbered learning outcomes. After completing the chapter, you can:

    1. O1 — Name the generated object. Represent a point pattern as a locally finite counting measure and distinguish finite, simple, and non-simple configurations.
    2. O2 — Separate the law's layers. Distinguish cardinality, locations conditional on cardinality, and marks conditional on locations.
    3. O3 — Control the reference convention. Derive cardinality and conditional-location laws from Janossy densities and translate to the declared finite-set integral.
    4. O4 — Audit representations. State when counting measures, finite sets, ordered tuples, padded arrays, and timestamp sequences preserve or erase information.
    5. O5 — Compute a finite law. Normalize a small configuration model and check its empty-set and count probabilities.
    6. O6 — Transfer the configuration contract. Specify the state space, window, reference measure, cardinality, geometry, marks, and stopping rule for a new application.

    The generated object¤

    A point process \(N\) on \(\mathsf S\) is a random locally finite counting measure. A realization may be written

    \[ N=\sum_i \delta_{x_i}, \qquad N(A)=\sum_i \mathbf 1\{x_i\in A\}. \]

    Local finiteness means \(N(A)<\infty\) almost surely for every bounded measurable \(A\). A process is simple if \(N(\{x\})\le 1\) almost surely for every location \(x\); otherwise coincident points can carry multiplicity. When the process is simple and the observation window is bounded, the same realization can be viewed as the finite set \(X=\{x_1,\ldots,x_n\}\).

    This representation separates three random quantities that padding can obscure:

    1. the cardinality \(|X|=N(W)\);
    2. the point locations conditional on that cardinality; and
    3. any marks or attributes conditional on the locations and history.

    Janossy densities: the factorial is part of the definition¤

    Assume \(X\) is finite in \(W\) and its Janossy measures have symmetric densities \(j_n\) with respect to \(\mu^n\). We use the convention

    \[ j_0+ \sum_{n=1}^{\infty}\frac{1}{n!} \int_{W^n}j_n(x_{1:n})\,\mu^n(dx_{1:n})=1, \]

    and therefore

    \[ p_n=\Pr(|X|=n) =\frac{1}{n!}\int_{W^n}j_n(x_{1:n})\,\mu^n(dx_{1:n}). \]

    If \(f_n\) is a symmetric density on an ordered \(n\)-tuple conditional on \(|X|=n\) and integrates to one on \(W^n\), then

    \[ \boxed{j_n(x_{1:n})=n!\,p_n\,f_n(x_{1:n}).} \]

    The factor \(n!\) compensates for the \(n!\) orderings of the same simple set. Some authors absorb this factor into the definition of a set density or choose a unit-rate Poisson process as the reference. Neither choice is wrong, but moving a formula between conventions without moving the reference measure changes its normalization.

    The finite-set integral convention used in finite-set statistics makes the same combinatorics explicit:

    \[ \int F(X)\,\delta X =F(\varnothing)+ \sum_{n=1}^{\infty}\frac{1}{n!} \int_{W^n}F(\{x_1,\ldots,x_n\})\,\mu^n(dx_{1:n}). \]

    Under this convention, a symmetric set density \(F\) normalizes when the displayed set integral equals one. If instead \(F\) is a Radon–Nikodym derivative with respect to a unit-rate Poisson process on \(W\), an additional window-dependent factor appears. Always declare which object is called “the density.”

    Complete worked example: a homogeneous Poisson configuration¤

    Let \(W=[0,1]\), let \(\mu\) be Lebesgue measure, and let the rate be \(\lambda=2\). The count is

    \[ p_n=e^{-2}\frac{2^n}{n!}, \]

    and, conditional on \(N(W)=n\), the ordered locations are iid uniform, so \(f_n(x_{1:n})=1\) on \([0,1]^n\). Hence

    \[ j_n(x_{1:n})=n!\,p_n f_n(x_{1:n})=e^{-2}2^n. \]

    Normalization now exposes the role of the factorial:

    \[ j_0+\sum_{n=1}^{\infty}\frac1{n!}\int_{[0,1]^n}j_n\,dx_{1:n} =e^{-2}\sum_{n=0}^{\infty}\frac{2^n}{n!}=1. \]

    The probability of exactly two points is \(p_2=2e^{-2}\), while the probability of no point is \(p_0=e^{-2}\). A model that predicts plausible locations but assigns the wrong empty-set or count probabilities has not learned this configuration law.

    Marks and projections¤

    A marked point process lives on a product space \(\mathsf S\times\mathsf M\). A mark may be a discrete type, a bounding-box shape, a molecular species, a severity, or a vector. One useful factorization is

    \[ p(x_{1:n},m_{1:n},n) =p_n\,f_n(x_{1:n})\,q(m_{1:n}\mid x_{1:n}), \]

    but conditional independence among marks is an additional assumption, not part of the definition. Removing marks maps a marked process to its ground process; conditioning on one mark class produces a thinned subprocess. These operations need not preserve the original family.

    What this family is not¤

    • A fixed-length point cloud conditions on or fixes cardinality unless it supplies an explicit count law.
    • A timestamp sequence model is a point-process model only if its stopping/no-event probability and continuous-time reference measure are defined.
    • A random function \(f(t)\) is not a random event configuration, although a random nonnegative function can parameterize an intensity.
    • A set encoder is an architecture; it does not define a probability law on finite sets.

    Chapter 1 misconception checks¤

    If two models have the same intensity function, do they have the same point-process law?

    No in general. The ordinary first-order intensity describes expected local counts. Higher-order dependence can differ. A Poisson process is determined by its intensity measure because independent increments are also assumed; clustered, Cox, Gibbs, and determinantal processes can share a first-order intensity while having different joint laws.

    Can we omit the terminal no-event interval after the final observed event?

    Not when the observation window end is known and informative. In a temporal likelihood, survival from the last event to the window boundary is part of the probability of observing no further event. Omitting it changes the target law and can bias rate estimates.

    Chapter 1 retrieval questions¤

    1. What does local finiteness exclude?
    2. Under this volume's Janossy convention, how are \(j_n\), \(p_n\), and \(f_n\) related?
    3. Which assumption lets a finite simple point process be represented as a random finite set?
    Complete solutions
    1. It excludes infinitely many points in a bounded observation region, almost surely.
    2. \(j_n=n!p_nf_n\), where \(f_n\) is a symmetric ordered-tuple density integrating to one.
    3. Finiteness in the observation window and simplicity, so coincident multiplicities do not need to be retained.

    Chapter 1 synthesis and bridge¤

    A point process is a probability law on counting measures or configurations, not merely a predictor for the next coordinate. Cardinality, ordering conventions, reference measures, and empty-space probabilities are part of the object. Chapter 2 specializes these semantics to continuous time, where survival turns a conditional intensity into a normalized next-event law.

    2. Temporal Point Processes: Intensity, Survival, and History¤

    Chapter 2 prerequisites and learning objectives¤

    This chapter assumes Chapter 1 and basic survival analysis.

    Numbered learning outcomes. After completing the chapter, you can:

    1. O1 — Interpret intensity predictably. Distinguish conditional intensity, marginal rate, integrated intensity, and compensator without allowing an event to predict itself.
    2. O2 — Derive normalized event laws. Move from intensity to survival, next-event density, terminal survival, and the complete finite-window likelihood.
    3. O3 — Compare temporal constructions. Explain how Poisson, renewal, Hawkes, and self-correcting processes use time and history differently.
    4. O4 — Compute a Hawkes model. Evaluate event intensities, the compensator, branching ratio, and the assumption-bounded stationary mean.

    Conditional intensity and compensator¤

    For a simple temporal point process with history \(\mathcal H_t\) immediately before \(t\), the predictable conditional intensity is informally

    \[ \lambda(t\mid\mathcal H_t) =\lim_{\Delta\downarrow0} \frac{\Pr\{N(t,t+\Delta]=1\mid\mathcal H_t\}}{\Delta}, \]

    provided the regularity conditions for the limit hold. Predictability matters: an event at \(t\) cannot be used to predict itself. The integrated intensity or compensator increment is

    \[ \Lambda(a,b\mid\mathcal H) =\int_a^b\lambda(s\mid\mathcal H_s)\,ds. \]

    Under the standard integrability and non-explosion conditions, \(N(t)-\Lambda(0,t)\) is a local martingale. The compensator is history dependent in general; writing \(\Lambda(t)\) does not make it a deterministic cumulative rate.

    Survival derives the next-event density¤

    Given the history through the previous event time \(t_i\), the probability of no event before \(t>t_i\) is

    \[ S(t\mid\mathcal H_{t_i}) =\exp\!\left[-\int_{t_i}^{t}\lambda(s\mid\mathcal H_s)\,ds\right]. \]

    The next-event density is therefore

    \[ f(t\mid\mathcal H_{t_i}) =\lambda(t\mid\mathcal H_t)S(t\mid\mathcal H_{t_i}). \]

    For observed events \(0<t_1<\cdots<t_n\le T\) with a known observation endpoint \(T\), the log likelihood is

    \[ \boxed{ \log p(t_{1:n}\mid T) =\sum_{i=1}^{n}\log\lambda(t_i\mid\mathcal H_{t_i}) -\int_0^T\lambda(s\mid\mathcal H_s)\,ds.} \]

    The integral is not an optional regularizer. It is the survival contribution, including the no-event interval \((t_n,T]\). For discrete marks \(m_i\), a common decomposition uses a ground intensity \(\lambda(t\mid\mathcal H_t)\) and a normalized mark law \(q(m\mid t,\mathcal H_t)\), adding \(\sum_i\log q(m_i\mid t_i,\mathcal H_{t_i})\) to the log likelihood.

    The intensity-to-likelihood chain¤

    This chain is the continuous-time analogue of an autoregressive model with a stop decision. The stop probability is survival to the window boundary, and the compensator is what normalizes all possible event and no-event outcomes.

    Four classical constructions¤

    Poisson processes¤

    A homogeneous Poisson process has \(\lambda(t\mid\mathcal H_t)=\lambda\), independent stationary increments, and \(N(0,T]\sim\operatorname{Poisson}(\lambda T)\). A nonhomogeneous Poisson process replaces the constant by a deterministic \(\lambda(t)\); disjoint increments remain independent.

    Renewal processes¤

    Let \(\tau_i=t_i-t_{i-1}\) be iid interarrival times with density \(g\), distribution \(G\), survival \(\bar G=1-G\), and hazard \(h(\tau)=g(\tau)/\bar G(\tau)\). Then

    \[ \lambda(t\mid\mathcal H_t)=h(t-t_{N(t)}). \]

    The exponential gap distribution gives a Poisson process. A non-exponential renewal process remembers the age since the last event but not the full history. Stationary renewal initialization requires an equilibrium age distribution; starting with an event at time zero is a different boundary condition.

    Linear Hawkes processes¤

    A univariate linear Hawkes process uses

    \[ \lambda(t\mid\mathcal H_t) =\mu+\sum_{t_i<t}g(t-t_i), \qquad g(u)\ge0. \]

    With \(g(u)=\alpha e^{-\beta u}\), each event temporarily raises future intensity. The branching ratio is

    \[ \eta=\int_0^\infty g(u)\,du=\frac{\alpha}{\beta}. \]

    For this nonnegative linear stationary construction, \(\eta<1\) is the subcritical condition and the stationary mean intensity is \(\mu/(1-\eta)\). This statement does not transfer unchanged to nonlinear, signed-kernel, finite-horizon, or nonstationary Hawkes variants. In a multivariate process, the matrix of kernel integrals replaces \(\eta\) and a spectral-radius condition is used under the corresponding assumptions.

    Self-correcting processes¤

    The exponential self-correcting form of Isham and Westcott (1979) is

    \[ \lambda(t\mid\mathcal H_t)=\exp\{\mu t-\alpha N(t^-)\}, \qquad \mu,\alpha>0. \]

    Intensity rises between events and drops multiplicatively after each event, encouraging a more regular count trajectory. It is history dependent but inhibitory rather than self-exciting.

    Complete worked example: Hawkes likelihood and stability¤

    Take \(T=2\), events \(t_1=0.5\) and \(t_2=1.25\), and

    \[ \lambda(t)=0.5+0.8\sum_{t_i<t}e^{-2(t-t_i)}. \]

    The event intensities use only prior events:

    \[ \lambda(t_1)=0.5, \qquad \lambda(t_2)=0.5+0.8e^{-2(0.75)}\approx0.6785. \]

    The compensator is

    \[ \begin{aligned} \Lambda(0,2) &=0.5(2)+\frac{0.8}{2} \sum_{i=1}^{2}\left[1-e^{-2(2-t_i)}\right]\\ &=1+0.4\left[(1-e^{-3})+(1-e^{-1.5})\right]\\ &\approx1.6908. \end{aligned} \]

    Thus

    \[ \log p(t_1,t_2\mid T=2) =\log0.5+\log0.6785-1.6908 \approx-2.7718. \]

    The branching ratio is \(0.8/2=0.4<1\), so the corresponding infinite-horizon stationary linear model is subcritical and has mean intensity \(0.5/(1-0.4)=5/6\). The finite-window likelihood calculation and the stationary claim are related but logically separate.

    Autoregressive factorization without semantic collapse¤

    The survival-derived conditional densities yield

    \[ p(t_{1:n},N(T)=n) =\prod_{i=1}^n f(t_i\mid\mathcal H_{t_{i-1}}) \;S(T\mid\mathcal H_{t_n}). \]

    This is an autoregressive factorization, but it includes a terminal survival factor and continuous-time densities. Discretizing time into tokens changes the base measure and may merge simultaneous events, introduce bin-size dependence, or condition implicitly on a maximum length. The two models can approximate each other under a declared limiting argument; they are not identical by notation alone.

    Chapter 2 misconception checks¤

    Does a large Hawkes kernel coefficient prove that one event type causes another?

    No. It is a predictive dependence parameter relative to the model and observed filtration. Latent common causes, missing events, observation bias, nonstationarity, or misspecified kernels can produce excitation-like associations. A causal claim needs an intervention semantics and identification assumptions.

    Is \(\eta<1\) required for every finite-horizon Hawkes likelihood?

    No. It is the standard subcritical stationarity condition for nonnegative linear Hawkes processes on an unbounded horizon. A finite observed path can have a defined likelihood outside that stationary regime, but explosion, extrapolation, and interpretation require separate analysis.

    First-order intensity does not determine interaction¤

    For a spatial point process, the intensity measure is

    \[ \nu(A)=\mathbb E[N(A)]. \]

    If \(\nu\) has a density with respect to \(\mu\), then \(\nu(A)=\int_A\rho(x)\,\mu(dx)\). The function \(\rho\) answers an expected-count question. It does not by itself say whether points cluster, repel, or occur independently. Second-order product densities, pair-correlation functions, empty-space probabilities, and the full configuration law provide different information.

    The distinction is easiest to see at a fixed constant intensity:

    • a Poisson process places points independently;
    • a Cox process can cluster them through a shared random intensity;
    • a Gibbs or determinantal process can inhibit close configurations;
    • a cluster process can create parent–offspring groups.

    All four may have the same first-order expected density after parameters are matched.

    Same intensity, different geometry¤

    The top node is deliberately shared: an intensity map can be perfect while the model's burstiness, gaps, or repulsion is wrong. The lower diagnostics are projections of the law, so several complementary checks are needed.

    Poisson, Cox, and cluster constructions¤

    A Poisson point process with locally finite intensity measure \(\nu\) is defined by

    \[ N(A)\sim\operatorname{Poisson}(\nu(A)) \]

    for measurable \(A\), with independent counts on disjoint regions. On a bounded window with intensity density \(\rho\), its Janossy densities are

    \[ j_n(x_{1:n}) =\exp[-\nu(W)]\prod_{i=1}^n\rho(x_i). \]

    A Cox process first draws a random locally finite intensity measure \(\Xi\), then draws a Poisson process conditional on \(\Xi\). A log-Gaussian Cox process sets \(\rho(x)=\exp Z(x)\) for a Gaussian random field \(Z\), following the construction studied by Møller, Syversveen, and Waagepetersen (1998). Conditional independence given \(Z\) becomes marginal dependence after \(Z\) is integrated out.

    A Poisson cluster process first samples parent points and then a random offspring configuration around each parent. The observed process can omit the parents. A Hawkes process has a related immigration–branching construction under its nonnegative linear assumptions, but temporal causality in the construction and spatial cluster geometry should not be conflated.

    Gibbs processes and Papangelou conditional intensity¤

    On a bounded window, a Gibbs point process often has a density of the form

    \[ f(X)=\frac{1}{Z(\theta)}\exp[-U_\theta(X)] \]

    with respect to a declared Poisson reference process. The energy \(U_\theta\) can penalize close pairs, enforce hard cores, or reward selected structures. The partition function \(Z(\theta)\) generally depends on the parameters and can be intractable.

    For a hereditary density, the Papangelou conditional intensity can be written informally as the density ratio

    \[ \lambda_{\mathrm P}(u\mid X) =\frac{f(X\cup\{u\})}{f(X)}, \]

    whenever the denominator is positive and the reference conventions are fixed. It describes the relative propensity to insert a point at \(u\) given the rest of the configuration. It is not the same object as the temporal predictable intensity, although both are called conditional intensities. Spatial pseudolikelihood methods exploit \(\lambda_{\mathrm P}\) because the global normalizer cancels in this ratio.

    Determinantal point processes¤

    A determinantal point process (DPP) models diversity or repulsion through determinants. On a finite ground set \(\mathcal Y=\{1,\ldots,M\}\), an \(L\)-ensemble uses a positive-semidefinite matrix \(L\) and assigns

    \[ \Pr(X=Y)=\frac{\det(L_Y)}{\det(I+L)}, \qquad Y\subseteq\mathcal Y, \]

    where \(L_Y\) is the principal submatrix indexed by \(Y\) and \(\det(L_\varnothing)=1\). The marginal kernel is

    \[ K=L(I+L)^{-1}, \]

    and inclusion probabilities satisfy \(\Pr(A\subseteq X)=\det(K_A)\). Positive semidefiniteness alone does not make every arbitrary continuous kernel a valid trace-class DPP on an unbounded domain; existence conditions depend on the state space and operator.

    Let

    \[ L=\begin{bmatrix}2&1\\1&2\end{bmatrix}. \]

    Then \(\det(I+L)=\det\bigl[\begin{smallmatrix}3&1\\1&3\end{smallmatrix}\bigr]=8\). The four subset masses are

    Random finite sets¤

    A random finite set (RFS) is a random variable whose value is a finite subset of a state space. On finite simple configurations, point-process and RFS descriptions can represent the same laws. The vocabulary differs because finite-set statistics emphasizes multi-object state estimation, set integrals, cardinality distributions, and recursive Bayes filtering.

    Important finite-set families include:

    • Bernoulli RFS: empty with probability \(1-r\); otherwise a singleton with spatial density \(p(x)\);
    • multi-Bernoulli RFS: a union of independent Bernoulli components, with care about collisions and labels;
    • Poisson RFS: Poisson cardinality and iid locations conditional on cardinality;
    • iid-cluster RFS: an arbitrary cardinality law and iid conditional locations;
    • labeled RFS: each element includes a unique label, making identity and trajectory queries explicit.

    For a Bernoulli RFS,

    \[ F(\varnothing)=1-r, \qquad F(\{x\})=r\,p(x), \qquad F(X)=0\ \text{for }|X|>1 \]

    under the set-integral convention of Chapter 1. Normalization follows from \((1-r)+r\int p(x)\,d\mu(x)=1\).

    Point clouds, sets, and multisets¤

    Permutation invariance is necessary for an unlabeled set law but not sufficient to specify one. Three choices remain:

    1. Cardinality: fixed, conditioned, or generated;
    2. Coincidence: forbidden, allowed as multiplicity, or represented by marks;
    3. Identity: unlabeled, labeled, or quotient by a symmetry group.

    A model trained on padded point clouds can represent a point process if it supplies a coherent mask/count law and does not let padding symbols leak arbitrary ordering. A labeled tracking state is not permutation invariant in the same way as an unlabeled spatial pattern; label permutations and object identities require their own convention.

    Does a DPP always generate a fixed-size diverse subset?

    No. An ordinary \(L\)-ensemble has random cardinality. A \(k\)-DPP conditions on \(|X|=k\). Conditioning changes the law and the normalizer.

    Is every random finite set a simple point process?

    With appropriate measurability, an RFS corresponds to a finite simple point process because a mathematical set cannot retain duplicate elements. A non-simple point process with multiplicities needs a counting-measure or marked representation rather than an ordinary set.

    Exact likelihood and numerical integration¤

    For a temporal intensity \(\lambda_\theta\), the negative log likelihood is

    \[ \mathcal L(\theta) =-\sum_{i=1}^{n}\log\lambda_\theta(t_i\mid\mathcal H_{t_i}) +\int_0^T\lambda_\theta(s\mid\mathcal H_s)\,ds. \]

    If the integral has no closed form, deterministic quadrature introduces discretization error. Uniform Monte Carlo points \(U_k\sim\operatorname{Uniform}(0,T)\) give

    \[ \widehat\Lambda_\theta(0,T) =\frac{T}{K}\sum_{k=1}^K \lambda_\theta(U_k\mid\mathcal H_{U_k}), \]

    an unbiased estimator of the compensator integral under integrability. This makes the displayed log-likelihood estimator unbiased when the event log intensities are exact, but exponentiating it does not generally produce an unbiased likelihood estimator. Gradients require the same care about random nodes, variance, and history discontinuities.

    Complete worked example: a nonhomogeneous Poisson MLE¤

    Let \(\lambda_\theta(t)=\theta(1+t)\) on \([0,1]\) and observe \(n=3\) events. Then

    \[ \ell(\theta) =3\log\theta+\sum_{i=1}^{3}\log(1+t_i) -\theta\int_0^1(1+t)\,dt =3\log\theta+C-\frac32\theta. \]

    Differentiating gives

    \[ \frac{d\ell}{d\theta}=\frac3\theta-\frac32, \qquad \widehat\theta=2. \]

    The estimate depends on the count but not the individual times because \(\theta\) is only a global scale. The times identify the fixed shape \(1+t\) only if that shape is also parameterized. Omitting the compensator would make the objective increase without bound as \(\theta\to\infty\).

    Branching augmentation for linear Hawkes processes¤

    Under a nonnegative linear Hawkes model, an event can be represented as either an immigrant from the baseline or an offspring of one earlier event. Latent parent indicators turn the complete-data likelihood into weighted baseline and offspring terms. An EM algorithm alternates between:

    1. posterior probabilities that each event is an immigrant or child of each valid predecessor; and
    2. parameter updates for the baseline and kernel.

    This augmentation is exact for the declared Poisson-cluster representation. It is not available unchanged for arbitrary nonlinear or inhibitory neural intensities, and a high posterior parent probability is not an identified real-world causal parent.

    Latent intensities and random measures¤

    For a Cox process,

    \[ p(X)=\int p(X\mid\Xi)\,p(d\Xi). \]

    The conditional Poisson likelihood is tractable once the random intensity \(\Xi\) is known, but integrating the field can be difficult. Log-Gaussian Cox models combine a Gaussian-field prior with a non-Gaussian point-pattern likelihood, motivating grid approximations, Laplace methods, MCMC, or variational inference. Discretizing space changes the numerical representation, not the underlying continuous-space model, only to the extent that convergence and cell approximations are controlled.

    Bayesian nonparametric random measures can also drive Poisson or Cox constructions. A completely random measure may supply random atom weights; normalization, if any, changes the generated object. The same random-measure prior can support point processes, feature allocations, or mixture models through different likelihoods, so prior vocabulary alone does not determine the family.

    Gibbs likelihood, pseudolikelihood, and composite objectives¤

    For a Gibbs density \(f_\theta(X)=\widetilde f_\theta(X)/Z(\theta)\), the normalizer usually prevents direct likelihood evaluation. The spatial log pseudolikelihood based on a Papangelou intensity has the schematic form

    \[ \log\operatorname{PL}(\theta;X) =\sum_{x_i\in X}\log\lambda_{\mathrm P,\theta}(x_i\mid X\setminus\{x_i\}) -\int_W\lambda_{\mathrm P,\theta}(u\mid X)\,du. \]

    It resembles a temporal likelihood but has different conditioning semantics. Pseudolikelihood is an alternative estimating objective, not an exact rewrite of the global configuration likelihood. Consistency and efficiency depend on the asymptotic regime, model, window growth, and regularity assumptions.

    Observation mechanisms: censoring is part of the model¤

    Point-pattern data are commonly incomplete:

    • right censoring: the observation window ends after the last recorded event;
    • left truncation: the history before the start is unknown;
    • interval censoring or rounding: event times are known only within bins;
    • dependent thinning: detection probability depends on location, mark, or latent state;
    • dead time: nearby events cannot both be recorded;
    • missing marks: event occurrence is observed but an attribute is absent;
    • superposition: events from several sources are observed without source labels.

    Ignoring these mechanisms can alter both intensity and interaction. For example, independent thinning of a Poisson process remains Poisson with a reduced intensity, but dependent thinning can create apparent repulsion. Left-truncated Hawkes data omit ancestors and can distort excitation estimates unless the initial history is modeled or a burn-in approximation is justified.

    Identifiability and causal restraint¤

    Likelihood identifiability asks whether different parameters induce the same observed law. Causal identification asks whether an interventional quantity follows from the observational law plus assumptions. They are not the same question.

    Common confounders include unobserved event types, shared latent intensity, time-varying exposure, spatial covariates, reporting delays, and feedback from interventions. A Hawkes influence matrix, attention map, or learned parent distribution is observational unless an intervention model and identification conditions are added. Predictive ablation can diagnose model reliance; it does not by itself establish causal effect.

    Three laws, not one dataset¤

    The latent law, recording law, and inference approximation are three separate objects. A flexible optimizer cannot recover events that the observation model declares unknowable, and a surrogate objective should not be relabeled as exact likelihood after optimization.

    Chapter 4 misconception checks¤

    If a Monte Carlo compensator estimate is unbiased, is the exponentiated likelihood estimate unbiased?

    No. In general \(\mathbb E[e^{-\widehat\Lambda}]\ne e^{-\mathbb E[\widehat\Lambda]}\). Unbiasedness does not survive a nonlinear transformation. Pseudo-marginal methods require a nonnegative unbiased estimator of the likelihood itself, not merely of its logarithm.

    Does a lower training NLL prove better point-process generation?

    No. It establishes an in-sample objective comparison under a particular observation and reference convention. Count calibration, mark calibration, residual structure, held-out windows, long-horizon rollouts, leakage, and domain utility still require evaluation.

    One target, several possible samplers¤

    A sampler is exact only relative to assumptions. The same point-process law may admit several algorithms, and the same algorithmic pattern can target different laws. Runtime, numerical error, mixing, and approximation bias are properties of the full model–sampler pair.

    Count first, then locate points¤

    For a finite Poisson process on \(W\) with intensity measure \(\nu\):

    1. draw \(N\sim\operatorname{Poisson}(\nu(W))\);
    2. draw \(N\) iid locations from \(\nu(dx)/\nu(W)\).

    This is exact when \(0<\nu(W)<\infty\). If \(\nu(W)=0\), return the empty configuration. For an iid-cluster RFS, replace the Poisson count with its declared cardinality law. For non-Poisson processes, conditioning on \(N\) generally does not make locations iid.

    Inverse integrated hazard¤

    Suppose the history is fixed after event \(t_i\) until the next event and the cumulative hazard

    \[ A_i(t)=\int_{t_i}^{t}\lambda(s\mid\mathcal H_s)\,ds \]

    can be evaluated and inverted. Draw \(E\sim\operatorname{Exp}(1)\) and solve \(A_i(t_{i+1})=E\). If the solution lies beyond the observation endpoint, stop and retain the corresponding survival event.

    Complete worked example: nonhomogeneous Poisson inversion¤

    Let \(\lambda(t)=1+t\) on \([0,1]\). Its cumulative intensity is

    \[ \Lambda(t)=t+\frac{t^2}{2}, \qquad \Lambda^{-1}(s)=-1+\sqrt{1+2s}. \]

    Suppose exponential increments are \(0.4\), \(0.7\), and \(0.6\). Their cumulative sums are \(0.4\), \(1.1\), and \(1.7\). The first two event times are

    \[ t_1=-1+\sqrt{1.8}\approx0.3416, \qquad t_2=-1+\sqrt{3.2}\approx0.7889. \]

    Because \(\Lambda(1)=1.5\) and the third transformed time is \(1.7>1.5\), no third event occurs before the window end. Stopping is part of the generated law, not an external maximum-length rule.

    Ogata-style thinning¤

    When inversion is unavailable, thinning simulates candidate events from a dominating Poisson process. Given current time \(s\) and history:

    1. choose a predictable bound \(M(u)\) satisfying \(M(u)\ge\lambda(u\mid\mathcal H_u)\) until the next bound update;
    2. propose a candidate from the process with intensity \(M\);
    3. accept a candidate at \(u\) with probability \(\lambda(u\mid\mathcal H_u)/M(u)\);
    4. update history and bounds after accepted or rejected proposals as required.

    The classical efficient construction is due to Ogata (1981). A constant local bound gives exponential candidate gaps; a nonconstant bound may itself require inversion. If the bound is ever too small, clipping the acceptance probability to one does not repair exactness—it silently biases the process. Loose bounds preserve correctness but waste proposals.

    For multivariate marked intensities \(\lambda_k(t)\), one may dominate the total intensity \(\sum_k\lambda_k(t)\), accept the time, and then sample mark \(k\) with probability proportional to \(\lambda_k(t)\), provided the decomposition and bound are valid.

    Thinning as an executable correctness argument¤

    The diagram is also a test plan: instrument the bound check, acceptance ratio, accepted history, proposal clock, and endpoint. A generated sample without these invariants may look plausible while targeting a different law.

    Superposition, branching, and cluster sampling¤

    Independent point processes can be superposed by taking the union or sum of their counting measures. Poisson superposition remains Poisson with summed intensity. Independent thinning labels each Poisson event into subprocesses and gives independent Poisson processes under the standard marking assumptions.

    For a subcritical nonnegative linear Hawkes process, cluster simulation draws immigrant events from the baseline process and recursively generates offspring with delay density proportional to the excitation kernel. The expected total cluster size is \(1/(1-\eta)\) in the univariate stationary branching construction. Finite windows require ancestors before the left boundary or a controlled approximation; simply starting empty changes the boundary law.

    Spectral DPP sampling¤

    For a finite DPP with marginal kernel \(K=V\operatorname{diag}(\kappa_i)V^\top\), \(0\le\kappa_i\le1\):

    1. independently retain eigenvector \(v_i\) with probability \(\kappa_i\);
    2. repeatedly select an item with probability proportional to the squared row norm in the retained subspace;
    3. project the subspace orthogonally away from that selected coordinate and continue until the subspace is empty.

    This yields the DPP exactly up to numerical linear-algebra error. A \(k\)-DPP instead selects a size-\(k\) eigenvector subset according to elementary symmetric polynomials before the item-selection phase. Low-rank approximations trade accuracy for cost and must be evaluated as approximations to the kernel law.

    Birth–death chains and configuration MCMC¤

    For a target density \(f(X)\) on finite configurations, a Markov chain can propose:

    • a birth \(X\to X\cup\{u\}\);
    • a death \(X\to X\setminus\{x\}\);
    • a move replacing one location or mark;
    • a split, merge, or block update.

    Metropolis–Hastings acceptance must include the target density ratio, proposal densities, and combinatorial forward/reverse factors. For Gibbs processes, local energy differences often make ratios computable even when the global partition function is not. Detailed balance is sufficient for invariance, not proof of fast mixing; multimodal cardinality or geometry can trap local chains.

    Learned add–thin and configuration transitions¤

    Add and Thin constructs a noising process on point configurations by adding Poisson noise and thinning existing points, then learns reverse transitions. This preserves a variable-cardinality configuration state rather than padding it to a fixed vector. It is a discrete/continuous iterative sampler composed with point-process semantics, not a replacement definition of those semantics.

    Other configuration-valued transition models may use spatial birth–death processes, jump generators, transport maps conditional on cardinality, or diffusion-like moves over attributes. A transition loss determines the intended target only with its endpoint law, time reversal or matching theorem, discretization, and terminal sampler specified.

    Sampler validation contract¤

    For a known finite case, verify at least:

    1. count probabilities, including the empty set;
    2. first-order intensity or inclusion marginals;
    3. pair or higher-order statistics that detect interaction;
    4. mark frequencies and conditional dependence;
    5. invariance to arbitrary input ordering for unlabeled sets;
    6. boundary and stopping behavior;
    7. convergence with tighter numerical tolerances, more proposals, or longer chains.

    A sampler can match the mean count while missing repulsion, clustering, or mark dependence. Visual plausibility is therefore a weak diagnostic.

    3. P1.3. \(\nu(W)=1\), so \(N\sim\operatorname{Poisson}(1)\), \(\Pr(N=0)=e^{-1}\), and conditional locations have density \(2x\) on \([0,1]\). Draw the count, then draw that many iid locations—for example by \(x=\sqrt U\) with \(U\) uniform. Check. \(\int_0^1 2x\,dx=1\).
    4. P1.4. The empty-selection probability is \((0.8)(0.3)=0.24\), both-selection probability is \((0.2)(0.7)=0.14\), and exactly-one probability is \(1-0.24-0.14=0.62\). The expected cardinality is \(0.2+0.7=0.9\). Check. \(0(0.24)+1(0.62)+2(0.14)=0.9=\operatorname{tr}K\).
    5. P1.5. The forward proposal probability is \(b/(5-2)=(1/2)/3\), while the reverse is \(d/3=(1/2)/3\). Their ratio is one, so the Hastings ratio is the target ratio \(2\) and the acceptance probability is \(\min(1,2)=1\). Check. The two denominators happen to match only because both the absent and post-birth present sets have size three.
    6. P1.6. First enumerate a tiny marked target and compare empirical mass for every configuration with simultaneous confidence control. At scale, test count and empty mass, intensity, pair/nearest-neighbor summaries, conditional mark dependence, and invariance to input order. Repeat across reverse-step counts and discretizations; report proposals, accepted changes, wall time, and memory. Predeclare failure when exact finite confidence intervals miss target mass or refinement fails to stabilize the declared metrics. Rubric. Full credit requires all requested statistical, invariance, convergence, cost, and falsification layers.

    Chapter 5 misconception checks¤

    Can thinning remain exact if the intensity bound fails only rarely?

    No. Any underbound makes the nominal acceptance probability invalid and changes the law. Empirical rarity is not a correctness proof. Use an analytically valid envelope, adaptive bound with proven validity, or a different sampler.

    Does an MCMC acceptance rate near 0.234 certify a good point-process sampler?

    No. Such heuristics arise in specific asymptotic random-walk settings. Mixing depends on proposal geometry, dimension, cardinality moves, interactions, and modes. Inspect effective sample sizes, between-chain behavior, count transitions, and known finite targets.

    Intensity-based sequence models¤

    RMTPP uses a recurrent history embedding to parameterize timing and marks. Neural Hawkes uses a continuous-time LSTM state with event-driven jumps and continuous decay. Self-Attentive Hawkes Process replaces recurrent history aggregation with self-attention and time-aware encoding. These are architectural choices inside a declared intensity model; their names do not guarantee Hawkes branching semantics.

    Fully Neural Network Based Models for General Temporal Point Processes parameterize a monotone cumulative hazard so its derivative supplies the intensity and the compensator is evaluated directly. Monotonicity, positivity, differentiability, and tail behavior are structural constraints, not cosmetic activation choices.

    Intensity-free conditional distributions¤

    Point Process Flows parameterizes conditional inter-event distributions using normalizing flows rather than choosing an intensity form. This can give normalized gap densities and reparameterized samples, but the model still needs a coherent mark law, observation-window convention, and terminal survival probability. “Intensity free” means the implementation does not use intensity as its primary parameterization; it does not remove point-process probability semantics.

    For space–time events, Neural Spatio-Temporal Point Processes combine continuous-time neural dynamics and conditional spatial flows. Factorizing time and location can ease normalization, but it imposes an ordering of conditionals whose adequacy must be tested on joint spatial–temporal structure.

    Configuration-valued models and a score limitation¤

    Configuration transition models operate on the whole random finite set and can express parallel births, deaths, or moves. Their advantage over next-event factorization depends on cardinality accuracy, interaction recovery, simulation cost, and finite-step approximation—not parallelism alone.

    Preprint. Score Matching for Estimating Finite Point Processes formalizes scores relative to Janossy measures and shows an important boundary: a spatial score alone need not identify a general nonparametric finite point-process law because it can omit cardinality or survival information. The paper augments score matching with a survival-classification term. This is current preprint evidence, not a settled universal impossibility theorem for every restricted family or reference convention.

    In-context inference and constraint-aware generation¤

    Peer-reviewed research. FIM-PP, accepted at ICLR 2026, amortizes inference of temporal point-process intensity functions from example sequences. It changes the inference interface: a pretrained estimator can adapt in context without per-dataset optimization. It does not change the underlying distinction among Poisson, Hawkes, renewal, or other target laws, and its uncertainty must be evaluated under family shift.

    Peer-reviewed research. Forward-Chaining Temporal Point Process, published as an ICML 2026 regular paper, combines continuous-time event generation with a latent symbolic state and differentiable multi-step rules for constraint-aware generation. Separate non-archival workshop versions also exist, so the venue claim here refers specifically to the linked regular-paper record. Constraint satisfaction, event-law calibration, and the validity of the rule layer are separate evaluation axes.

    Peer-reviewed research. Towards Reliable Detection of Empty Space, accepted at ICLR 2026, treats object detections as a marked spatial point process and emphasizes empty-space probabilities. It illustrates why evaluating only matched detected objects can ignore the probability assigned to regions containing no object.

    Preprint/submission. Asynchronous diffusion and speculative multi-event proposals have also been proposed for temporal point processes. Until venue decisions, assumptions, and independent comparisons are established, they remain provisional compositions rather than canonical replacements for intensity, flow, or thinning methods.

    Likelihood is necessary but not sufficient¤

    Held-out point-process log likelihood must report:

    • the observation window and boundary history;
    • continuous-time or discretized reference measure;
    • whether the terminal survival term is included;
    • mark factorization and missing-mark treatment;
    • compensator approximation and tolerance;
    • normalization per event, per sequence, per unit exposure, or per area;
    • treatment of empty sequences and zero-event windows.

    Per-event NLL can hide count errors by excluding empty windows. Per-window NLL can be dominated by exposure length. Report both when they answer different questions.

    Time rescaling and residual analysis¤

    For a correctly specified simple temporal point process under the standard regularity, completeness, and non-termination assumptions, transformed inter-event compensator increments

    \[ z_i=\int_{t_{i-1}}^{t_i} \lambda(s\mid\mathcal H_s)\,ds \]

    are iid \(\operatorname{Exp}(1)\), and

    \[ u_i=1-e^{-z_i} \]

    are iid uniform. The classical diagnostic is developed by Brown et al. (2002). A 2025 generalized time-rescaling theorem makes clear that terminating processes and incomplete realizations need additional treatment rather than automatic application of the standard theorem.

    Complete worked diagnostic¤

    Suppose a fitted model yields compensator increments

    \[ z=(0.2,\ 1.0,\ 2.0). \]

    Then

    \[ u=1-e^{-z}\approx(0.1813,\ 0.6321,\ 0.8647). \]

    These three values are consistent with the transformation but far too few for a useful uniformity or independence judgment. A responsible analysis pools comparable held-out sequences only when their conditioning and censoring conventions match, plots transformed residuals, tests serial dependence as well as marginal uniformity, and checks the terminal censored interval separately.

    Time rescaling is not an omnibus proof of correctness. A test can have low power, parameter fitting affects null distributions, and passing temporal residuals does not validate marks or spatial structure.

    From a score to a defensible claim¤

    Moving downward adds a different falsification opportunity. No single rung can be substituted for all later ones: good NLL does not prove empty-space calibration, and one benchmark reversal does not rank model families universally.

    Controlled comparison protocol¤

    A fair family comparison matches:

    1. training windows, exposure, missingness, and marks;
    2. boundary histories and censoring treatment;
    3. conditioning information and external covariates;
    4. likelihood/reference conventions;
    5. parameter, active-compute, and sampling budgets;
    6. numerical accuracy for compensators, determinants, or chains;
    7. domain baselines and leakage controls;
    8. both predictive and generative metrics.

    EarthquakeNPP, published in TMLR in 2026, provides useful counterevidence: after correcting data leakage and evaluation issues, a classical ETAS baseline remained competitive with or better than evaluated neural point-process systems in that setting. The correct conclusion is bounded to its protocol and domain. It supports retaining strong mechanistic baselines, not declaring neural point processes generally ineffective.

    Chapter 6 misconception checks¤

    If transformed residuals look uniform, is the fitted model correct?

    No. Uniformity is one necessary implication under stated assumptions. Independence, terminal censoring, marks, spatial structure, test power, parameter estimation, and alternative misspecifications still matter.

    Does an in-context point-process model define a new generated-object family?

    No. It is an inference architecture over a collection of point-process laws. The generated object remains a configuration; the family and reference conventions must still be stated.

    1. What distinguishes intensity, cumulative-hazard, and intensity-free neural models?
    2. Why can spatial score matching miss part of a finite point-process law?
    3. What two properties should be checked after time rescaling, in addition to terminal censoring?

    Complete solutions
    1. They directly parameterize different normalized interfaces: a local hazard requiring integration, a monotone integral whose derivative is the hazard, or a normalized next-gap distribution. Each still needs marks, stopping, and observation-window semantics.
    2. A location score can be invariant to cardinality-dependent factors and therefore fail to identify count or survival mass without an additional term or restricted family.
    3. The transformed increments should have the correct marginal distribution and be independent.
    4. Under its leakage-corrected earthquake protocol, the classical ETAS baseline remained essential and could outperform evaluated neural alternatives; this does not establish a universal ranking.
