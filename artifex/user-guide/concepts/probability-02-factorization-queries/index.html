    3. Chain Rule and Autoregressive Factorization¤

    Factorization, information, and query learning objectives¤

    This projection assumes joint and conditional probability. By the end, you should be able to derive the chain rule under an ordering, distinguish a probability identity from an autoregressive architecture, compute discrete entropy, cross-entropy, KL, and code length, explain coordinate dependence of differential entropy, and separate generation, density estimation, conditional prediction, representation, and decision queries.

    The chain rule turns one joint question into a sequence of conditional questions. For three variables, one possible reading order is

    The arrows visualize the selected factorization, not a claim that \(X_1\) causes \(X_2\) or \(X_3\). Another order gives another exact factorization of the same joint law.

    Repeated application of the product rule gives, for any ordering \(\pi\) of \(D\) variables,

    \[ p(x_1,\ldots,x_D) =\prod_{i=1}^{D}p\!\left(x_{\pi(i)}\mid x_{\pi(<i)}\right). \]

    This is an identity for every joint distribution, not an assumption that data are intrinsically sequential. An autoregressive model chooses an ordering and parameterizes those conditionals. The ordering affects architecture, inductive bias, sampling cost, and approximation error even though the exact chain rule is valid for all orderings.

    For a graph or set, one can factor a distribution over a construction sequence, but several trajectories may lead to the same terminal object. The object probability is the sum over those trajectories unless a canonical ordering makes the map one-to-one. This distinction becomes central in GFlowNets and order-agnostic generation.

    4. Entropy, Cross-Entropy, and KL Divergence¤

    The coding question

    If an outcome is surprising under \(q\), it needs a long ideal description; if it is common, it needs a short one. Entropy is the best expected length when the source law is known, cross-entropy is the expected length when a possibly wrong model supplies the code, and KL is the excess.

    For a discrete distribution \(p\),

    \[ H(p)=-\mathbb E_{x\sim p}[\log p(x)]. \]

    The cross-entropy of data distribution \(p\) under model \(q\) is

    \[ H(p,q)=-\mathbb E_{p}[\log q(x)], \]

    and decomposes as

    \[ H(p,q)=H(p)+D_{\mathrm{KL}}(p\|q). \]

    Because \(H(p)\) does not depend on model parameters, minimizing expected negative log-likelihood is equivalent at the population level to minimizing forward KL. Finite data replace the expectation by an empirical average, introducing estimation and generalization questions that the identity alone does not resolve.

    Differential Entropy Is Coordinate-Dependent¤

    For continuous variables, \(h(X)=-\int p(x)\log p(x)dx\) can be negative and changes under rescaling. If \(Y=aX\), then \(h(Y)=h(X)+\log|a|\). KL divergence remains invariant under a shared smooth bijection because the Jacobian terms cancel. Do not compare differential entropies across representations as if they were intrinsic uncertainty measures.

    Proper Scoring Rules¤

    Log loss is a strictly proper scoring rule: in the population limit, a forecaster minimizes expected score by reporting the true distribution. Other proper scores include the Brier score for categorical outcomes and the continuous ranked probability score for scalar distributions. A loss can be useful without being a proper score, but calibration claims then require separate evidence.

    From probability to code length¤

    Take logarithms in base two and suppose a binary prefix code represents each discrete symbol. No codeword may be the prefix of another, so a decoder can determine where one symbol ends without looking ahead. Its integer lengths \(\ell(x)\) must satisfy the Kraft inequality

    \[ \sum_x 2^{-\ell(x)}\leq 1. \]

    A probability model \(q\) suggests the ideal length

    \[ \ell_q(x)=-\log_2q(x). \]

    This length need not be an integer. Huffman coding finds an optimal integer prefix code for a known finite symbol distribution, while arithmetic, range, and asymmetric-numeral-system coders operate across sequences and can approach the sequence negative log-likelihood up to finite-precision, termination, and metadata overhead. Thus “\(-\log_2 q(x)\) bits” is an ideal coding quantity, not a promise that every individual file has exactly that many bits. This coding connection originates in Shannon's source-coding analysis.

    Worked prefix code: four directions¤

    Let a source emit four symbols with a dyadic distribution:

    Table PROB.1. Worked prefix code: four directions
    Symbol \(p(x)\) \(-\log_2p(x)\) Codeword
    north \(1/2\) 1 bit 0
    east \(1/4\) 2 bits 10
    south \(1/8\) 3 bits 110
    west \(1/8\) 3 bits 111

    The code is prefix-free and its expected length is

    \[ \mathbb E_p[\ell_p(X)] =\tfrac12(1)+\tfrac14(2)+\tfrac18(3)+\tfrac18(3) =1.75\ \text{bits}. \]

    Because the probabilities are powers of two, the code attains the entropy exactly. Now encode the same source with a uniform model \(q(x)=1/4\), which assigns a two-bit codeword to every symbol. The expected length becomes

    \[ \underbrace{H_2(p,q)}_{2.00\ \text{bits}} =\underbrace{H_2(p)}_{1.75\ \text{bits}} +\underbrace{D_{\mathrm{KL},2}(p\|q)}_{0.25\ \text{bits}}. \]

    Here the subscript 2 declares base-two logarithms. The extra quarter-bit per symbol is the expected price of using the wrong probability model. Over a sequence, an autoregressive model supplies a different \(q(x_i\mid x_{<i})\) at each position, so its total ideal length is the sum of token NLLs in bits. If it assigns \(q(x)=0\) to an event the source can emit, both log loss and ideal code length are infinite; practical codecs therefore require a support-safe probability table.

    Coding language has a declared scope

    The direct code-length interpretation applies to discrete messages and a specified lossless coding construction. A continuous log density depends on its reference measure and units, so it is not a standalone file length. Quantization or another discrete observation model must be declared first. Lossy coding additionally requires a distortion criterion and is developed in Rate–Distortion and Information Bottlenecks.

    The complete Information, Coding, and Generative Models volume continues from this finite example to mutual-information estimation, source and channel coding, learned codecs, generative reconstruction, information bottlenecks, MDL, and algorithmic-information boundaries.

    5. What a Generative Model May Be Asked to Do¤

    Return to the four opening questions and add two more: which internal representation is useful? and which structured answer obeys the constraints? The resulting task list is broader than unconditional sample generation.

    Table PROB.2. 5. What a Generative Model May Be Asked to Do
    Task Query Examples
    Sampling Draw \(x\sim p_\theta\) ancestral decoding, flow inversion, reverse diffusion, MCMC
    Density evaluation Evaluate or estimate \(p_\theta(x)\) AR likelihood, flow change of variables, VAE ELBO, AIS for EBMs
    Conditional generation Draw \(x\sim p_\theta(x\mid c)\) class/text guidance, conditional decoders, constrained generation
    Inference Estimate \(p_\theta(z\mid x)\) VAE encoder, message passing, SMC
    Imputation Draw missing variables conditioned on observed ones masked models, Gibbs sampling, circuits
    Representation learning Map observations to useful latent variables VAE encoders, tokenizer models, self-supervised features
    Structured prediction Model coupled outputs under constraints CRFs/EBMs, graph generators, causal models
    Constructive generation Execute a map, program, grammar, policy, or solver deterministic decoders, program generators, constraint solvers

    No family dominates all queries. A GAN may sample quickly while lacking a tractable density; a probabilistic circuit may answer exact conditionals while sacrificing unconstrained expressivity; a diffusion model may support flexible conditioning but require iterative sampling.

    Generation, Density Estimation, and Structured Prediction¤

    • Density estimation asks for a normalized probability law or a justified approximation to it.
    • Generation asks for samples whose distribution satisfies a target criterion; the density may be unavailable.
    • Conditional generation samples missing or target variables given context.
    • Structured prediction may return a point estimate, ranking, or constrained object without modeling all uncertainty.
    • Representation learning asks whether internal variables support specified downstream queries.
    • Constructive generation produces an object through declared operations; probability semantics exist only if the seeds, choices, or outputs have a declared law.

    One system may perform several tasks, but success on one is not evidence for all. A deterministic segmentation network is a structured predictor, not a generative model merely because its output is structured. A diffusion posterior sampler for an inverse problem is conditional generation, but it is calibrated only relative to its prior, likelihood, and numerical approximation.

    A Conditional Law Is Not a Decision Rule¤

    The expression \(p(y\mid c)\) describes uncertainty about an output after context \(c\) is known. It does not say whether a deployed system should sample, return a mode, minimize a task loss, abstain, or present several alternatives. Those are decision rules layered on top of the predictive law.

    Consider the finite conditional distribution

    Table PROB.3. A Conditional Law Is Not a Decision Rule
    outcome \(y\) probability \(p(y\mid c)\)
    \(0\) \(0.50\)
    \(1\) \(0.30\)
    \(4\) \(0.20\)

    An ancestral conditional draw returns \(0\), \(1\), and \(4\) with those frequencies. Under \(0\)--\(1\) loss, the Bayes action is the mode, \(a=0\). Under squared loss, the unrestricted Bayes action is instead the conditional mean,

    \[ \begin{aligned} a^*&=\mathbb E[Y\mid c]\\ &=0(0.50)+1(0.30)\\ &\qquad+4(0.20)\\ &=1.10. \end{aligned} \]

    If actions must belong to the candidate set \(\{0,1,4\}\), their conditional squared risks are

    \[ \begin{aligned} R(0)&=0.50(0-0)^2\\ &\quad+0.30(0-1)^2\\ &\quad+0.20(0-4)^2=3.50,\\[2pt] R(1)&=0.50(1-0)^2\\ &\quad+0.30(1-1)^2\\ &\quad+0.20(1-4)^2=2.30,\\[2pt] R(4)&=0.50(4-0)^2\\ &\quad+0.30(4-1)^2\\ &\quad+0.20(4-4)^2=10.70. \end{aligned} \]

    The best allowed point action is therefore \(1\), even though \(0\) is the most probable individual outcome. Repeating either point action does not reproduce the conditional distribution. Conversely, a calibrated conditional law can be paired with the wrong task loss or a poor search procedure. Held-out proper scores assess the probabilistic forecast, while task losses assess decisions; they answer different questions (Gneiting and Raftery, 2007). The full treatment of minimum Bayes risk, candidate sets, and search appears in Conditioning, Steering, Constraints, and Selection.

    Output dimension does not create this distinction. A scalar class label can have a non-degenerate conditional law, and an image or sequence system can return one deterministic decision. The formula \(p(y\mid x)\) alone therefore does not determine a family label. State whether the system defines only that conditional law, a joint law, a constructive output mechanism, or merely a point decision—not how many reference answers a benchmark happens to provide.

    Factorization and query retrieval practice¤

    Retrieval practice: chain rules, information, and queries
    1. Does the chain rule make every joint distribution efficiently autoregressive?
    2. What identity relates cross-entropy, entropy, and KL for discrete laws?
    3. Why can differential entropy change under a smooth reparameterization while KL remains invariant?
    4. Why is a conditional probability law not yet a decision rule?

    Solutions. (1) No; the identity exists for any ordering, but representing, evaluating, and sampling its conditionals can still be intractable. (2) \(H(p,q)=H(p)+D_{\mathrm{KL}}(p\|q)\). (3) Both densities acquire Jacobian terms that cancel in KL, while a single entropy retains the coordinate-volume change. (4) A decision rule also requires actions, utility/loss, constraints, and possibly uncertainty or abstention policy.

    Factorization and query synthesis¤

    Factorization is a probability identity whose computational value depends on the chosen ordering and conditional family. Information measures connect prediction and coding but retain support and representation assumptions. A generator can be queried for samples, densities, conditionals, representations, or decisions; success on one is not success on all.
