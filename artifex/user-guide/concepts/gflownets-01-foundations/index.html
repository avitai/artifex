    1. Constructive State Graphs and Reward-Proportional Sampling¤

    Chapter 1 prerequisites and learning objectives¤

    This chapter assumes conditional probability, finite directed graphs, and ancestral sampling. After completing it, you should be able to:

    1. distinguish semantic terminal objects from states, actions, and their possibly many construction trajectories;
    2. derive state and edge flow from trajectory flow and prove source-to-terminal conservation on a finite constructive DAG;
    3. derive forward and backward kernels and the reward-proportional terminal law, including the Markovian-flow assumption or Markovianization step that justifies a product policy;
    4. audit explicit-stop, dead-end, reachability, and duplicate-terminal conventions before treating the graph as a probabilistic model;
    5. predict how scaling, tempering, clipping, shifting, or noisily estimating reward changes the target or its estimator; and
    6. transfer the construction to a new combinatorial object while separating path multiplicity, semantic identity, reward validity, and optimization evidence.

    Let \(\mathcal G=(\mathcal S,\mathcal E)\) be a directed acyclic graph with initial state \(s_0\). A state is a partially constructed object; an edge \(s\to s'\) is a valid action; a terminal state \(x\in\mathcal X\) is a completed object. A trajectory is

    \[ \tau=(s_0\to s_1\to\cdots\to s_n=x). \]

    Different trajectories may end at the same \(x\). Adding atoms in different orders can yield the same molecule; adding edges in different orders can yield the same graph. A correct object probability must sum over all such trajectories.

    The graph is part of the probabilistic model. It specifies which partial objects exist, which actions are valid, when construction stops, and which histories count as alternative routes to the same terminal. Two implementations with the same reward function but different state graphs need not induce the same optimization problem or even the same terminal sample space.

    Probabilistic Grammars and Program Derivations applies this distinction to partial derivations, expansion-order multiplicity, executable programs, and the difference between terminal reward, grammar probability, and observation likelihood.

    This construction graph is not necessarily the relational graph being generated. When terminal objects are themselves graphs, Random Graphs and Relational Generators supplies the additional invariance, automorphism, sparsity, and graph-evaluation contracts; the GFlowNet supplies a reward-proportional constructive sampler.

    Stops, Terminals, and Complete Objects¤

    There are two clean conventions:

    1. a state \(x\) is terminal and has no outgoing action; or
    2. a completed-but-not-yet-terminal state takes an explicit STOP edge to a terminal sink representing the object.

    Either convention works, but mixing them can count one object twice or let probability leak past completion. If variable-length strings may stop at every prefix, STOP is a genuine action whose probability affects length. A dead end that is not a valid terminal is not an implicit stop: it is a graph-construction bug unless an explicit failure terminal receives the flow.

    Trajectory, Edge, and State Flow¤

    Assign nonnegative flow \(F(\tau)\) to complete trajectories. Derived quantities are

    \[ F(s\to s')=\sum_{\tau\ni(s\to s')}F(\tau), \]
    \[ F(s)=\sum_{\tau\ni s}F(\tau). \]

    Because the graph is acyclic, a trajectory visits a state or edge at most once. In cyclic constructions, the notation \(\tau\ni s\) must count visits or the state must be augmented with construction time; the simple equations below otherwise lose information.

    For a nonterminal state, conservation requires

    \[ \sum_{p\in\operatorname{Pa}(s)}F(p\to s) =F(s) =\sum_{c\in\operatorname{Ch}(s)}F(s\to c). \]

    At a terminal state, incoming flow is matched to reward:

    \[ F(x)=R(x). \]

    The source flow is the partition function:

    \[ F(s_0)=Z=\sum_xR(x). \]

    To see why, sum conservation over every internal state. Each internal edge appears once as outgoing and once as incoming and cancels. Only source outflow and terminal inflow remain. Anchoring terminal flow to \(R(x)\) therefore forces source flow to equal the total reward.

    Forward and Backward Policies¤

    Normalize outgoing and incoming edge flows:

    \[ P_F(s'\mid s)=\frac{F(s\to s')}{F(s)}, \qquad P_B(s\mid s')=\frac{F(s\to s')}{F(s')}. \]

    Sampling \(P_F\) from \(s_0\) to termination gives terminal probability \(R(x)/Z\) when the balance constraints hold. \(P_B\) is a training factorization over construction histories; it need not be the physical reverse of an environment.

    There is a precise factorization boundary here. An arbitrary trajectory flow can remember its earlier path, so its normalized trajectory measure need not equal a product of state-only transition kernels. A Markovian flow is one for which the next-action distribution depends only on the current state; then both the forward and backward products below reproduce the same trajectory law. If one starts only from compatible conserved edge flows on a DAG, normalizing them constructs a Markovian flow with those state and edge marginals. This is a Markovianization step, not a proof that every history-dependent trajectory flow was already Markovian.

    Formally, the probability of a complete trajectory is

    \[ P_F(\tau)=\prod_{t=0}^{n-1}P_F(s_{t+1}\mid s_t), \]

    whereas the probability of the terminal object is the many-to-one pushforward

    \[ P_F(X=x)=\sum_{\tau:s_n=x}P_F(\tau). \]

    Using \(P_F(s'\mid s)=F(s\to s')/F(s)\) and summing the induced trajectory flows gives

    \[ P_F(X=x)=\frac{F(x)}{F(s_0)}=\frac{R(x)}{Z}. \]

    This short derivation contains the central idea of the family: distribute a terminal mass across paths, conserve it through partial states, then normalize only once at the source.

    Worked Diamond Graph¤

    Suppose \(s_0\) branches to \(a\) and \(b\), both lead to terminal \(x\), and a separate branch leads to terminal \(y\). Let \(R(x)=3\), \(R(y)=1\). Total source flow is \(Z=4\), so terminal probabilities must be \(P(x)=3/4\), \(P(y)=1/4\).

    The flow of 3 into \(x\) may be split between paths through \(a\) and \(b\) in many ways. Object probability is fixed while trajectory probability is not. This degree of freedom lets a backward policy select how credit is allocated among construction orders.

    Make the numbers explicit. Give the two \(x\) paths flows 1 and 2 and the \(y\) path flow 1. Then

    \[ P_F(a\mid s_0)=\tfrac14, \quad P_F(b\mid s_0)=\tfrac24, \quad P_F(y\mid s_0)=\tfrac14, \]

    and both \(a\) and \(b\) deterministically finish at \(x\). Hence \(P_F(x)=1/4+2/4=3/4\). At \(x\), the backward policy is \(P_B(a\mid x)=1/3\) and \(P_B(b\mid x)=2/3\). Changing the split to \(3/2\) and \(3/2\) changes the trajectory law and backward policy but leaves the object law unchanged.

    Read the converging arrows as a sum, not as duplicate objects. The two routes carry different trajectory probabilities, but the terminal receives one total flow of 3. This is the visual invariant to retain when a graph, molecule, set, or program admits several construction orders.

    Counterexample: Path Multiplicity Is Not Reward¤

    Suppose objects \(x\) and \(y\) both have reward 1, but the implementation represents two orderings for \(x\) as two distinct terminals while \(y\) has one. If it assigns reward 1 to each encoding, the represented terminal space has total mass 3 and the semantic object probability becomes \(P(x)=2/3\), not the intended \(1/2\). The repair is to canonicalize terminal objects, aggregate all paths into one terminal, or divide/aggregate rewards according to a rigorously declared quotient representation. “The network will learn to ignore duplicates” is not a valid repair.

    Reward Is an Unnormalized Density¤

    If \(R(x)=e^{-E(x)/T}\), a GFlowNet amortizes sampling from an energy-based target. If \(R(x)\) is an acquisition function, it samples candidates in proportion to acquisition value. Raising reward to power \(\beta\) changes the target:

    \[ P_\beta(x)\propto R(x)^\beta. \]

    Reward clipping, adding \(\epsilon\), or rank transforms also change the target and must not be described as mere numerical stabilization.

    Rewards need not be normalized, differentiable, or available for partial states. They must, however, be evaluable on sampled terminals for the basic objectives. Negative utility cannot be used directly as flow: choose a nonnegative mapping and acknowledge the resulting distribution. If only an unbiased noisy reward estimate is available, putting that estimate inside a logarithm generally introduces bias; the noise model becomes part of the estimator design.

    What the Population Result Does—and Does Not—Say¤

    Established. On the specified graph, exact nonnegative conserved flow with terminal boundary \(F(x)=R(x)\) produces \(P_F(x)=R(x)/Z\).

    It does not say that stochastic gradient descent finds that flow, that a finite neural network represents it, that unvisited modes are corrected, that the reward is scientifically valid, or that a sampled object is novel. Those are approximation, exploration, and external-validity questions developed in Chapter 6.

    Chapter 1 retrieval Questions¤

    1. Why can two correct GFlowNets have different trajectory probabilities but the same object distribution?
    2. What changes if a reward is replaced by \(R^2\)?
    3. Why must a variable-length stop decision be represented explicitly?
    Complete solutions
    1. Conservation fixes total flow reaching each terminal, but that total may be split among multiple construction histories in many ways.
    2. The target becomes \(P(x)\propto R(x)^2\); it is more concentrated and is not a stabilization-equivalent target.
    3. Stop determines which prefix is a terminal and therefore contributes to both the trajectory probability and the length distribution.

    Chapter 1 Synthesis¤

    GFlowNets conserve flow through a constructive DAG and terminate with flow equal to reward. The induced object law is reward-proportional even when many paths reach one object. Chapter 2 derives local training constraints.

    Why not multiply the probabilities of one chosen construction path and call that the object probability?

    Because the object probability is the sum over every path that constructs it. Ignoring multiplicity biases objects with many or few construction orders.

    Independent practice and assessment map: constructive flow¤

    1. P1.1 — Predict before computing. In the diamond, keep \(R(x)=3\) and \(R(y)=1\) but move all of \(x\)'s flow to the path through \(a\). Predict which of the object law, trajectory law, forward policy, and backward policy change; then calculate them.
    2. P1.2 — Conservation derivation. A source sends flows 2 and 5 into two partial states that merge at terminal \(u\), and flow 3 directly to terminal \(v\). Enumerate state, edge, terminal, and source flows; normalize \(P_F\); and prove the two terminal probabilities both by path summation and by \(R/Z\).
    3. P1.3 — Markovianity audit. Two histories merge at state \(s\). A history-dependent controller chooses left with probability \(0.9\) after the first history and \(0.1\) after the second. Explain why one state-only \(P_F(\cdot\mid s)\) cannot reproduce both conditional trajectory laws, and give two valid repairs.
    4. P1.4 — Stop and quotient counterexample. Variable-length sets may stop after one or two insertions, and insertion order is retained in terminal IDs. Construct the smallest example in which one semantic set receives too much mass. Repair both the stop semantics and the semantic quotient.
    5. P1.5 — Reward intervention. For rewards \((1,4,9)\), compute the target before and after multiplying all rewards by 10, squaring them, and adding 1. State which interventions preserve terminal probabilities and how \(Z\) changes. Then explain why an unbiased random reward cannot generally be inserted unbiasedly into \(log R\).
    6. P1.6 — Transfer. Design a constructive graph for unlabeled three-node graphs. Specify state identity, valid action, stop rule, terminal canonicalization, nonnegative reward, and a toy exact-enumeration test that distinguishes path coverage from semantic-object coverage.

    Complete solutions to P1
    1. The object law remains \((P(x),P(y))=(3/4,1/4)\) because terminal flows remain \((3,1)\). The \(x\) path law changes from a positive split across \(a,b\) to probability one through \(a\) conditional on reaching \(x\). At the source, \(P_F(a\mid s_0)=3/4\), \(P_F(b\mid s_0)=0\), and \(P_F(y\mid s_0)=1/4\). At \(x\), \(P_B(a\mid x)=1\) and \(P_B(b\mid x)=0\). Thus object probability is invariant to a valid reallocation of terminal flow, but both path kernels change.
    2. The two incoming edge flows to \(u\) are 2 and 5, so \(F(u)=R(u)=7\); \(F(v)=R(v)=3\); and \(F(s_0)=Z=10\). Source action probabilities are \(2/10,5/10,3/10\). Summing the first two complete-path probabilities gives \(P_F(u)=7/10\), while the direct path gives \(P_F(v)=3/10\). The boundary calculation gives the same values: \(R(u)/Z=7/10\) and \(R(v)/Z=3/10\).
    3. Conditioning on the visible state \(s\) loses which incoming history occurred, yet the stipulated next-action laws differ. One state-only kernel would have to be both \((0.9,0.1)\) and \((0.1,0.9)\). Either augment the state with the relevant history so the two cases no longer merge, or replace the history-dependent trajectory flow by a declared Markovianized flow that preserves chosen state/edge marginals while accepting a different full trajectory law.
    4. Let elements be \(a,b\). If ordered terminals [a,b] and [b,a] each receive reward 1 while singleton [a] receives reward 1, semantic set \(\{a,b\}\) gets mass \(2/3\) rather than \(1/2\). Use one explicit STOP decision at each valid set size and map both insertion orders to the same canonical terminal \(\{a,b\}\), with total terminal reward assigned once. Alternatively, a formally declared quotient can divide and aggregate encoding rewards, but its induced object law must be proved.
    5. Initially \(Z=14\) and \(p=(1,4,9)/14\). Multiplication by 10 gives \(Z'=140\) and the same \(p\). Squaring gives \(Z'=98\) and \(p'=(1,16,81)/98\); adding 1 gives \(Z'=17\) and \(p'=(2,5,10)/17\). Only common positive scaling preserves normalized probabilities. For a positive unbiased estimate \(\widehat R\), Jensen's inequality generally gives \(\mathbb E[\log\widehat R]\ne\log R\); zeros can make the log undefined as well.
    6. Use isomorphism classes of partial simple graphs as semantic states, add one missing edge as an action, and allow one explicit stop after the declared minimum size. Canonical graph labeling supplies state and terminal IDs; invalid duplicate edges are masked. Assign reward once per terminal isomorphism class. For three vertices, enumerate every labeled action path, map paths to canonical graph IDs, and compare both path visitation and aggregated terminal frequencies with exact \(R/Z\). The first diagnoses exploration; the second diagnoses semantic proportionality.
