    2. Variable Elimination and the Cost of Intermediate Factors¤

    Chapter 2 prerequisites and learning objectives¤

    This chapter assumes the factor and query semantics from Chapter 1. By the end, you should be able to:

    • derive variable elimination from distributivity;
    • execute an elimination step and track its new factor scope;
    • construct the induced graph and identify fill-in edges;
    • distinguish order-specific induced width from graph treewidth; and
    • state time and memory cost using variable cardinalities rather than node count alone.

    Distributivity is the algorithm¤

    Suppose the query retains variables \(Q\) and eliminates \(R\). For one variable \(X_i\in R\), collect every current factor whose scope contains \(X_i\):

    \[ h_i(x_{S_i\setminus\{i\}}) =\sum_{x_i}\prod_{a:\,i\in a}f_a(x_a), \qquad S_i=\bigcup_{a:\,i\in a}a. \]

    Replace those factors by \(h_i\) and continue. Factors that do not contain \(X_i\) remain outside the sum. This is ordinary algebra, but the order of reassociation determines the size of every intermediate table.

    factors = initial factors with evidence incorporated
for variable in elimination_order:
    bucket = every factor whose scope contains variable
    product_factor = multiply(bucket)
    new_factor = sum_out(product_factor, variable)
    factors = (factors without bucket) plus new_factor
return multiply(factors)  # normalize if the query requires a probability

    The pseudocode is exact for finite tables. A robust implementation also tracks named axes, base measures, sparse or deterministic structure, log scaling, and the difference between summing and maximizing.

    Worked example: eliminate the leaves, retain \(B\)¤

    For the running chain, eliminate \(A\) first:

    \[ r_{A\to B}(b)=\sum_a u_A(a)f_{AB}(a,b). \]

    The two entries are

    \[ r_{A\to B}(0)=2(3)+1(1)=7, \qquad r_{A\to B}(1)=2(1)+1(3)=5. \]

    Eliminating \(C\) gives

    \[ r_{C\to B}(0)=2(1)+1(4)=6, \qquad r_{C\to B}(1)=1(1)+2(4)=9. \]

    The unnormalized marginal over \(B\) is their pointwise product,

    \[ \widetilde p(B)=(7,5)\odot(6,9)=(42,45). \]

    Its sum is \(87\), recovering the enumerated normalizer, and normalization gives \((14/29,15/29)\). Elimination did not approximate the table; it evaluated the same sum with reusable partial results.

    Scope growth, fill-in, and induced width¤

    When eliminating a variable, all its uneliminated neighbors become arguments of the new factor. Graphically, connect those neighbors into a clique before removing the variable. Newly required edges are fill-in edges.

    Consider a four-cycle \(A-B-C-D-A\). Eliminating \(A\) combines the factors on \((A,B)\) and \((D,A)\) and creates a new factor on \((B,D)\), adding the chord \(B-D\). The largest factor created by a complete order determines that order's induced width: largest induced clique size minus one.

    The treewidth of a graph is the minimum induced width over all elimination orders. It is a property of the graph after the relevant transformations, whereas induced width belongs to a particular order. A tree has treewidth one; a cycle with at least four vertices has treewidth two; a complete graph on \(n\) vertices has treewidth \(n-1\).

    For at most \(K\) states per variable and width \(w\), a common coarse bound is

    \[ \text{time}=O(nK^{w+1}), \qquad \text{peak table memory}=O(K^{w+1}). \]

    With heterogeneous cardinalities, the cost of a factor on scope \(S\) is proportional to \(\prod_{i\in S}|\mathcal X_i|\), not \(K^{|S|}\). Sparse support, determinism, context-specific independence, algebraic decision diagrams, low-rank structure, or Gaussian linear algebra may reduce actual cost. Treewidth is therefore a central structural control, not a complete description of every representation.

    Ordering is a combinatorial problem¤

    Finding an order of minimum width is NP-hard. Practical systems use heuristics such as:

    • min-degree: eliminate a variable with the fewest current neighbors;
    • min-fill: eliminate a variable that introduces the fewest fill-in edges;
    • weighted min-fill: weight a prospective clique by state cardinalities;
    • nested dissection: recursively separate sparse spatial graphs; or
    • domain-specific orders that preserve repeated or temporal structure.

    Report the chosen order or heuristic, induced width, maximum factor entries, and whether deterministic or sparse compression was used. Reporting only the number of variables hides the main cost driver.

    Exact does not mean cheap, and neural does not mean approximate¤

    Exactness concerns the returned query. Cost concerns the representation and order. A large low-treewidth tabular model can support exact inference; a small dense model can be expensive. A neural network may parameterize a local factor while exact elimination remains possible, or it may parameterize an amortized posterior that is approximate even when the original graph is a tree.

    For a Gaussian graphical model, elimination corresponds to Schur complements and sparse matrix factorization. The relevant width controls fill-in in the Cholesky factor. Exact Gaussian algebra avoids enumerating continuous states, but poor conditioning, floating-point error, and dense fill remain practical concerns.

    Counterexample: the parameter count is not the elimination cost

    One compact network may evaluate a high-order factor with few learned parameters. Eliminating a neighboring variable can still require representing that factor over exponentially many discrete configurations. Parameter sharing does not shrink an exact table unless the inference algorithm exploits its functional structure.

    Chapter 2 synthesis¤

    Variable elimination is distributivity organized as an algorithm. Every eliminated variable produces a factor over its remaining neighbors, so order determines induced width and memory. Chapter 3 interprets the same intermediate factors as directional messages and reuses them to obtain every marginal on a tree.

    Retrieval practice: elimination and width
    1. What arguments remain in the factor created when eliminating \(X_i\)?
    2. Why did eliminating \(A\) and \(C\) in the running chain cost less than enumeration?
    3. What is the difference between induced width and treewidth?
    4. Why can two variables with the same graph degree have different elimination costs?

    Solutions. (1) The union of scopes of factors containing \(X_i\), minus \(X_i\). (2) Each leaf sum was computed once and reused for both values of \(B\). (3) Induced width is for one order; treewidth is the minimum over orders. (4) Their state cardinalities and the cardinalities or structure of their neighbors may differ.

    Bridge. An elimination factor summarizes everything on one side of a graph boundary. On a tree, that summary can be cached as a message in both directions. Continue continuously or open Chapter 3.
