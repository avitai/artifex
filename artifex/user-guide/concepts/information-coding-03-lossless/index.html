    3. Lossless Coding, Likelihood, and Generative Prediction¤

    Chapter 3 prerequisites and learning objectives¤

    You should understand entropy and autoregressive factorization. By the end of this chapter, you should be able to:

    • distinguish a prefix code, a uniquely decodable code, and an ideal real-valued codelength;
    • apply the Kraft–McMillan condition and the source-coding bound;
    • trace how a normalized generative model drives Huffman, arithmetic, range, or ANS coding;
    • convert sequence likelihood into a correctly normalized coding metric; and
    • identify every source of overhead between model NLL and a stored file.

    A code is a protocol, not a metaphor¤

    A binary prefix code assigns each source symbol a finite bitstring such that no codeword begins another. Its lengths \(\ell(x)\) satisfy the Kraft inequality

    \[ \sum_x2^{-\ell(x)}\leq1. \]

    Conversely, integer lengths satisfying this inequality can be realized by a binary prefix code. The McMillan extension shows that every uniquely decodable code obeys the same inequality. Prefix codes are convenient because concatenated messages can be decoded immediately from left to right, but unique decodability is the more general property.

    For any source distribution \(p\) and uniquely decodable binary code,

    \[ \mathbb E_p[\ell(X)]\geq H_2(X). \]

    Choosing Shannon lengths \(\ell(x)=\lceil-\log_2p(x)\rceil\) gives an expected length below \(H_2(X)+1\). An optimal Huffman code is no worse for a finite alphabet. These are single-symbol integer-code bounds; block and streaming coders can amortize the sub-bit gap.

    Complete prefix-code example¤

    Return to \(p=(1/2,1/4,1/8,1/8)\). Assign

    Table INFO.3. Complete prefix-code example
    Symbol Probability Code Length
    \(A\) \(1/2\) 0 1
    \(B\) \(1/4\) 10 2
    \(C\) \(1/8\) 110 3
    \(D\) \(1/8\) 111 3

    The Kraft sum is \(1/2+1/4+1/8+1/8=1\), and the expected length is \(1.75\) bits—exactly the entropy because all probabilities are dyadic. The stream 010111 decodes uniquely as \(A,B,D\).

    If the true source is \(p\) but code lengths are induced by the mismatched \(q\) from Chapter 1, expected ideal length increases from \(1.75\) to \(2\) bits. The \(0.25\)-bit difference is \(D_{\mathrm{KL}}(p\|q)\).

    The source-coding theorem is asymptotic and source-specific¤

    For a finite-alphabet memoryless source, block codes can approach entropy rate with vanishing excess rate and vanishing or zero decoding error according to the coding formulation. More general stationary ergodic sources use an entropy rate

    \[ \bar H(X)=\lim_{n\to\infty}\frac1nH(X_{1:n}), \]

    when that limit exists. Nonstationarity, side information, streaming delay, bounded memory, finite block size, and nonzero error criteria define different operational problems. “[Entropy is] the number of bits” is therefore incomplete unless the source class and code regime are stated. The foundational noiseless and noisy coding results originate in Shannon's 1948 analysis.

    Sequential probability models become sequential coders¤

    For a discrete sequence,

    \[ q(x_{1:T})=\prod_{t=1}^{T}q(x_t\mid x_{<t}), \]

    so ideal length decomposes as

    \[ -\log_2q(x_{1:T}) =\sum_{t=1}^{T}-\log_2q(x_t\mid x_{<t}). \]

    An arithmetic or range coder refines an interval according to each conditional probability table. An asymmetric numeral system updates a discrete coder state using the same model probabilities, often with stack-like ordering. The encoder and decoder must produce identical quantized probability tables in identical order; a numerically excellent model is not a valid codec if the two sides disagree by one rounding decision.

    Huffman coding rounds each individual symbol length to a tree depth. Arithmetic, range, and ANS methods amortize fractional ideal lengths across a sequence. None makes the exact bitstream length equal to \(-\log_2q(x)\) for every file: termination, finite-state normalization, probability quantization, and container conventions remain.

    Metrics must declare their denominator¤

    For a test sequence with total negative log likelihood \(L=-\log q(x_{1:T})\) in nats:

    \[ \operatorname{PPL}=\exp(L/T) \]

    is token perplexity, whereas

    \[ \operatorname{bits/token}=\frac{L}{T\log2}. \]

    For \(D\) declared scalar image dimensions,

    \[ \operatorname{BPD}=\frac{L}{D\log2}. \]

    If \(L\) is already a mean rather than a sum, dividing by \(T\) or \(D\) again is an error. Perplexity changes with tokenizer because the unit called a token changes. Bits per byte or character can improve comparability only when corpus bytes, encoding, context treatment, and evaluation boundaries match. Image BPD additionally depends on bit depth, color transform, quantization, dequantization, and whether \(L\) is an exact likelihood or a variational bound.

    Actual size has more terms than model cross-entropy¤

    A reproducible compression report separates

    \[ L_{\mathrm{file}} =L_{\mathrm{payload}} +L_{\mathrm{side}} +L_{\mathrm{model}} +L_{\mathrm{container}} +L_{\mathrm{termination}}. \]
    • Payload is the entropy-coded data under synchronized probability tables.
    • Side information includes shapes, hyperlatents, masks, or indexes required by the decoder.
    • Model cost matters unless encoder and decoder are assumed to share the trained parameters.
    • Container and termination costs include headers, padding, checksums, final coder state, and block boundaries.

    A large pretrained model can compress a benchmark well when its weights are free shared context. Counting those weights answers a different universal or two-part coding question. Both conventions are valid when declared; silently switching between them is not.

    Generation and compression use the same conditionals in opposite roles¤

    An autoregressive generator draws a symbol from \(q_t\) and updates its prefix. A lossless decoder reads coded state using \(q_t\) and recovers the unique transmitted symbol. These operations share the normalized conditional model, but their randomness and guarantees differ. Generation samples a new sequence; decompression must reproduce a particular original sequence exactly.

    A low NLL model is not automatically a deployable compressor

    It may use nondeterministic kernels, floating-point probabilities that are not synchronized across hardware, unavailable future context, an intractable latent marginal, or enormous shared parameters. Validate a real round trip, bit count, random-access policy, latency, memory, and failure behavior in addition to test NLL.

    Chapter 3 synthesis and retrieval¤

    Entropy lower-bounds expected lossless description length only for a declared source and coding regime. A generative model supplies probabilities; an entropy coder turns discrete synchronized tables into bits. Model mismatch contributes forward-KL redundancy, while real systems add side information and finite implementation overhead. NLL-derived metrics remain comparable only under matched probability units and denominators.

    Retrieval questions
    1. Why can an ideal length \(-\log_2q(x)\) be noninteger while a prefix-code length cannot?
    2. What expected redundancy does a mismatched model add?
    3. Why can two models with the same sequence likelihood have different token perplexities?
    4. What must a codec test prove beyond low NLL?

    Solutions. (1) The ideal value is a real-valued logarithmic accounting quantity; a finite binary codeword has an integer number of bits. Sequence coders amortize fractional values. (2) \(D_{\mathrm{KL}}(p\|q)\) in the selected units. (3) Different tokenizers divide the same sequence log likelihood by different token counts. (4) Deterministic encoder–decoder synchronization, exact round trip, measured total size, side-information accounting, resource cost, and behavior on malformed or unsupported input.

    Bridge. Lossless coding preserves every source distinction. Chapter 4 permits reconstruction error and asks which distinctions are worth transmitting.
