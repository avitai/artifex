    Bayesian Flow Networks (BFNs) generate data by evolving the parameters of simple input distributions through Bayesian updates. A neural network reads those parameters and predicts an interdependent output distribution. The state is therefore a belief about data, not merely a corrupted sample.

    The original framework by Graves et al. derives discrete- and continuous-time losses for continuous, discretized, and discrete data. Later work establishes SDE correspondences with diffusion under specific parameterizations. This volume develops both views without declaring them identical at every algorithmic level.

    1. The BFN Generative Story¤

    Chapter 1 prerequisites and learning objectives¤

    This chapter assumes Bayes' rule, factorized distributions, KL divergence, and latent-variable ELBOs. After completing it, you should be able to:

    1. name the input, output, sender, receiver, update, and flow distributions without conflating them;
    2. derive the receiver as a sender mixture under the network's current output belief;
    3. explain additive accuracy as a semigroup property and derive direct time-\(t\) teacher-flow sampling;
    4. derive the discrete-time transmission loss as both a relative-codelength term and part of a VAE bound;
    5. distinguish exact within-family Bayes updates from the learned receiver and deployed generator; and
    6. transfer the BFN construction to a new conjugate belief family while identifying which properties require a new proof.

    The evidence-ledger intuition¤

    The Alice–Bob story is useful only when every role remains mathematical. Alice knows training datum \(x\) and sends a noisy message from \(p_S\). Bob does not know \(x\); his network defines \(p_O\), from which he predicts the message law \(p_R\). Bayes' rule turns the observed message into Bob's next input state.

    An intuitive way to remember the construction is as an evidence ledger. The input parameters record what evidence has accumulated; the neural network reads the whole ledger and predicts what clean datum could explain it; the receiver converts that prediction back into a message of the same type as the sender; and Bayes' rule posts the new message to the ledger. Training has an auditor who knows \(x\) and can generate the correct sender message. Generation does not, so it must use the learned receiver.

    BFNs separate five objects that short summaries often collapse:

    1. an input distribution \(p_I(x\mid\theta)\) with parameters \(\theta\);
    2. a neural network \(\Psi_\phi(\theta,t)\);
    3. an output distribution \(p_O(x\mid\theta,t)\) parameterized by the network;
    4. a sender distribution \(p_S(y\mid x;\alpha)\) that produces a noisy message about data \(x\) at accuracy \(\alpha\);
    5. a Bayesian update \(\theta'=h(\theta,y,\alpha)\).

    The receiver distribution predicts what message will arrive under the model's current output belief:

    \[ p_R(y\mid\theta,t,\alpha) =\int p_S(y\mid x;\alpha) p_O(x\mid\theta,t)\,dx. \]

    For discrete \(x\), the integral is a sum. Training presents messages generated from real data; generation samples messages from the receiver, updates the input parameters, and repeats.

    If both sender and output factorize across \(D\) coordinates, the receiver does too:

    \[ p_R(y\mid\theta,t,\alpha) =\prod_{d=1}^D \int p_S(y_d\mid x_d;\alpha) p_O(x_d\mid\theta,t)\,dx_d. \]

    The output factors are not context-free: every parameter emitted for coordinate \(d\) can depend on the entire \(\theta\) through the neural network. Dependencies therefore enter across network evaluations even though a receiver step commonly samples coordinate messages independently.

    The two message branches have the same message space and update rule but different sources. That shared interface makes the loss and sampler possible; it does not make a learned receiver message a true-data sender message.

    Why Two Distributions?¤

    The input distribution provides a simple factorized belief state suitable for exact Bayesian updating. The output distribution uses a neural network to introduce dependencies across coordinates. If the input distribution factorizes across pixels/tokens, the network can still predict an output distribution whose parameters depend on the entire current belief tensor.

    This resembles amortized inference but has a different role. The input parameters are the evolving generative state; the network is trained to predict data/message distributions along that flow.

    A factorized input state is a deliberately tractable memory, not a claim that the data coordinates are independent. For two perfectly correlated bits, the input may store two marginal probabilities while the network uses both marginals to predict compatible output factors. A coarse one-step receiver still cannot jointly draw the bits with their residual dependence; repeated states or a block output law must mediate it.

    Natural Parameters Accumulate Evidence¤

    Bayesian updates are simplest in natural parameters. If a prior and message likelihood are conjugate, posterior natural parameters equal prior natural parameters plus message sufficient statistics. An accuracy schedule controls how much information arrives over time.

    For intuition, suppose the input family is exponential:

    \[ p_I(x\mid\eta)=h(x)\exp\!\left\{ \eta^\top T(x)-A(\eta) \right\}, \]

    and the sender likelihood, viewed as a function of \(x\), contributes a compatible term \(s(y,\alpha)^\top T(x)\). Bayes' rule then updates the natural parameter by

    \[ \eta'=\eta+s(y,\alpha). \]

    This equation motivates Gaussian precision and categorical log-evidence updates, but it is not a universal BFN theorem. A candidate family must still provide a normalized sender, a tractable receiver mixture, a compositional update distribution, a loss, and endpoint semantics.

    For independent messages with accuracies \(\alpha_1,\ldots,\alpha_n\), the total accuracy often adds:

    \[ \beta_n=\sum_{i=1}^{n}\alpha_i. \]

    The continuous-time schedule \(\beta(t)\) describes cumulative information. Confusing per-step accuracy \(\alpha_i\) with cumulative accuracy \(\beta(t)\) produces incorrect samplers and loss weights.

    Additivity is a distributional property, not merely the scalar identity above. If \(p_U\) denotes the distribution over updated parameters after marginalizing a sender message, then the required semigroup relation is

    \[ p_U(\theta''\mid\theta,x;\alpha_a+\alpha_b) =\int p_U(\theta'\mid\theta,x;\alpha_a) p_U(\theta''\mid\theta',x;\alpha_b)\,d\theta'. \]

    It permits a training state at time \(t\) to be sampled directly as

    \[ p_F(\theta\mid x,t) =p_U(\theta\mid\theta_0,x;\beta(t)), \]

    without simulating all earlier messages. A proposed nonconjugate or geometric update that lacks this property may still define an iterative generator, but it does not inherit the canonical BFN flow or schedule algebra automatically.

    Training Flow and Generative Flow¤

    During training, a true datum \(x\) generates sender messages, inducing a distribution over input parameters \(p_F(\theta\mid x,t)\) called the Bayesian flow distribution. The network is trained on states drawn from this analytically known or simulable flow.

    During generation, \(x\) is unknown. The receiver samples messages using the network's output distribution. Exact agreement between training and generation flows requires the learned receiver to match the true sender-message mixture.

    For step times \(t_i=i/n\) and increments \(\alpha_i=\beta(t_i)-\beta(t_{i-1})\), the transmission loss is

    \[ L_n(x)= \mathbb E_{\theta_{0:n-1}\mid x} \sum_{i=1}^n D_{\mathrm{KL}}\!\left( p_S(\cdot\mid x;\alpha_i) \,\|\, p_R(\cdot\mid\theta_{i-1},t_{i-1},\alpha_i) \right). \]

    The KL direction is sender to receiver. In relative-entropy coding it is the expected excess nats for communicating a sender draw using a receiver belief. It also equals the latent-code KL term when messages \(y_{1:n}\) are treated as a VAE latent sequence. A reconstruction term

    \[ L_r(x)=-\mathbb E_{p_F(\theta\mid x,1)} \log p_O(x\mid\theta,1) \]

    completes the negative variational bound \(L_n+L_r\). The original work optimizes the transmission term and chooses a high terminal accuracy so reconstruction is small; therefore “directly optimizes compression” does not mean a finite implementation already reports realized coder bits.

    Complete worked example: finite-state evidence accumulation¤

    Suppose \(x\) is one of three categories and the input state is probabilities \(\theta=(1/3,1/3,1/3)\). A noisy message favors category 2. Bayes' rule increases \(\theta_2\) but need not collapse the state to a one-hot token. Repeated messages sharpen the belief. The neural network reads all positions' simplex-valued beliefs and predicts correlated category distributions.

    For a binary illustration, let the input prior be \((1/2,1/2)\) and suppose a message has likelihood ratio \(p_S(y\mid x=1)/p_S(y\mid x=0)=e^{1.2}\). Posterior log odds equal prior log odds plus 1.2, so

    \[ P(x=1\mid y)=\sigma(1.2)\approx0.769. \]

    A second independent message with log-likelihood ratio \(-0.4\) gives posterior \(\sigma(0.8)\approx0.690\), identical to one message whose log-likelihood ratio is \(0.8\). This is the finite log-odds form of accumulated evidence; Chapters 2 and 3 show when the random message distribution also has additive accuracy.

    Chapter 1 Synthesis¤

    A BFN evolves distribution parameters by Bayes' rule. Sender messages define training evidence; receiver messages define generation; the network couples otherwise simple beliefs. Chapter 2 makes the update exact for Gaussian continuous data.

    Independent practice and assessment map: the BFN generative story¤

    1. P1.1 — Predict before computing. A binary input belief starts at odds \(1:1\). Before calculating, predict whether two independent log-likelihood-ratio messages \(+1.4\) and \(-0.6\) leave class 1 above or below probability \(1/2\); then compute the posterior and explain why message order cannot matter.
    2. P1.2 — Receiver derivation. For a three-state output belief \(p_O=(0.2,0.5,0.3)\) and sender densities \(s_k(y)=p_S(y\mid x=k;\alpha)\), write the receiver density, prove that it normalizes, and identify the condition under which it assigns positive density wherever the true sender does.
    3. P1.3 — Additivity audit. Starting from the update-kernel composition integral, explain why the scalar equality \(\beta=\alpha_a+\alpha_b\) is insufficient. State the additional distributional identity needed for direct time-\(t\) training.
    4. P1.4 — Latent-message VAE. Write the finite message-sequence posterior \(q(y_{1:n}\mid x)\), autoregressive prior \(p_\phi(y_{1:n})\), and decoder. Show which two terms form the negative ELBO and identify which term the original continuous-time training objective does not directly optimize.
    5. P1.5 — Exactness boundary. Suppose every individual Bayes update is computed without numerical error, but \(p_O\) puts all mass on the wrong category. Decide which of the four exactness levels in the orientation table pass and which fail.
    6. P1.6 — Transfer. Propose a BFN-like construction for a Poisson-rate belief with Gamma conjugacy. Name the input parameter, message sufficient statistics, and update, then list the sender normalization, accuracy-semigroup, receiver, loss, and endpoint claims that must be derived before calling it a canonical BFN.

    Complete solutions to P1
    1. The total log odds are \(1.4-0.6=0.8>0\), so class 1 remains above \(1/2\). Its probability is \(\sigma(0.8)\approx0.690\). Independent evidence adds in log odds, so reversing the two messages leaves the sum and posterior unchanged.
    2. \(p_R(y)=0.2s_1(y)+0.5s_2(y)+0.3s_3(y)\). Its integral is the same convex combination of three unit integrals and equals one. Finite coding loss requires \(p_R(y)>0\) wherever the data-conditioned \(p_S(y\mid x)\) is positive; otherwise the sender-to-receiver KL is infinite.
    3. The scalar sum labels cumulative information but says nothing about the random updated state. One needs \(p_U(\theta''\mid\theta,x;\alpha_a+\alpha_b)=\int p_U(\theta'\mid\theta,x;\alpha_a)p_U(\theta''\mid\theta',x;\alpha_b)d\theta'\). Only then may all previous teacher messages be collapsed into one draw at \(\beta(t)\).
    4. \(q(y_{1:n}\mid x)=\prod_i p_S(y_i\mid x;\alpha_i)\), while \(p_\phi(y_{1:n})=\prod_i p_R(y_i\mid\theta_{i-1},t_{i-1},\alpha_i)\) and \(p_\phi(x\mid y_{1:n})=p_O(x\mid\theta_n,1)\). The negative ELBO is the message KL \(L_n\) plus reconstruction \(L_r\); the original work directly optimizes the transmission term or its continuous limit, not \(L_r\) itself.
    5. The arithmetic of the within-family update and the implementation of that update pass. The learned receiver fails to match the sender mixture under data, so teacher and generative flows differ; the repeated deployed generator is consequently wrong even if its declared kernel is sampled exactly.
    6. A Gamma input belief for rate \(x>0\) can update shape and rate using Poisson counts and exposure as additive sufficient statistics. That is only a candidate. One must derive a valid sender over messages, show that marginalized updates compose under an accuracy parameter, form and sample the receiver mixture, derive the finite/continuous loss, and specify a terminal observation law. Conjugacy alone proves none of those remaining claims.
    Retrieval questions
    1. Why can a factorized BFN output still use global context?
    2. What does the receiver integrate or sum over?
    3. What stronger property is required beyond writing \(\beta=\sum_i\alpha_i\)?
    4. In which direction does the transmission KL run?
    5. Is the canonical BFN state a sample \(x_t\)?
    Complete solutions
    1. Each output factor is parameterized by a network that reads all input parameters, although messages may be sampled independently conditional on that network state.
    2. It averages the sender distribution over every possible datum \(x'\) weighted by the output belief \(p_O(x'\mid\theta,t)\).
    3. The marginalized Bayesian-update kernels must compose as a semigroup: two independent accuracies must induce the same parameter law as their summed accuracy.
    4. \(D_{\mathrm{KL}}(p_S\|p_R)\), sender to receiver, because the sender produces the message and the receiver supplies its coding/predictive distribution.
    5. No. It is \(\theta_t\), the parameters of an input distribution. Noisy-sample or SDE coordinates are derived representations.

    2. Continuous Data: Gaussian Bayesian Flow¤

    Chapter 2 prerequisites and learning objectives¤

    This chapter assumes Gaussian conjugacy, natural parameters, and differential entropy. After completing it, you should be able to:

    1. derive the posterior update and the random update distribution after marginalizing a Gaussian sender message;
    2. derive the original entropy-linear accuracy schedule and closed-form flow marginal;
    3. reconstruct the original noise-prediction output, receiver distribution, and weighted continuous-time loss;
    4. distinguish continuous density, discretized bin mass, and a measurement-noise reconstruction convention;
    5. separate the posterior over \(x\), teacher-flow law over \(\mu_t\), receiver law over \(y\), and unconditional state marginal; and
    6. transfer the natural-parameter analysis to a richer Gaussian belief while identifying tractability and cost changes.

    Consider a scalar input distribution

    \[ p_I(x\mid\mu,\rho)=\mathcal N(x;\mu,\rho^{-1}), \]

    where \(\rho\) is precision. A sender observes true \(x\) through Gaussian noise:

    \[ y\mid x,\alpha\sim\mathcal N(x,\alpha^{-1}). \]

    Conjugate Bayesian updating gives

    \[ \rho'=\rho+\alpha, \qquad \mu'=\frac{\rho\mu+\alpha y}{\rho+\alpha}. \]

    Precision is additive. The mean moves toward the message in proportion to its precision.

    Marginalizing \(y\sim\mathcal N(x,\alpha^{-1})\) gives an update distribution for the new mean:

    \[ \mu'\mid\mu,\rho,x,\alpha \sim \mathcal N\!\left( \frac{\rho\mu+\alpha x}{\rho+\alpha}, \frac{\alpha}{(\rho+\alpha)^2} \right), \qquad \rho'=\rho+\alpha. \]

    The posterior \(p_I(x\mid\mu',\rho')\) and the random law of \(\mu'\) are different distributions. The former is Bob's belief over data after one realized message; the latter describes training states across possible sender noise.

    Derivation in Natural Parameters¤

    Ignoring constants independent of \(x\),

    \[ \log p_I(x\mid\mu,\rho) =-\frac12\rho x^2+\rho\mu x+\text{const}, \]
    \[ \log p_S(y\mid x;\alpha) =-\frac12\alpha x^2+\alpha yx+\text{const}. \]

    Adding log prior and log likelihood yields posterior natural parameters \((\rho+\alpha,\rho\mu+\alpha y)\), giving the update above.

    Worked Update¤

    Let the prior be \(\mathcal N(0,1)\), so \(\mu=0,\rho=1\). A message \(y=2\) with precision \(\alpha=3\) gives

    \[ \rho'=4, \qquad \mu'=\frac{0+3\cdot2}{4}=1.5. \]

    The posterior is \(\mathcal N(1.5,0.25)\). The mean does not jump fully to the noisy observation because prior and message uncertainty are both retained.

    After another message \(y_2\) with precision \(\alpha_2\), the precision becomes \(1+3+\alpha_2\). Sequential updating equals one batch update when messages are conditionally independent and the conjugate assumptions hold.

    For example, adding \(y_2=-1\) with precision 1 gives total precision 5 and mean \((1\cdot0+3\cdot2+1\cdot(-1))/5=1\). Combining the two messages first produces the sufficient statistics total precision 4 and weighted observation \(5\), followed by the unit prior, and yields the same posterior. Reversing message order also yields the same result.

    Accuracy schedule and exact flow marginal¤

    The original continuous BFN starts from \(\theta_0=(\mu_0=0,\rho_0=1)\) and chooses terminal input standard deviation \(\sigma_1\in(0,1)\). Requiring the Gaussian input entropy to decrease linearly in time gives

    \[ \beta(t)=\sigma_1^{-2t}-1, \qquad \rho(t)=1+\beta(t)=\sigma_1^{-2t}, \]
    \[ \alpha(t)=\beta'(t) =-2\log(\sigma_1)\,\sigma_1^{-2t}. \]

    Define

    \[ \gamma(t)=\frac{\beta(t)}{1+\beta(t)} =1-\sigma_1^{2t}. \]

    Using the marginalized update distribution once at cumulative accuracy \(\beta(t)\) gives the Bayesian flow

    \[ \mu_t\mid x \sim\mathcal N\!\left( \gamma(t)x, \gamma(t)(1-\gamma(t)) \right), \]

    with deterministic precision \(\rho(t)\). This equality is the direct-training shortcut: sample \(t\), draw one Gaussian mean at the correct marginal, and avoid unrolling sender messages.

    The teacher-flow Gaussian is available only because training knows \(x\). Generation cannot jump directly to the corresponding conditional draw; it must repeatedly obtain evidence from the learned receiver or integrate a derived generative SDE/ODE.

    Vector and Factorized Inputs¤

    For \(D\) coordinates with diagonal precision, updates apply elementwise. The factorized input distribution does not model cross-coordinate covariance. The network output introduces dependence by predicting each coordinate using the full parameter state. This is analogous to a mean-field state processed by a globally coupled denoiser.

    The flow marginal's covariance is diagonal conditional on \(x\). It does not assert the unconditional distribution of \(\mu_t\) is diagonal: mixing over correlated data produces cross-coordinate dependence. Confusing conditional sender noise with the data marginal leads to incorrect independence claims.

    Output and Receiver Distributions¤

    In the original continuous construction, write

    \[ \mu_t=\gamma(t)x+ \sqrt{\gamma(t)(1-\gamma(t))}\,\epsilon, \qquad \epsilon\sim\mathcal N(0,I). \]

    The network predicts \(\hat\epsilon(\mu_t,t)\) and reconstructs

    \[ \hat x(\mu_t,t) =\frac{\mu_t}{\gamma(t)} -\sqrt{\frac{1-\gamma(t)}{\gamma(t)}} \hat\epsilon(\mu_t,t), \]

    for \(t>0\). The canonical output is the Dirac law \(p_O(x\mid\theta,t)=\delta(x-\hat x)\), so convolving with the sender produces the exact Gaussian receiver

    \[ p_R(y\mid\theta,t,\alpha) =\mathcal N(y;\hat x(\theta,t),\alpha^{-1}I). \]

    Other output families are possible generalizations, but they change the receiver integral and loss. They should not be retroactively attributed to the original model.

    Continuous BFN losses can often be expressed as weighted regression after simplifying Gaussian KL terms. The regression target, weight, and output parameterization depend on the schedule. Calling the loss “MSE” without the probabilistic derivation loses the compression and receiver semantics.

    Because sender and receiver have equal covariance,

    \[ D_{\mathrm{KL}}\!\left[ \mathcal N(x,\alpha^{-1}I) \,\|\, \mathcal N(\hat x,\alpha^{-1}I) \right] =\frac{\alpha}{2}\|x-\hat x\|^2. \]

    Taking the infinitesimal limit under the entropy-linear schedule gives the original continuous-time transmission loss

    \[ L_\infty(x) =-\log\sigma_1\, \mathbb E_{t,\mu_t\mid x} \left[ \frac{\|x-\hat x(\mu_t,t)\|^2}{\sigma_1^{2t}} \right]. \]

    Substituting the noise parameterization produces a time-weighted noise-prediction loss; it is not an unweighted MSE. Near \(t=0\), \(\gamma(t)\to0\) and the reconstruction formula is singular, so implementations need a declared threshold or endpoint rule.

    More explicitly, because

    \[ x-\hat x =-\sqrt{\frac{1-\gamma(t)}{\gamma(t)}} \bigl(\epsilon-\hat\epsilon\bigr), \]

    the data-prediction integrand can be converted to a noise-prediction integrand only with the schedule-dependent factor implied by \(\gamma(t)\). Dropping it changes which times dominate optimization. A source may write the weight differently after reversing time or rescaling the network output; those are coordinate changes to reconcile, not constants to copy by name.

    Discretized Continuous Data¤

    Integer pixels can be modeled by integrating a continuous output density over bins or predicting probabilities for discretization intervals. Rounding a Gaussian sample is not the same likelihood model unless bin masses are used in training/evaluation. Edge bins and bounded support require explicit treatment.

    For bin \(k\) with boundaries \((l_k,r_k]\) and predicted Gaussian location/scale \((\mu_x,\sigma_x)\), an interior probability is

    \[ p_O(k\mid\theta,t) =\Phi\!\left(\frac{r_k-\mu_x}{\sigma_x}\right) -\Phi\!\left(\frac{l_k-\mu_x}{\sigma_x}\right). \]

    For bounded data, tail mass below the first edge and above the last edge must be assigned explicitly, commonly to the edge bins. This output is categorical over bins even though it is parameterized through a continuous density. The receiver becomes a Gaussian mixture over bin representatives, so its finite-step KL generally needs Monte Carlo or another estimator.

    Truly continuous data cannot be reconstructed to infinite precision with finite nats. One must declare a measurement-noise model, finite discretization, or another observation density. The original continuous discussion uses fixed measurement variance to define a finite reconstruction KL; that convention is not a universal continuous likelihood.

    3. Discrete and Discretized Data¤

    Chapter 3 prerequisites and learning objectives¤

    This chapter assumes categorical distributions, logits/softmax, Gaussian sums, and the sender–receiver loss. After completing it, you should be able to:

    1. derive the canonical discrete sender in redundant logit space and its Bayesian update;
    2. prove accuracy additivity by adding Gaussian messages and composing softmax updates;
    3. sample the categorical Bayesian flow directly at arbitrary teacher-flow time;
    4. derive the receiver mixture and continuous-time squared-probability objective with its schedule weight;
    5. distinguish latent logits, simplex beliefs, output categories, discretized-continuous outputs, and Gumbel/embedding relaxations; and
    6. transfer the construction to a structured discrete representation while preserving support, symmetry, and legal-transition semantics.

    For a categorical variable with \(K\) outcomes, the input parameter lies on the probability simplex:

    \[ \theta_k\ge0, \qquad \sum_{k=1}^{K}\theta_k=1, \qquad p_I(x=k\mid\theta)=\theta_k. \]

    A generic Bayesian message update is

    \[ \theta_k' =\frac{\theta_k p_S(y\mid x=k;\alpha)} {\sum_j\theta_j p_S(y\mid x=j;\alpha)}. \]

    Equivalently, log odds accumulate log-likelihood-ratio evidence. The original BFN construction chooses sender distributions that produce smooth differentiable simplex states even though final data are discrete.

    Canonical Gaussian logit sender¤

    Let \(e_x\in\mathbb R^K\) be the one-hot vector for class \(x\) and \(\mathbf1\) the all-ones vector. The original discrete BFN sends a continuous vector

    \[ p_S(y\mid x;\alpha) =\mathcal N\!\left( y;\alpha(Ke_x-\mathbf1),\alpha K I \right). \]

    Its Bayesian update has the multiplicative form

    \[ \theta_k' =\frac{e^{y_k}\theta_k} {\sum_{j=1}^K e^{y_j}\theta_j} =\operatorname{softmax}(\log\theta+y)_k. \]

    The accuracy argument is redundant in \(h(\theta,y,\alpha)\) once \(y\) has been drawn; accuracy determines the message distribution. The all-ones logit direction is also redundant because adding a constant to every \(y_k\) leaves the softmax unchanged. This redundancy is useful for symmetric Gaussian algebra but must be handled in numerical and SDE interpretations.

    Two independent messages add:

    \[ y_a+y_b \sim\mathcal N\!\left( (\alpha_a+\alpha_b)(Ke_x-\mathbf1), (\alpha_a+\alpha_b)KI \right), \]

    and applying two updates gives \(\operatorname{softmax}(\log\theta+y_a+y_b)\). Therefore the full update distribution has additive accuracy, not only its mean.

    Why the Simplex Matters¤

    Diffusing one-hot vectors in Euclidean space produces continuous states that require a categorical interpretation. BFNs instead represent uncertainty explicitly as normalized probabilities. Gradients with respect to simplex-valued beliefs support conditioning and guidance, but differentiability alone does not guarantee calibrated discrete probabilities.

    Starting from the uniform prior \(\theta_0=\mathbf1/K\), direct sampling at cumulative accuracy \(\beta(t)\) becomes

    \[ z_t\mid x\sim \mathcal N\!\left( \beta(t)(Ke_x-\mathbf1),\beta(t)KI \right), \qquad \theta_t=\operatorname{softmax}(z_t). \]

    The original heuristic schedule is \(\beta(t)=\beta(1)t^2\). Unlike the continuous Gaussian case, it was not obtained from a closed-form entropy calculation; the original paper chose it as an empirical approximation to decreasing expected categorical entropy. Later periodic and geometric BFNs revisit precisely this schedule/additivity issue.

    The latent logits \(z_t\) and simplex state \(\theta_t\) are not interchangeable coordinates globally: softmax is many-to-one because \(z\) and \(z+c\mathbf1\) represent the same probabilities. A theorem stated for a linear SDE in \(z\) must not be restated as a linear SDE in \(\theta\).

    The upper route is the direct teacher-flow construction because \(x\) is known. The lower loop is generation because a hypothetical category must come from the network. The gauge family maps to one simplex point; retaining or removing that redundant direction changes latent-SDE bookkeeping but not categorical probabilities.

    Discrete Output Distribution¤

    The neural network maps the collection of simplex parameters and time to categorical output probabilities:

    \[ p_O(x\mid\theta,t)=\prod_d \operatorname{Cat}(x_d;\pi_{\phi,d}(\theta,t)), \]

    in a common factorized-output formulation. Each \(\pi_d\) depends on the whole state, so positions are conditionally coupled through the network across steps, even if sampled independently within a step.

    For coordinate \(d\), the receiver message is a \(K\)-component Gaussian mixture:

    \[ p_R(y_d\mid\theta,t,\alpha) =\sum_{k=1}^K \pi_{\phi,d}(k\mid\theta,t) \mathcal N\!\left( y_d;\alpha(Ke_k-\mathbf1),\alpha KI \right). \]

    The finite-step transmission KL from the single sender Gaussian to this mixture has no general closed form. In the infinitesimal limit, the canonical discrete loss simplifies to

    \[ L_\infty(x) =K\beta(1) \mathbb E_{t,\theta_t\mid x} \left[t\,\|e_x-\hat e_\phi(\theta_t,t)\|^2\right], \]

    where \(\hat e_\phi\) concatenates the network's expected one-hot outputs. The \(t\) weight and factor \(K\beta(1)\) belong to the derivation. Replacing the squared probability error by cross-entropy defines a different objective unless another equivalence is proved.

    Discrete, Discretized, and Continuous Are Different Cases¤

    • Discrete: categories are intrinsic, such as characters or codebook IDs.
    • Discretized: an underlying continuous quantity is observed in bins, such as 8-bit intensities.
    • Continuous: the observation model has a density relative to a continuous reference measure.

    Using a categorical model for pixel values defines a different likelihood from integrating a continuous density over bins. Neither is universally correct; preprocessing and intended queries decide.

    The three cases also use different receiver families. Continuous canonical output gives one Gaussian receiver; discretized-continuous output gives a Gaussian mixture over bin centers; intrinsic categorical output gives a Gaussian mixture over one-hot logit means. Similar-looking neural tensors do not make their KLs or endpoint likelihoods interchangeable.

    Message Loss and Codelength¤

    At each step, the sender knows the real datum while the receiver predicts the message from the model output. A natural loss is

    \[ \mathbb E_{\theta\sim p_F(\theta\mid x,t)} D_{\mathrm{KL}} \bigl(p_S(y\mid x;\alpha) \|p_R(y\mid\theta,t,\alpha)\bigr). \]

    Summed over messages, this measures extra coding cost when using the receiver rather than the data-informed sender. Exact constants and continuous-time limits depend on data type and schedule. “Directly optimizes compression” should be understood through this sender–receiver codelength derivation, not as a claim that every implementation already includes an arithmetic coder.

    The continuous-time squared loss is convenient for training, while an \(n\)-step compressor or likelihood bound depends on the finite-step mixture KLs and reconstruction term. Training with \(L_\infty\) and sampling with 10 steps does not mean the 10-step discrete-time bound was directly optimized.

    Worked Categorical Update¤

    Let \(\theta=(0.5,0.3,0.2)\) and a message have likelihoods \((0.2,0.6,0.1)\) under the three categories. Unnormalized posterior weights are

    \[ (0.10,0.18,0.02), \]

    which normalize to

    \[ \theta'=(1/3,0.6,1/15). \]

    The message favors category 2, but the prior probability still influences the posterior. A zero prior entry remains zero under ordinary Bayes updating, so initialization/support design matters.

    Under the canonical update, choose \(\theta=(0.5,0.3,0.2)\) and logit message \(y=(0,\log3,-\log2)\). Then

    \[ e^y\odot\theta=(0.5,0.9,0.1), \]

    which normalizes to \((1/3,0.6,1/15)\)—the same posterior as the likelihood-vector example. Implement this as \(\operatorname{softmax}(\log\theta+y)\) using log-sum-exp, rather than exponentiating extreme messages first. The uniform canonical initialization has full support; exact zeros usually arise from a modified prior, masking rule, or finite-precision underflow.

    Within-Step Independence¤

    If the receiver samples one message or category independently per coordinate, correlations must emerge through repeated network evaluations. Fewer steps expose the factorized output approximation. Joint/block output distributions increase expressivity at additional normalization and sampling cost.

    As a counterexample, let two bits satisfy \(P(00)=P(11)=1/2\). At an uninformative state, exact output marginals are both Bernoulli\((1/2)\). One factorized receiver step can generate evidence toward 01 or 10 even though those data states have zero mass. More, smaller updates let the network react to the evolving joint belief, but finite iterations do not automatically eliminate the mismatch.

    4. Training, Generation, Compression, and Guidance¤

    Chapter 4 prerequisites and learning objectives¤

    This chapter assumes the closed-form Gaussian and categorical flows. After completing it, you should be able to:

    1. implement direct time-\(t\) training with the correct time-sampling estimator and without unrolling updates;
    2. sample the exact receiver mixture for a chosen finite accuracy increment and distinguish its two random sources;
    3. audit schedule, endpoint, numerical solver, randomness, and final-decoding conventions;
    4. distinguish theoretical transmission codelength, a finite-step likelihood bound, a continuous-time objective, and realized compression;
    5. apply normalized categorical guidance while stating the changed receiver and target; and
    6. design oracle-replacement tests that isolate model, schedule, solver, guidance, and decoder error.

    Training Procedure¤

    A continuous-time training step commonly follows:

    1. sample data \(x\) and time \(t\);
    2. sample input parameters \(\theta\sim p_F(\theta\mid x,t)\) using the analytically derived Bayesian flow;
    3. compute output parameters with \(\Psi_\phi(\theta,t)\);
    4. evaluate the data-type-specific continuous-time loss.

    Sampling \(\theta\) directly at time \(t\) avoids unrolling every Bayesian update during training when the flow distribution is known. Generation still discretizes the information path.

    repeat
    draw datum x and time t
    compute cumulative accuracy beta(t)
    draw theta directly from p_F(theta | x,t)
    predict output distribution p_O(. | theta,t)
    evaluate the data-type-specific continuous-time integrand
    update network parameters and log metrics by t and modality
until the training budget is exhausted

    The time-sampling distribution is part of the Monte Carlo estimator. If \(t\sim r(t)\) rather than uniformly, the loss integrand requires the corresponding importance weight. Clipping that weight or excluding an endpoint may stabilize training but changes the estimator and must be recorded.

    The gradient-estimator taxonomy provides the general audit: distinguish the exact continuous-time expectation, the sampled time law, its importance weight, and any clipped or relaxed finite surrogate.

    Generative Procedure¤

    Initialize \(\theta_0\) from an uninformative/simple prior. For steps \(i=1,\ldots,n\):

    1. compute output distribution \(p_O(x\mid\theta_{i-1},t_{i-1})\);
    2. form/sample receiver message \(y_i\sim p_R(y\mid\theta_{i-1},t_{i-1},\alpha_i)\);
    3. update \(\theta_i=h(\theta_{i-1},y_i,\alpha_i)\).

    Finally sample or decode \(x\) from the output/input distribution at high accuracy. Step count, accuracy increments, stochastic message sampling, and final decoding define the sampler.

    The receiver mixture can be sampled without enumerating or integrating it:

    1. draw a hypothetical datum \(x_i'\sim p_O(\cdot\mid\theta_{i-1},t_{i-1})\);
    2. draw \(y_i\sim p_S(\cdot\mid x_i';\alpha_i)\);
    3. set \(\theta_i=h(\theta_{i-1},y_i,\alpha_i)\).

    Marginally, \(y_i\) has exactly the receiver law. This ancestral mixture trick is distinct from drawing the mode or mean of \(p_O\). It also exposes two random sources—output sampling and sender noise—that require explicit, non-reused RNG streams.

    theta = prior_parameters
for i in 1..n
    t_prev, t_next = schedule[i-1], schedule[i]
    alpha = beta(t_next) - beta(t_prev)
    output = network_output(theta, t_prev, condition)
    proposed_x = sample(output, rng_output[i])
    message = sample_sender(proposed_x, alpha, rng_message[i])
    theta = bayes_update(theta, message, alpha)
return sample_or_decode(network_output(theta, 1, condition))

    This is a finite-step sampler for the learned receiver system. It is not an exact draw from the unknown data distribution merely because each Bayes update is exact.

    The training shortcut removes an unrolled teacher trajectory, not the generative loop. The two branches meet at the learned output but have different conditioning information and different Monte Carlo estimators.

    Accuracy Scheduling¤

    Uniform steps in time need not be uniform in information. A schedule should state cumulative accuracy \(\beta(t)\) and increments

    \[ \alpha_i=\beta(t_i)-\beta(t_{i-1}). \]

    Negative increments are invalid for ordinary evidence accumulation. Very large final accuracy can create numerical saturation. Few-step generation requires a solver/schedule trained or derived for large increments; merely skipping steps changes the receiver update approximation.

    Worked four-step schedule¤

    For categorical \(\beta(t)=4t^2\) and four uniform time intervals, the accuracy increments are

    \[ (\alpha_1,\alpha_2,\alpha_3,\alpha_4) =(0.25,0.75,1.25,1.75). \]

    They sum to 4 but are far from uniform. Replacing them by four increments of 1 preserves terminal cumulative accuracy, not the intermediate flow or the states on which the network was trained. Conversely, a nonuniform time grid can be chosen to make information increments closer to uniform.

    Compression Interpretation¤

    If sender and receiver distributions are shared between encoder and decoder, bits-back or relative-entropy coding ideas can translate KL mismatch into expected codelength. A deployable compressor additionally needs quantized probabilities, synchronized randomness/messages, an entropy coder, headers, and finite-precision accounting.

    Report theoretical NLL/codelength separately from realized bits per symbol. The latter includes coder overhead and numerical constraints.

    The coding identity also has a support condition: the receiver must assign positive density wherever the sender can emit a message. A numerically zero mixture weight can create an infinite ideal code length even when a clipped training loss looks finite. Realized coding additionally needs a discretized message alphabet or a continuous-coding construction; transmitting floating-point vectors verbatim is not the claimed KL cost.

    Guidance¤

    For conditional generation, the network predicts output distributions with and without context. Guidance may combine logits, natural parameters, or scores derived under an SDE correspondence. The standard linear CFG formula is not automatically probability-preserving on a simplex; logits must be renormalized.

    For categorical output probabilities, one explicit log-space convention is

    \[ \pi_w(k\mid\theta,c) \propto \pi_u(k\mid\theta)^{1-w} \pi_c(k\mid\theta,c)^w, \]

    implemented by softmaxing \(\log\pi_u+w(\log\pi_c-\log\pi_u)\). Here \(w=0\) is unconditional and \(w=1\) is the ordinary conditional output. The guided output induces a new receiver mixture and hence a new generative process. Zero probabilities require support handling before taking logarithms.

    Gradient-based guidance can differentiate a property through \(\theta\) or output parameters. This modifies the receiver/generative dynamics. A guided sample is generally not from the original unconditional model and may not retain its compression interpretation.

    Guiding \(p_O\), guiding the sampled message, and perturbing \(\theta\) after Bayes updating are three different kernels. Only the first keeps the same receiver construction with a replaced output belief. The others need a new transition derivation and can violate additivity or Bayesian semantics.

    Pseudocode-Level Invariants¤

    • all categorical parameters remain finite, nonnegative after softmax, and sum to one;
    • Gaussian precision stays positive and increases with positive accuracy;
    • time and cumulative accuracy are monotone;
    • sender and receiver use identical message parameterization;
    • training loss and generation schedule agree on endpoint conventions;
    • final discretization matches the evaluation likelihood;
    • categorical updates use log-sum-exp and never materialize unstable \(e^y\) directly;
    • Gaussian precisions, variances, and schedule endpoints use one declared dtype and clipping policy;
    • conditional and unconditional branches use the documented guidance convention and NFE count;
    • output-sampling and sender-noise RNG streams are split rather than reused.

    Chapter 5 prerequisites and learning objectives¤

    BFN and Diffusion¤

    Both create a time-indexed family from low to high information and train a network on intermediate states. The most useful distinction is state choice:

    • diffusion commonly evolves a noisy sample \(x_t\);
    • BFN canonically evolves parameters \(\theta_t\) of an input distribution.

    Xue et al. identify linear SDEs corresponding to BFN noise-addition processes, align BFN regression losses with denoising score matching, and interpret the original sampler as a first-order solver for the corresponding reverse SDE. This is a distributional/dynamical equivalence under a derived coordinate system and assumptions, not proof that every discrete BFN, guidance rule, or finite-step implementation is identical to a diffusion model.

    Time runs in opposite directions across the two primary sources¤

    The original BFN paper uses an information-addition time \(s\): \(s=0\) is the uninformative prior state and \(s=1\) is the high-accuracy data side. The ICML SDE analysis explicitly adopts reverse time for alignment with diffusion, using \(t=1-s\) and a schedule equivalent to \(\gamma_{\mathrm{SDE}}(t)=\gamma_{\mathrm{BFN}}(1-t)\). Its forward linear SDE therefore runs from the data side toward the prior side, and generation solves the associated reverse dynamics from \(t=1-\eta\) toward \(t=0\).

    Table BFN.13. Time runs in opposite directions across the two primary sources

    An equation copied from one convention without the substitution \(t=1-s\) can reverse a schedule increment, endpoint, or score sign while retaining familiar symbols. Every solver comparison should therefore begin with its time map.

    For continuous data, the coordinate is the BFN mean after reindexing it as \(\widetilde\mu_t=\mu_{s=1-t}\). Substituting \(s=1-t\) into the analytic marginal from Chapter 2 determines a linear forward SDE on the truncated interval \([0,1-\eta]\). The associated reverse-time SDE has the usual form

    \[ d\mu_t= \left[F(t)\mu_t-G(t)^2\nabla_\mu\log p_t(\mu_t)\right]dt +G(t)d\bar W_t, \]

    with \(F,G\) fixed by the BFN schedule. The BFN noise-regression predictor can be algebraically converted to the marginal score under this parameterization. The original finite update is a first-order discretization of an equivalent parameterized reverse SDE; specialized ODE/SDE solvers integrate the linear part more accurately.

    The endpoint matters. In this reversed convention, \(t\to1\) is the uninformative side: the conditional mean law collapses toward the deterministic zero state and its density score becomes singular. The SDE derivation therefore truncates at \(1-\eta\); because the exact unconditional marginal there is not a convenient known reference density, the implementation approximates it by a zero-mean isotropic Gaussian with small variance. An “exact equivalence” claim that silently sets \(\eta=0\) omits both the singular boundary and this terminal initialization approximation.

    For categorical data, the linear SDE lives in redundant latent logits \(z_t\), not in simplex probabilities \(\theta_t=\operatorname{softmax}(z_t)\). The BFN loss corresponds to denoising score matching in \(z\) after the paper's reparameterization. Because softmax is nonlinear and many-to-one, the original parameter-space sampling rule does not directly become the same discretization of the latent linear SDE. The specialized discrete BFN solver is derived in the latent coordinate.

    There is a further categorical qualification. The original sampler first draws a category from the network output and then draws its Gaussian message. In Proposition 5.3 of the SDE analysis, the stated first-order reverse-SDE discretization is the categorical BFN update with that categorical sampling step omitted. The paper reports an empirical ablation in which removal helps, but neither the proposition nor that experiment proves that every original finite categorical sampler is exactly Euler--Maruyama for the latent SDE.

    This map records the proof obligations. A shared marginal SDE is upstream of a specific discretization; it does not erase native-state semantics, endpoints, or the extra random operations of an implementation.

    BFN and Variational Inference¤

    Bayesian updates are exact within the chosen input family and sender likelihood. The neural output distribution is learned; it is not necessarily the exact posterior of a global latent-variable model. The term “Bayesian” names the state update, not a posterior over network weights.

    The message sequence \(y_{1:n}\) does define a legitimate auxiliary-variable model for the finite-step VAE derivation: sender messages form the variational posterior, receiver messages form an autoregressive prior, and \(p_O(x\mid\theta_n,1)\) is the decoder. This algebraic identity does not turn \(\theta_t\) into a variational posterior over network weights or make every BFN extension an amortized-inference method.

    Bayesian sample inference starts from a broader modeling idea: treat the generated datum as an unknown and repeatedly update a belief using learned pseudo-observations. BFNs are one carefully chosen sender/update/output construction within that space. A paper that contains BFNs as a special case is a formal generalization, not evidence that its benchmark results transfer to the canonical BFN.

    BFN and Discrete Iterative Generation¤

    Masked diffusion evolves a corrupted token state; categorical BFNs evolve token probability vectors. Both can update positions in parallel and mediate dependencies through repeated networks. BFN simplex states make uncertainty explicit, while absorbing diffusion has an explicit mask-state Markov chain. Their objectives and transition laws differ.

    The latent-logit SDE correspondence adds another relationship: a categorical BFN can be analyzed with continuous diffusion mathematics in \(z\) without becoming a token-space CTMC. Conversely, a masked CTMC has exact categorical jump semantics without maintaining a full simplex belief at every position. These are alternative state augmentations, not algebraically identical implementations.

    BFN and Flow Matching¤

    Cumulative accuracy traces a path in distribution-parameter space. One may derive deterministic or stochastic dynamics for transformed parameters, but ordinary flow matching learns a vector field satisfying a continuity equation in sample state space. The shared “flow” word is an implementation analogy until a formal map establishes equivalent marginals/dynamics.

    At a more abstract level, a Bayesian update can be written as a KL-proximal operation: posterior density minimizes expected negative log likelihood plus KL to the prior. Replacing KL by another divergence yields a proposed proximal-flow generalization, but it changes the update semantics and may lose conjugacy, message coding, or accuracy additivity. “Proximal flow” is therefore not a synonym for BFN.

    What the SDE Correspondence Provides¤

    The SDE view imports numerical solvers and error analysis. The Bayesian view clarifies data-type-specific belief parameters, message coding, and simplex guidance. Retaining both views is more useful than renaming one family as the other.

    Worked coordinate and solver boundary¤

    Let \(z=(1,0,-1)\) and \(z'=z+7\mathbf1=(8,7,6)\). Both give the same simplex state because

    \[ \operatorname{softmax}(z')=\operatorname{softmax}(z). \]

    A Gaussian density or Euclidean score on unrestricted \(z\) nevertheless distinguishes their positions unless the redundant direction is handled consistently. Thus a latent-logit SDE contains a gauge direction absent from the simplex. One cannot recover its score uniquely by differentiating with respect to \(\theta\).

    Now consider two solvers using the same perfect score: one Euler step and two half steps. They target the same reverse SDE in the infinitesimal limit, but their finite transition laws and errors differ. Mapping the BFN predictor to a diffusion score therefore identifies the continuous-time object; it does not make NFE, stability, or finite-step quality identical.

    Failure Modes¤

    • Schedule mismatch: training samples cumulative accuracies not represented by the deployed step sequence.
    • Belief saturation: categorical logits or Gaussian precisions become numerically extreme.
    • Support lockout: zero categorical probabilities cannot recover under Bayes multiplication.
    • Factorized output error: too few iterations leave mutually dependent coordinates inconsistent.
    • Sender–receiver mismatch: implementation parameterizations differ, invalidating the loss/coding interpretation.
    • Discretization mismatch: continuous output is rounded differently from the evaluated bin likelihood.
    • Guidance distortion: property gradients or CFG overwhelm calibrated receiver dynamics.
    • Objective substitution: an implementation replaces the derived weighted MSE with convenient CE/MSE and retains the original likelihood claim.
    • Gauge mishandling: categorical latent logits drift in the all-ones direction or a latent-score theorem is applied directly to simplex probabilities.
    • Endpoint singularity: a solver evaluates \(t=0\) or \(t=1\) where its parameter conversion or reverse score is undefined.
    • Symmetry mismatch: periodic, rotational, permutation, or space-group equivalence is not preserved by the input family and update.
    • Evaluator shortcut: a protein/crystal property predictor rewards artifacts or training-set proximity rather than physical validity.

    Debugging Ladder¤

    1. Verify scalar conjugate updates against closed form.
    2. Verify categorical updates normalize and match direct Bayes calculations.
    3. Compare direct time-\(t\) flow samples with sequential sender updates.
    4. Overfit a tiny dataset and inspect belief trajectories.
    5. Compare teacher-forced sender trajectories with free receiver trajectories.
    6. Sweep generation steps at fixed total accuracy.
    7. Evaluate independent seeds and condition slices.
    8. Compare original message sampling with a high-accuracy specialized solver using the same network.
    9. Test final decoding/binning independently of the flow.
    10. For structured data, apply symmetry transforms and verify equivariance/invariance numerically.

    Modalities and Research¤

    BFNs have been explored for images, character-level text, proteins, molecules, graphs, crystals, planning, periodic variables, and mixed discrete/continuous structures. The evidence maturity varies:

    • the original general framework is a technical report/preprint;
    • the SDE correspondence and fast solvers are peer-reviewed research work;
    • application-specific results may be peer-reviewed, preprint, or under review and must be labeled individually.

    Graph and molecular systems must address permutation invariance, variable size, validity, and continuous geometry; a simplex over an adjacency tensor does not by itself solve graph isomorphism or chemical constraints. Geometric systems also need an input family whose support and update respect translations, rotations, periodic boundaries, or manifold topology. A network can be equivariant while its sender or endpoint observation model is not.

    Peer-Reviewed Research Checkpoints¤

    • Periodic Bayesian Flow Networks with Additive Accuracy, ICML 2026 regular, embeds a periodic scalar on the unit circle and performs Gaussian updates in the Cartesian embedding to restore additive accuracy. It is evidence for that representation and its crystal/light-field protocols, not a universal solution for manifolds.
    • Symmetry-aware Bayesian Flow Networks, npj Computational Materials 2026, incorporates crystallographic symmetry and property conditioning and reports protocol-specific stability and speed results. Space-group consistency and external stability predictors remain part of the evaluated system.
    • Discrete Bayesian Sample Inference for Graph Generation, ICLR 2026 Poster, develops a related Bayesian-sample-inference graph generator. It belongs in the broader Bayesian iterative family; containing or comparing with BFN does not make its one-shot method the canonical BFN sampler.

    Preprints and Work Under Review¤

    • General Proximal Flow Networks replaces the KL/Bayesian proximal step by other divergences. This is Preprint evidence for a proposed generalization; only the KL case recovers the canonical update.
    • Variational Bayesian Flow Network for Graph Generation uses structured Gaussian precisions and linear solves to couple node/edge beliefs. This is Preprint evidence; its geometry and empirical claims await peer-reviewed confirmation.
    • Generative Modeling with Bayesian Sample Inference is under review at TMLR and presents BFNs as a special case of a broader iterative Gaussian posterior framework. Until accepted, its theory and ImageNet comparisons remain preprint-level evidence.

    These developments extend the design space from one Euclidean conjugate family toward geometry-aware belief families, structured precisions, conditional inference, and alternative proximal updates. They do not establish that every such system retains the original codelength identity, accuracy semigroup, or SDE solver.

    Cumulative worked study: a symmetry-aware crystal BFN¤

    Consider a crystal representation containing atom count \(N\), categorical element types \(a_{1:N}\), periodic fractional coordinates \(u_{1:N}\in[0,1)^3\), lattice parameters \(\ell\), and a space-group label \(g\). A defensible BFN design declares one belief family per variable:

    1. categorical logit beliefs for \(N\), \(a\), and possibly \(g\);
    2. unit-circle embeddings \((\cos2\pi u,\sin2\pi u)\) with Gaussian evidence for each periodic scalar;
    3. a constrained continuous parameterization for valid lattices;
    4. masks so padded sites do not receive ordinary atom evidence.

    The neural output must couple all modalities while respecting permutation of sites, periodic translations, and the declared space-group action. Direct training samples each belief state from its modality-specific \(p_F\) at one common time or at explicitly calibrated modality times. The loss is a sum of derived modality integrands with justified weights—not an arbitrary average chosen after observing benchmark performance.

    During generation, each output belief produces a receiver message in its own sender family and every input belief is updated. If a predicted space group constrains allowed sites, either the transition law enforces that constraint or a post-hoc projection must be reported as a changed kernel. Final decoding maps probability beliefs to atom types and periodic embeddings back to coordinates, with degeneracy and wrap-around rules stated.

    Evaluation separates:

    • validity before and after any relaxation;
    • uniqueness and novelty against train/validation structures;
    • distributional agreement in composition, density, space group, and lattice geometry;
    • stability under the named surrogate and, where available, higher-fidelity calculations;
    • equivariance/invariance tests under equivalent crystal descriptions;
    • NFE, wall clock, hardware, relaxation cost, and guidance branches.

    Ablations replace the periodic belief by a naive Euclidean Gaussian, remove symmetry conditioning, vary receiver steps at fixed terminal accuracy, and compare the original message sampler with an SDE/ODE-derived solver. This isolates representation, network, and sampler contributions rather than crediting the family label for the whole system.

    Complete solutions to P1
    1. The belief-update implementation passes, but the receiver/model rung fails before schedule or decoder evaluation: it emits evidence centered on the wrong output. More steps repeatedly query the same misspecified receiver and are not guaranteed to repair systematic error; they can reinforce it.
    2. Learned/coarse contains model plus coarse-step error; learned/refined retains model error with reduced discretization; oracle/coarse retains coarse-step error without learned-output error; oracle/refined is the numerical reference. Compare learned with oracle at a fixed schedule to isolate model contribution, and coarse with refined at a fixed output source to isolate schedule contribution. Interactions should be reported rather than assumed additive.
    3. Name the exact architecture/checkpoint and representation; original message, reverse-SDE, probability-flow-ODE, or specialized solver; whether one stage uses multiple network calls; conditional/unconditional guidance branches; batch size, precision, accelerator, and wall-clock measurement; \(\eta\) and terminal initialization; final decoding; and relaxation or property-evaluator time. “Step” alone normalizes none of these.
    4. Clamping is a projection or fixed-state rule; sender substitution injects data-informed messages but does not generally derive the posterior over unobserved sites; a conditional network learns a declared conditional output; SMC targets a sequence of conditional/intermediate distributions using particles and resampling. Their kernels, approximation error, and compute differ, so conditional correctness must be derived and tested for each.
    5. The original derivation is primary preprint evidence for the sender--receiver/VAE identity, not proof of realized coder throughput. The ICML theorem is peer-reviewed evidence under its coordinate, truncation, and regularity assumptions, not equality of every finite sampler. The NeurIPS benchmark is peer-reviewed empirical evidence on its datasets and evaluators, not wet-lab or clinical validity. The proximal preprint is proposal-level evidence, not a peer-reviewed guarantee that non-KL updates retain additivity or coding semantics.
    6. Declare nucleotide sequence, backbone/geometry, length, and condition; use categorical beliefs for bases and a geometry-aware family for structure with rotation/translation and chain-index handling. Test direct versus sequential teacher flows, tiny exact receivers, equivariance, fixed-condition recovery, and oracle samplers. Split by homologous families to limit leakage, report all network/particle/relaxation compute, evaluate diversity and novelty, and validate selected designs with an independent structure/energy method or experiment rather than only the training predictor.

    Research Exercises¤

    1. Derive the scalar Gaussian update by completing the square.
    2. Show that two Gaussian messages can be combined in either order.
    3. Implement the three-category Bayes update and test support lockout.
    4. Compare equal-time and equal-cumulative-accuracy discretizations.
    5. Apply logit guidance to a categorical output and measure entropy versus condition accuracy.
    6. Transform a Gaussian mean/precision state into the noisy coordinate used by an SDE derivation and list assumptions.
    Exercise sketches
    1. Write prior and message log densities as \(-\rho x^2/2+\rho\mu x\) and \(-\alpha x^2/2+\alpha yx\) plus constants. Adding them gives precision \(\rho+\alpha\) and linear natural parameter \(\rho\mu+\alpha y\), hence the stated mean.
    2. After two messages the sufficient statistics are \(\rho_0+\alpha_1+\alpha_2\) and \(\rho_0\mu_0+\alpha_1y_1+\alpha_2y_2\). Addition is commutative, so message order does not change the posterior.
    3. Evaluate \(\operatorname{softmax}(\log\theta+y)\) in log space. If \(\theta_k=0\), then \(\log\theta_k=-\infty\) and finite evidence cannot revive it. The canonical uniform prior avoids exact zeros; test underflow separately.
    4. For a chosen \(\beta(t)\), compare \(t_i=i/n\) with \(t_i=\beta^{-1}(i\beta(1)/n)\). The second grid equalizes cumulative-accuracy increments but not necessarily numerical error or model uncertainty.
    5. Apply \(\log\pi_u+w(\log\pi_c-\log\pi_u)\) followed by softmax. Sweep \(w\) and report condition success, entropy, coverage, and support failures; extrapolation can sharpen or collapse outputs.
    6. Match the analytic marginal mean and covariance in the transformed coordinate, derive its forward SDE, convert the network predictor to the marginal score, state \([0,1-\eta]\), and compare finite solvers separately.

    1. What six distributions/operators define the canonical BFN state and message process?
    2. Which properties justify direct time-\(t\) training?
    3. Why can exact Bayes updates coexist with an approximate generator?
    4. What survives under the SDE correspondence, and what remains algorithm-specific?
    5. What must be re-proved for a geometric or proximal BFN extension?
    Complete retrieval solutions
    1. Input \(p_I\), output \(p_O\), sender \(p_S\), receiver \(p_R\), Bayesian update/kernel \(h,p_U\), and Bayesian flow \(p_F\), together with the accuracy schedule.
    2. The marginalized update kernels have additive accuracy, so their semigroup collapses all earlier messages into \(p_U(\theta\mid\theta_0,x;\beta(t))\).
    3. Bayes is exact only inside the chosen input family for a sampled message. The neural receiver, factorization, finite schedule, solver, guidance, and observation model can still differ from data.
    4. The derived marginal SDE and population score/denoising correspondence survive in the specified coordinate. Endpoint initialization, solver discretization, finite network error, guidance, and NFE remain algorithm-specific.
    5. Support and normalization, update tractability, accuracy composition, receiver construction, loss/codelength identity, endpoint semantics, symmetry, and any claimed SDE or benchmark correspondence.

    Final Synthesis¤

    Bayesian Flow Networks make a probability distribution's parameters the generative state. Noisy sender messages and exact Bayesian updates define training flows; neural output distributions define receiver messages for generation. Gaussian precision and categorical log evidence provide concrete additive coordinates. Continuous BFNs admit diffusion/SDE correspondences under derived transformations, while discrete BFNs retain a distinctive simplex-valued belief view. A complete implementation must align input family, sender, receiver, update, accuracy schedule, loss, sampler, and final observation model.
