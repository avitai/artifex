    6. Evaluation, Applications, and Research¤

    Chapter 6 prerequisites and learning objectives¤

    This capstone assumes all preceding process families. By the end, you should be able to select a process model from its query and consistency requirements, evaluate calibration, coherence, dynamics, extremes, and downstream utility across grids and horizons, design controlled replacements in a cumulative study, and distinguish evidence for interpolation on observed grids from evidence for a coherent continuous-domain generator.

    Prerequisites. Chapters 1–5 and the research-practice modules on evaluation, compute, and reproducibility.

    Numbered learning outcomes. After completing the chapter, you can:

    1. O1 — Select by query and law. Match a scientific need to a GP, Neural Process, latent dynamics model, stochastic operator, or function transport without choosing by benchmark rank alone.
    2. O2 — Build a layered evaluation. Combine pointwise, finite-joint, projective, conditional, pathwise, discretization, physical, and decision tests.
    3. O3 — Prevent task leakage. Design independent-function, context/target, horizon, regime, and resolution splits that separate interpolation from extrapolation and adaptation.
    4. O4 — Report uncertainty and compute. Disaggregate observation, process, parameter, approximation, and numerical uncertainty alongside training and per-query cost.
    5. O5 — Audit evidence boundaries. Trace each research claim to its named model, data, function space, discretization, approximation, and review status.
    6. O6 — Specify a reproducible study. Predeclare the object, conditioning information, baselines, ablations, metrics, failure criteria, artifacts, and negative-result policy.

    Model Selection¤

    Table PROC.4. Model Selection
    Need Strong starting point Main limitation
    Small-data calibrated regression GP kernel design and cubic scaling
    Fast adaptation across related tasks CNP/TNP meta-distribution shift and joint coherence
    Multimodal global function uncertainty latent/flow/diffusion NP inference and sampling cost
    Irregular continuous-time dynamics latent SDE/state-space model solver and latent identifiability
    Implicit realistic paths neural SDE GAN no pointwise likelihood; critic dependence
    Resolution-flexible fields stochastic operator/function diffusion discretization consistency and boundary assumptions

    Evaluation Contract¤

    Report context and target selection, observation noise, query resolution, marginal and joint log scores, calibration, coherent-sample diagnostics, interpolation versus extrapolation, task-distribution shift, solver error, and wall clock. For physical fields, include symmetries, conservation residuals, boundary conditions, and units.

    Table PROC.5. Evaluation Contract
    Layer Minimum test What it catches
    pointwise RMSE/NLL and coverage by location/regime mean error and marginal miscalibration
    finite joint joint NLL, energy/kernel score, covariance dependence errors hidden by marginals
    projective nested-query restriction and permutations query-set inconsistency
    conditional add-as-context versus probabilistic conditioning learned conditioning gap
    pathwise increments, spectra, hitting/event statistics wrong temporal law
    discretization coarse/fine pushforward and refinement grid-specific generator
    physical boundary, conservation, units, symmetries implausible scientific fields
    decision control, BO, or inverse-task regret uncertainty irrelevant to use

    Use proper scoring rules where tractable and report Monte Carlo estimator error. Energy/MMD scores on finite vectors depend on coordinate weighting and grid density; quadrature-aware norms are needed before comparing resolutions. Coverage without interval width rewards vacuous uncertainty, while sharpness without coverage rewards overconfidence.

    The ladder prevents a pointwise score from standing in for a process-level claim. A study need not pass every layer, but its conclusion must stop at the highest layer actually tested.

    Data and Compute Protocol¤

    Split by independent functions/tasks before splitting coordinates; otherwise observations from the same latent function can leak across train and test. Report the number of tasks, observations per task, context-size distribution, query mechanism, spatial/temporal resolution, normalization units, and whether missingness is simulated or real.

    Compute reporting includes parameters, active parameters, training function evaluations, solver NFE, memory, wall clock/hardware, sampling batch size, and cost per queried coordinate or field. For amortized methods, separate one-time training cost from per-task adaptation and sampling.

    Failure Modes¤

    • correct marginals but incoherent target covariance;
    • context aggregation bottlenecks;
    • latent collapse and underdispersed samples;
    • meta-training leakage between tasks/functions;
    • interpolation-heavy evaluation presented as forecasting;
    • grid-specific noise scaling;
    • neural operator violating boundary or conservation constraints;
    • SDE solver error mistaken for stochastic uncertainty;

    • symmetry assumptions failing under nonstationarity.

    • learned conditioning updates that disagree with the model's earlier joint law;

    • an identity-noise discretization presented as an \(H\)-valued Gaussian;
    • function-space claims without a named norm/reference measure;
    • test tasks sharing latent trajectories or PDE simulations with training;
    • stochasticity that conflates observation noise, parameter uncertainty, and intrinsic forcing.

    Cumulative Worked Study: Irregular Climate Fields¤

    Design a conditional generator for daily temperature fields from sparse stations and uncertain forcing.

    1. Object: a field \(f(t,s)\) with declared temporal interval, spatial domain, units, and regularity.
    2. Context: station observations with sensor-specific noise and a documented placement/missingness process.
    3. Conditioning: past observations imply forecasting; observations on both sides imply smoothing; forcing scenarios are separate covariates.
    4. Family: a GP is an interpretable small-data baseline; an equivariant NP amortizes across days/regions; a latent SDE handles temporal paths; function diffusion/operator flow represents non-Gaussian fields.
    5. Law: specify whether randomness represents weather variability, measurement error, parameter uncertainty, or unresolved physics.
    6. Evaluation: hold out whole weather systems/years, test station permutations and nested grids, check spectra/extremes/spell duration, calibrate regional integrals, and verify boundary/physical units.
    7. Evidence: compare at equal data and compute, report independent-task uncertainty, and avoid calling reanalysis agreement an independent physical validation.

    The “best” family depends on which of these semantics and workloads matter. A lower pointwise RMSE does not compensate for incoherent extreme-event probabilities.

    Independent practice and assessment map: evaluation and research¤

    1. P1.1 — Predict before computing. Type: predict · Difficulty: foundation · Typical time: 10 min. For sparse stations and few historical fields, predict which of a GP, factorized CNP, latent SDE, and function diffusion provides the strongest transparent baseline for calibrated small-data regression. Then name the conditions that could reverse the choice.
    2. P1.2 — Evaluation-stack design. Type: design · Difficulty: core · Typical time: 18 min. Build a minimum evaluation stack for a claimed resolution-flexible field generator. Include one pointwise, finite-joint, projective, physical, and decision metric, plus uncertainty for estimated metrics.
    3. P1.3 — Leakage diagnosis. Type: diagnose · Difficulty: core · Typical time: 12 min. A dataset randomly splits coordinates from every simulated trajectory between train and test. Explain which generalization claims remain possible, which are invalid, and how to redesign the split.
    4. P1.4 — Uncertainty-and-compute ledger. Type: evaluate · Difficulty: stretch · Typical time: 18 min. Specify separate measurements for observation, process, parameter, inference, discretization, and Monte Carlo uncertainty, then pair them with training and per-query compute reporting.
    5. P1.5 — Evidence-boundary audit. Type: research · Difficulty: stretch · Typical time: 20 min. Choose one work from the research evidence map and write a supported claim, an unsupported stronger claim, the decisive experiment or theorem, and one independent reproduction target.
    6. P1.6 — Transfer. Type: transfer · Difficulty: stretch · Typical time: 25 min. Pre-register a matched comparison of a GP, Neural Process, and stochastic operator on irregular climate fields. Declare data units, conditioning query, budgets, ablations, failure criteria, artifacts, and how negative results will be reported.

    Complete solutions to P1
    1. P1.1. A GP is the strongest transparent starting baseline in the stated small-data regime because it declares a kernel, observation law, exact finite Gaussian posterior under conjugacy, and calibrated quantities that can be checked. Many related tasks can justify an amortized NP; multimodal field laws can justify function diffusion; irregular temporal dynamics can justify a latent SDE. Check. The answer chooses from the query and evidence regime rather than a universal family ranking.
    2. P1.2. One stack uses quadrature-aware RMSE and NLL/coverage, an energy or kernel score for finite joints, a coarse-versus-restricted-fine two-sample discrepancy, boundary/conservation residuals, and downstream forecast/control regret. Every Monte Carlo metric reports standard error or interval across samples and seeds. Rubric. A second pointwise metric cannot substitute for the joint or projective layer.
    3. P1.3. The split can assess interpolation at held-out coordinates of already-seen trajectories, subject to leakage through preprocessing and context selection. It cannot support independent-function, new-regime, new-simulation, or forecasting claims. Hold out complete trajectories/functions or regimes first, then form context/target splits only inside held-out test tasks; use directional time splits for forecasting. Check. No latent trajectory crosses the train/test task boundary.
    4. P1.4. Use replicated sensors or a likelihood for observation noise; repeated future paths conditional on fixed inputs for process variability; posterior draws or sensitivity for parameters; method comparisons and gap diagnostics for inference; mesh/step refinement for discretization; and repeated samples/seeds for Monte Carlo error. Report one-time training data/steps/hardware, parameters and memory, solver NFE, sampling throughput, and cost per queried field/coordinate. Rubric. Combining these into one predictive interval without provenance fails the ledger.
    5. P1.5. For Operator Flow Matching, a supported claim is that the named method learns stochastic-process priors with finite-query density and regression machinery in its declared function-space construction and experiments. An unsupported claim is that every neural operator gives exact, calibrated likelihoods on arbitrary grids. The paper's construction and its sampled-query, ODE, trace, and posterior approximations define the boundary; an independent target is a restriction-consistency and density audit across held-out grids with estimator error. Rubric. Another evidence-map work is acceptable when the four requested components are equally explicit.
    6. P1.6. Predeclare field units/domain, station and forcing observation models, forecast versus smoothing query, complete-year/system train-test splits, equal data and compute envelopes, kernel/NP/operator choices, uncertainty sources, and seed count. Compare point, joint, projective, physical, extreme-event, and cost metrics; ablate shared latents, reference process, resolution training, and boundary features; publish configs, code, checkpoints, manifests, raw metrics, and failed runs. Rubric. Negative coherence or calibration results remain in the report and narrow the conclusion rather than disappearing behind the best RMSE.

    Research Exercises¤

    1. Construct a pair of finite-dimensional laws that violate marginal consistency.
    2. Sample a GP on nested grids and verify restriction consistency numerically.
    3. Compare independent CNP target sampling with a shared-latent NP on a periodic task.
    4. Evaluate interpolation and extrapolation with separate context splits.
    5. Simulate the OU transition at irregular gaps and compare with a fixed-step approximation.
    6. Test a function diffusion model by coarse/fine restriction consistency.
    7. Measure the conditioning consistency gap by turning a target into context.
    8. Compare identity grid noise with a trace-class GP reference under refinement.
    Exercise sketches
    1. Give one-point and two-point distributions with different marginals.
    2. Both are marginals of one joint Gaussian kernel matrix.
    3. Independent samples lack shared phase/function uncertainty.
    4. Random splits often overstate extrapolation.
    5. Exact decay and variance depend on \(\Delta\).
    6. Fine samples restricted to coarse points should match direct coarse laws within statistical/numerical error.
    7. Compare fresh re-encoding against the conditional of the earlier joint prediction.
    8. Identity noise does not approach the same regular \(H\)-valued law without an appropriate scaling/reference construction.

    Source-Tracing Checklist¤

    For every result, record the paper version and evidence status; process/function space and norm; finite-grid or infinite-dimensional claim; context/target and task split; observation/noise semantics; architecture and objective; query/grid resolution; solver and NFE; calibration/joint/path metrics; and independent reproduction status.

    Cumulative retrieval: stochastic-process generators
    1. What consistency condition distinguishes a process model from unrelated predictors for each query grid?
    2. Which uncertainty sources should a latent stochastic-dynamics model separate?
    3. Why can low fixed-grid error coexist with incoherent refinement behavior?
    4. What evaluation supports a claim of function-space rather than discretization-specific generation?

    Solutions. (1) Marginals on overlapping finite query sets must agree under restriction, up to explicitly bounded approximation. (2) Latent initial state, process/dynamics noise, parameter or posterior uncertainty, and observation noise. (3) The network can learn grid-index correlations without defining compatible laws when points are inserted, removed, or moved. (4) Held-out locations, irregular and refined grids, restriction consistency, calibrated joint samples, boundary/physics tests, and comparison across discretizations with the same underlying functions.

    Final Synthesis¤

    Stochastic-process generators model random functions rather than fixed vectors. GPs guarantee coherent finite-dimensional laws analytically; Neural Processes amortize context-conditioned prediction across tasks; latent variables and attention improve global uncertainty and local retrieval; neural SDEs generate path measures; function diffusion and stochastic operators extend denoising and learned solution maps to fields. The central acceptance test is cross-query and cross-resolution coherence, not merely the ability to accept variable-size tensors.
