    4. Planning, Imagination, and Model Exploitation¤

    Chapter 4 prerequisites and learning objectives¤

    This chapter assumes latent dynamics, objectives, and elementary planning. By the end, you should be able to distinguish prediction quality from planning utility, formulate rollout and value queries under model uncertainty, explain model exploitation and compounding error, compare online and offline imagination policies, and design conservative or uncertainty-aware controls with real- environment validation.

    Model-predictive control¤

    At time \(t\), MPC chooses an action sequence by solving

    \[ a_{t:t+H-1}^{*} =\arg\max_a\; \mathbb E_{\hat P} \left[\sum_{k=0}^{H-1}\gamma^k r_{t+k} +\gamma^H\hat V(s_{t+H})\right], \]

    executes only the first action, observes the environment, and replans. CEM samples action sequences, retains high-scoring elites, refits a proposal, and repeats. Gradient planners differentiate through the model. MCTS builds a selective search tree. Each adds approximation beyond model error.

    Imagination and actor learning¤

    Dreamer rolls the latent prior forward, trains a critic on imagined returns, and trains an actor using those returns. This amortizes planning into a policy. It reduces online search cost but can propagate model bias into both actor and critic. Mixing real and imagined targets, short horizons with terminal values, and repeated data collection can limit—but not eliminate—the feedback loop.

    Planner's curse¤

    Optimization does not query the model like a held-out predictor. It deliberately finds action sequences with unusually high predicted reward, including sequences where the model extrapolates badly. Suppose data contain \(a\in[-1,1]\) and a learned model predicts \(\hat r(a)=a\). An unconstrained optimizer selects \(a=100\) even if the real actuator saturates and breaks. Low test error inside \([-1,1]\) is irrelevant to the selected action.

    Defenses include action constraints, uncertainty penalties, pessimistic model sets, ensembles, behavior regularization, conservative value estimates, short-horizon replanning, and real-environment validation. All encode assumptions; excessive pessimism can also prevent useful exploration.

    Planning versus rendering¤

    Interactive video models such as Genie 2 and GameNGen demonstrate action-conditioned neural rendering in reported game-like settings. They are Vendor claim / technical report evidence until an archival paper and independent reproduction establish broader conclusions. Visual plausibility does not by itself measure reward accuracy, collision constraints, persistent object state, or policy regret.

    Conversely, latent planning can choose effective actions without generating pixels. A fair comparison reports the intended query: interactive media, control, data generation, or causal analysis.

    Distribution shift and feedback¤

    The behavior policy \(\mu\) determines collected actions and visited states. A new planner induces policy \(\pi\), changing its own input distribution. Online model-based RL can collect corrective data; offline planning cannot and therefore needs coverage assumptions or conservative objectives. A simulator trained on expert data may fail precisely on recovery states a learner visits.

    Safety and constraints¤

    Expected reward hides tail risk. For safety-critical planning report collision/constraint probabilities, calibration, worst-case or risk-sensitive objectives, and recovery behavior. A plausible sampled future is not evidence that rare hazards receive correct probability.

    The five-component loop for planning¤

    A world-model system is easier to reason about when it is decomposed into the same five components used for generative design:

    Table WORLD.3. The five-component loop for planning
    Component World-model realization Typical failure
    1. reference and support posterior belief over current state, behavior-policy coverage, and the learned trajectory support the planner requests states or actions absent from data or impossible in the environment
    2. predictive model transition, observation, reward, termination, and uncertainty models one-step or compounding model error; uncalibrated epistemic uncertainty
    3. target construction return, terminal value, goal likelihood, risk functional, and hard constraints proxy reward omits what matters or creates an unsafe trade-off
    4. inference engine CEM, MPC, MCTS, gradient planning, SMC, or an imagined actor finite-search error or deliberate exploitation of model error
    5. evidence and update execute, observe the environment, compare forecast with outcome, update belief/model, and replan open-loop deployment, delayed failures, or feedback that is never used to correct the model

    For a learned trajectory law \(\hat p(\tau\mid h_t)\), one possible stochastic planning target is

    \[ p_{\beta}(\tau\mid h_t) \propto \hat p(\tau\mid h_t) \exp\!\big(\beta U(\tau)\big) \mathbf 1\{\tau\in\mathcal C\}, \]

    where \(U\) is a declared utility and \(\mathcal C\) is a feasible set. Sampling this law, estimating an expectation under it, and taking an approximate maximizer are different queries. The limit \(\beta\to\infty\) may concentrate on model-optimal trajectories, but it can also magnify tiny optimistic errors. A deterministic planner need not admit this probabilistic interpretation; the equation is a useful target construction, not a definition of planning.

    The component boundary matters operationally. Replacing CEM by a differentiable planner changes inference without changing the learned dynamics. Replacing pixel prediction by value-equivalent dynamics changes the predictive target. Adding an uncertainty penalty changes component 3, while improving ensemble calibration changes component 2. Executing only the first action and observing the real transition activates component 5; generating a long video without correction does not.

    The three-candidate design example has a direct planning analogue. Candidate designs become action sequences, predicted property becomes predicted return, and prospective assay becomes real execution. In both cases, the optimizer selects where the learned model is most favorable, so planner-selected trajectories must be evaluated rather than relying only on behavior-policy holdout data.

    Retrieval question: which component is a world model?

    The learned transition/observation/reward system is component 2, together with the current-state inference needed to query it. A complete decision system additionally needs reachable support, an objective and constraints, a planner or policy, and an environment evidence loop. Calling the whole stack a “world model” hides which component failed.

    Complete planning accounting¤

    Report model parameters and training data, planning horizon, candidate count, optimization iterations, particles/samples, world-model NFE, decoder calls, replanning frequency, terminal value, hardware, latency, and environment interactions. “One planning step” can contain thousands of model evaluations.

    Retrieval question: why can a better predictor yield a worse controller?

    Aggregate predictive loss can improve on common or visually salient variables while worsening small decision-critical variables. The planner may also exploit a new extrapolation error, and its finite search algorithm may interact differently with the new model.

    Chapter 4 synthesis¤

    Planning is an adversarial query generator for model weaknesses. The five-component decomposition localizes whether failure came from support, prediction, the target, inference, or evidence. Chapter 5 asks when action-conditioned prediction supports the stronger language of interventions and counterfactuals.
