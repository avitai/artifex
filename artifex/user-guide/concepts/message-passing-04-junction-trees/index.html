    4. Junction Trees and Reusable Exact Inference¤

    Chapter 4 prerequisites and learning objectives¤

    This chapter assumes variable elimination, induced width, and exact tree messages. By the end, you should be able to:

    • construct the clique scopes induced by an elimination order;
    • state the running-intersection property;
    • derive a clique-to-clique separator message;
    • explain how junction-tree calibration reuses work across many marginals; and
    • connect elimination to compiled arithmetic and probabilistic circuits without equating their representation costs.

    Turn a loopy graph into a tree of clusters¤

    The junction-tree algorithm performs exact message passing on clusters. For a discrete undirected graph, a standard construction is:

    1. choose an elimination order and add its fill-in edges, producing a chordal graph;
    2. identify the maximal cliques of that chordal graph;
    3. connect cliques into a tree whose edge weights favor large intersections; and
    4. require the running-intersection property: all cliques containing any variable \(X_i\) form a connected subtree.

    For a directed Bayesian network, first incorporate evidence and convert the relevant factorization to undirected scopes. Graphically this includes connecting co-parents when their child conditional is treated as one factor; a practical implementation can work directly from factor scopes instead of relying on a hand-drawn moralization step.

    Let adjacent clique nodes have scopes \(C_i\) and \(C_j\) and separator \(S_{ij}=C_i\cap C_j\). Assign every original factor to one clique containing its scope and multiply assigned factors into clique potential \(\psi_i\). The message is

    \[ m_{i\to j}(x_{S_{ij}}) =\sum_{x_{C_i\setminus S_{ij}}} \psi_i(x_{C_i}) \prod_{k\in N(i)\setminus\{j\}} m_{k\to i}(x_{S_{ki}}). \]

    This is sum-product again: the “variable” transmitted across an edge is now a separator tuple. After a collect and distribute pass, neighboring clique beliefs agree on their separator marginals. Any node or factor marginal contained in a clique can then be extracted locally.

    Worked example: triangulate a four-cycle¤

    Suppose a model has pair factors on the cycle \(A-B-C-D-A\). Eliminating \(A\) first adds the chord \(B-D\). One valid pair of maximal cliques is

    \[ C_1=\{A,B,D\}, \qquad C_2=\{B,C,D\}, \]

    with separator

    \[ S_{12}=\{B,D\}. \]

    Assign \(f_{AB}\) and \(f_{DA}\) to \(C_1\), and \(f_{BC}\) and \(f_{CD}\) to \(C_2\). The left-to-right message is

    \[ m_{1\to2}(b,d)=\sum_a f_{AB}(a,b)f_{DA}(d,a). \]

    The right-to-left message similarly sums out \(c\). Calibrated clique beliefs can answer \(p(A,B)\) from \(C_1\), \(p(B,C)\) from \(C_2\), and \(p(B,D)\) from either clique. For binary variables, each clique table has \(2^3\) entries. Exactness was recovered by clustering, but its cost rose from pairwise tables to width-two clique tables.

    Variable elimination versus a junction tree¤

    Variable elimination is efficient for one target marginal and one evidence pattern when intermediate factors need not be reused. A calibrated junction tree stores messages or clique beliefs so that many compatible marginals can be read without repeating the complete elimination. Changing evidence may permit local repropagation; changing factor scopes or the elimination order may require recompilation.

    Table MSG.7. Variable elimination versus a junction tree
    Representation Main reusable object Strength Cost boundary
    one VE run intermediate factors for one order/query simple exact contraction repeated queries may repeat work
    junction tree calibrated clique and separator messages all contained marginals after calibration clique tables exponential in width
    arithmetic circuit DAG of sums and products repeated evaluation and differentiation under changing evidence/parameters compiled DAG may be large
    probabilistic circuit a distribution represented directly by constrained sums/products selected exact queries linear in circuit size compact valid representation is not guaranteed to exist or be easy to learn

    Elimination is compilation¤

    An elimination order creates an expression made of local additions and multiplications. Keeping that expression as a DAG rather than immediately evaluating it yields an arithmetic circuit. Shared subexpressions can be reused for new evidence, parameter values, derivatives, and related marginals. This is algorithmic amortization by compilation, not learned amortized inference.

    Probabilistic circuits make the compiled representation the model interface. Smoothness and decomposability support exact marginal evaluation by construction; determinism enables additional maximization queries; structured decomposability controls compatibility between circuits. These properties exploit more than a graph's coarse treewidth, including context-specific independence and logical support. The cost statement is linear in the compiled circuit size, which itself can be exponential in the original model.

    Query-aware exactness¤

    “The model has a junction tree” is incomplete because every finite graph can be triangulated. Ask instead:

    • What is the induced width under the actual order?
    • Which variable cardinalities determine clique entries?
    • Are the factors dense, sparse, deterministic, low rank, Gaussian, or symbolically integrable?
    • Is one query required, all marginals, repeated evidence, MAP, or sampling?
    • What compilation and calibration memory is retained?

    The tree decomposition can also guide hybrid algorithms: exact messages within tractable clusters, sampling across cutsets, variational approximations for selected factors, or neural proposals for costly local integrals. Each hybrid must identify which operations remain exact.

    Counterexample: a tree of huge cliques is not tractable

    The junction graph is always a tree after construction, but a single clique may contain most variables. Tree-shaped control flow does not overcome the exponential state table inside that clique.

    Chapter 4 synthesis¤

    Junction trees transform loops into cluster width. Calibration reuses exact elimination work across local queries, while arithmetic and probabilistic circuits preserve the same distributive computation in a compiled DAG. Chapter 5 keeps the original loopy graph and accepts approximation instead.

    Retrieval practice: clusters and compilation
    1. What must be true of the cliques containing one variable?
    2. What variables index a message between adjacent cliques?
    3. Why can junction-tree inference be exact but impractical?
    4. How does compilation differ from a learned amortized encoder?

    Solutions. (1) They form a connected subtree—the running-intersection property. (2) Their separator, the intersection of the clique scopes. (3) Clique tables are exponential in induced width and variable cardinalities. (4) Compilation caches an exact algebraic expression for a represented model; a learned encoder predicts an approximate answer after training on a task distribution.

    Bridge. Exact clustering pays exponential width. Loopy belief propagation keeps small local messages, but its fixed points require a different correctness contract. Continue continuously or open Chapter 5.
