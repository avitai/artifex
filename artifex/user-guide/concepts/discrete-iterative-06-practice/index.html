    6. Training, Sampling, Evaluation, and Research Directions¤

    Prerequisites and capstone objectives¤

    This chapter assumes all previous chapters plus basic experimental design. By the end, you should be able to:

    1. implement a target-aware training loop without conflating mask CE, ELBO, ratio, and flow losses;
    2. audit a sampler as a stochastic kernel, including numerical and policy approximations;
    3. design oracle and learned-model experiments that isolate denoiser error from sampler error;
    4. evaluate likelihood semantics, coverage, constraints, representation quality, and latency together;
    5. turn failure hypotheses into controlled interventions and falsifiable diagnostics; and
    6. place research results at their correct evidence level without turning formal or protocol-specific claims into universal rankings.

    The capstone principle is simple: the trained network is not the deployed generator. The deployed generator is the network, representation, transition law, numerical method, guidance, constraints, stopping rule, and randomness contract composed together.

    Moving upward in realism adds confounders. Moving downward in this ladder removes them. A credible diagnosis reports both directions: controlled oracle evidence for attribution and end-to-end evidence for relevance.

    The derivative layer is equally composite. Masked cross-entropy, transition likelihoods, ELBO terms, trajectory rewards, and straight-through discrete surrogates require different estimators. Use Gradients and Geometry in Generative Modeling to type the objective, random nodes, relaxation, and control variate before comparing training rules.

    Training Checklist¤

    1. State the corruption/mask distribution and time sampling law.
    2. State the target: clean token, previous token, ratio, rate, or transition logits.
    3. Define loss weighting and whether it is an ELBO, auxiliary CE, score entropy, or flow objective.
    4. Verify padding and mask tokens are not conflated unless intended.
    5. Test transition matrices for nonnegativity and row normalization; test CTMC generators for zero row sums.
    6. Measure accuracy/calibration across noise levels and mask ratios, not only the average.
    7. Audit tokenizer/codebook utilization for non-text modalities.

    A family-neutral training skeleton is:

    repeat
    draw clean object x0 and condition c
    draw training query: mask m, time t, or source endpoint x_source
    sample corrupted/intermediate state xt from the declared path
    predict the declared target from (xt, t, c)
    evaluate the matching loss and importance/time weight
    update parameters; record loss by time, corruption, length, and condition
until the data, optimization, and compute budget is exhausted

    Every noun in this pseudocode is part of the experiment. If time is sampled from proposal \(r(t)\) while the intended objective integrates weight \(w(t)\), an unbiased Monte Carlo estimate uses \(w(t)/r(t)\). Clipping that weight may improve stability but changes the optimized objective. Likewise, sampling mask ratios uniformly is not equivalent to sampling time uniformly when mask survival is a nonlinear function of time.

    Target and likelihood semantics¤

    Table DISC.5. Target and likelihood semantics
    Training target Population object Common evaluation boundary
    masked clean-token CE conditional clean-token marginals under mask law not a joint likelihood without another construction
    D3PM reverse KL / ELBO reverse transition matching under known forward chain variational bound can be loose
    absorbing Rao–Blackwellized CE a particular continuous-time masked-diffusion ELBO requires its reverse parameterization and weights
    denoising score entropy noisy marginal ratios on state-graph edges support, neighbor sum, and rate kernel matter
    discrete flow posterior CE endpoint/source posterior for a declared path and coupling correct population posterior does not remove sampler discretization
    consistency or distillation loss agreement across larger transitions or teacher paths adds training protocol and may change target system

    Do not compare raw values across rows. They have different units, reductions, time weights, and variational meanings.

    Sampler Audit¤

    Record the full state transition:

    • which positions are eligible to change;
    • whether proposals are sampled or argmax;
    • confidence computation and tie-breaking;
    • remasking/commit schedule;
    • temperature, top-k/top-p, and guidance;
    • number of model evaluations and corrector steps;
    • whether previously fixed tokens can change;
    • termination rule and invalid-state handling.

    Two papers using the same network objective but different confidence schedules can define materially different generators.

    A useful sampler skeleton makes policy decisions explicit:

    state = draw_reference(length, condition, rng)
for interval in schedule:
    estimate = model(state, interval.time, condition)
    kernel = build_transition(estimate, interval, guidance, constraints)
    state = sample_kernel(kernel, rng)
    state = optional_corrector_or_projection(state, estimate, rng)
return decode(state)

    For reproducibility, use a stateless or explicitly split random-key discipline. Proposal sampling, confidence perturbation, event counts, tie-breaking, and decoder stochasticity require separate recorded streams. Batched samples should not accidentally share a mask pattern or Poisson event count unless that coupling is intentional.

    Error decomposition by intervention¤

    It is tempting to write one additive “diffusion error,” but the terms interact. A more defensible diagnostic separates them experimentally:

    • reference/terminal error: the initialized law differs from the intended terminal marginal;
    • estimator error: learned posterior, ratio, or rate differs from its population target;
    • time-discretization error: a finite step approximates a time-varying process;
    • parallel-kernel error: independently sampled simultaneous coordinates omit conditional dependence;
    • policy error: confidence selection or adaptive scheduling changes the intended transition law;
    • intervention error: guidance, clipping, rejection, or validity projection changes rates or probabilities;
    • representation error: tokenizer or codec excludes or distorts relevant outputs.

    Use exact finite-state posteriors to remove estimator error, very fine steps to estimate discretization limits, one-coordinate updates to remove parallel factorization, and codec reconstructions to bound representation error. The interventions will not make the errors perfectly additive, but they identify which subsystem causes a discrepancy.

    Evaluation Contract¤

    Table DISC.6. Evaluation Contract
    Claim Required evidence
    Faster than AR same tokenizer/backbone/condition, wall clock, batch, hardware, quality operating point
    Better likelihood exact joint, variational bound, or surrogate identified; same tokenization/reduction
    Better infilling mask distribution, constraint satisfaction, diversity, and human/task evaluation
    Fewer steps what a step/NFE performs, parallel events, guidance branches, and decoder cost
    Better coverage precision/recall, rare-mode and condition slices, memorization checks
    Valid structured samples exact validity, uniqueness, novelty, and property calibration

    Likelihood and sample evaluation answer different questions. An ELBO evaluates a model/path objective; samples evaluate the numerical and policy composition. A strong held-out bound can coexist with a poor few-step sampler. Conversely, a guided sampler can score well on preferences while no longer sampling the unguided model whose bound was reported.

    For discrete language, report at least held-out likelihood or a clearly named bound, entropy, external-model generative perplexity with the evaluator named, diversity/duplication, task success, and human judgments where appropriate. External-model perplexity is not likelihood under the generator and inherits evaluator bias. For structured science domains, add validity, novelty relative to train data, property calibration, and downstream experimental uncertainty.

    An oracle finite-state benchmark is unusually valuable here. Choose a small HMM or graphical model with exact \(p(x_0)\) and exact denoising posteriors; run the production sampler with the oracle denoiser; enumerate its output distribution; then compute total variation, KL where support allows, transition statistics, and aggregate text-style metrics. This reveals whether a seemingly good metric can hide sampler-induced distributional error.

    Failure Modes¤

    • Incompatible masked conditionals: good local CE but incoherent iterative samples.
    • Early commitment: confident wrong tokens constrain all later rounds.
    • Oscillation: remasked tokens alternate between locally plausible states.
    • Parallel-factorization artifacts: mutually dependent positions update inconsistently.
    • Schedule mismatch: the model is evaluated at mask ratios/noise states poorly represented in training.
    • Codec bottleneck: prior improvements cannot exceed reconstruction support.
    • Guidance collapse: condition adherence rises while diversity falls.
    • Terminal mismatch: reverse sampling starts from an approximation that is not close enough under the metric used by the theorem.
    • Rate explosion or clipping: small marginal probabilities create large reverse ratios, then implementation clipping silently defines a different process.
    • Length/padding leakage: a fixed-length model learns padding artifacts or an evaluation strips states the model treated probabilistically.
    • Cache illusion: nominally parallel generation repeatedly recomputes clean context and is slower than a cache-aware baseline.

    Research Directions¤

    Active directions include adaptive CTMC solvers, state-size-independent error analysis under structured assumptions, discrete consistency/few-step maps, learned transition graphs, insertion/deletion processes, mixed discrete–continuous generators, and post-training for reasoning. Research claims should distinguish peer-reviewed empirical results from theory under ideal ratio error.

    Recent sampling theory studies tau-leaping convergence under score-entropy control, including structure-adaptive rates for masking processes. Such results are peer-reviewed research and assumption-bound; they do not certify an arbitrary large language model with clipped approximate ratios.

    Three COLT papers establish complementary results. Optimal Inference Schedules for Masked Diffusion Models gives an exact expected-divergence characterization for a distribution and unmasking schedule, proves that competing with the distribution-specific optimum can require strong prior knowledge, and relates achievable parallelism to total and dual total correlation. Efficient Sampling with Discrete Diffusion Models gives sharp/adaptive tau-leaping guarantees under score-entropy control: its uniform-process complexity has unavoidable ambient-dimension dependence in general, while its masking result adapts through an effective-total-correlation quantity. Quantized Transition Diffusion reformulates a continuous target through spatial quantization and density ratios, using truncated uniformization to obtain its stated total-variation query bound. These are peer-reviewed research results. Their metrics, corruption processes, estimator assumptions, and algorithms differ; none follows merely from using a masked Transformer.

    This 2026 theory complements, rather than supersedes, NeurIPS 2025 sampler guarantees for tau-leaping, Euler, and Tweedie-style methods and absorbing-process analysis. Successively improved bounds use different assumptions and algorithms; quoting only asymptotic iteration counts without the divergence, dimension, vocabulary, score-error, and initialization terms is not a meaningful comparison.

    Categorical Flow Maps (ICML 2026) take a different few-step route: a self-distilled map moves probability toward a predicted endpoint in the simplex, allowing continuous-map distillation techniques while ultimately generating categorical objects. The paper reports image, molecular-graph, and text experiments, including one-step settings. This is peer-reviewed empirical research for its parameterization and protocols—not evidence that its continuous simplex trajectory is the same object as the CTMC token trajectory in Chapter 3, nor that one-step quality is distributional correctness.

    Peer-Reviewed Language-Model Research¤

    The following results were verified against their ICLR/ICML 2026 OpenReview decisions on 2026-07-18. They address different objectives, samplers, and evaluations and should not be collapsed into one “diffusion LM” result:

    • AR-to-diffusion conversion: Efficient-DLM: From Autoregressive to Diffusion Language Models, and Beyond in Speed (ICML 2026) continues pretraining an autoregressive model with block-wise attention and position-dependent masking. Its reported 8B accuracy/throughput gains are comparisons against named models under the paper's serving protocol, not a theorem that converted DLMs dominate AR models.
    • Parallel-sampling theory: Diffusion Language Models are Provably Optimal Parallel Samplers (ICLR 2026) proves simulation results in a formal parallel-sampling model, using polynomial-length chain-of-thought and, for optimal space, revision or remasking. The theorem is about representational/sequential-depth capability; it does not give a trained finite network an optimal wall-clock sampler.
    • Variable-length state spaces: Beyond Masks: Efficient, Flexible Diffusion Language Models via Deletion-Insertion Processes (ICLR 2026) defines deletion and insertion CTMCs over the union of all finite sequence lengths, avoiding mandatory mask/pad states and deriving a likelihood-bounded objective with dynamic-programming subsequence counts.
    • Few-step distillation: Ultra-Fast Language Generation via Discrete Diffusion Divergence Instruct (ICLR 2026) distils a pretrained discrete-diffusion LM with an integral-KL objective and reports protocol-specific perplexity/acceleration trade-offs. The teacher, student, NFE, entropy, and extra training cost remain part of the system comparison.
    • Sampler correctness: Is Your Diffusion Sampler Actually Correct? A Sampler-Centric Evaluation of Discrete Diffusion Language Models (ICML 2026) uses an exact HMM denoiser to isolate sampling error. Its experiments show that few-step parallel samplers can be distributionally wrong even with an oracle denoiser and that NLL, generative perplexity, or MAUVE need not reveal the mismatch. This directly motivates the sampler audit above.
    • Predictor–corrector inference: The Diffusion Duality, Chapter II: Ψ-Samplers and Efficient Curriculum (ICLR 2026) generalizes discrete predictor–corrector sampling across noising processes and reports protocol-bounded gains for uniform-state language and image diffusion. These experiments challenge a universal preference for absorbing masks; they do not prove one corruption dominates across data and compute regimes.
    • Single-stage few-step consistency: Consistent Diffusion Language Models (ICML 2026) train agreement in expectation across exact discrete posterior bridges rather than importing a nonexistent categorical probability-flow ODE. Its few-step results are author experiments under the paper's model/data protocols, and its claimed limiting relationships require their stated bridge construction.
    • Architecture–sampler co-design: Efficient Parallel Samplers for Recurrent-Depth Models and Their Connection to Diffusion Language Models (ICML 2026 Spotlight) reinterprets refinement in recurrent-depth transformers to advance token positions while latent computation continues. The reported speedup applies to named 3.5B recurrent-depth models and hardware budgets; it is not a generic conversion of fixed-depth AR models.
    • Any-order/AR hybrid: Esoteric Language Models: A Family of Any-Order Diffusion LLMs (ICML 2026) connects masked diffusion to any-order autoregression and uses causal attention to obtain exact KV caching and exact likelihood for its construction while interpolating between parallel and sequential generation. Its reported speed–quality frontier is an empirical result under the paper's model, schedule, and serving protocol; it does not make exact caching or exact likelihood available to arbitrary bidirectional masked denoisers.

    Together these papers sharpen four separate questions: what distribution/objective is learned, what transition kernel is executed, how architecture exposes parallel work, and what latency/quality operating point is measured. Progress on one does not answer the other three.

    Cumulative worked study: an oracle denoiser with a wrong sampler¤

    Let the length-two data law be

    \[ p_{\mathrm{data}}(00)=p_{\mathrm{data}}(11)=\tfrac12, \qquad p_{\mathrm{data}}(01)=p_{\mathrm{data}}(10)=0. \]

    At the fully masked state [M,M], an oracle clean-token denoiser returns marginal probability \(1/2\) for 0 and 1 at both positions. A one-step parallel sampler draws the positions independently, so its output law is uniform over all four strings. Its total-variation distance from data is

    \[ \operatorname{TV}(p_{\mathrm{parallel}},p_{\mathrm{data}}) =\frac12\sum_x|p_{\mathrm{parallel}}(x)-p_{\mathrm{data}}(x)| =\frac12. \]

    The denoiser is exact; all error comes from the factorized simultaneous kernel. An order sampler that first draws either coordinate from \((1/2,1/2)\) and then draws the other from its exact conditional produces only 00 or 11 and is exact. Thus “perfect token predictions” and “perfect joint samples” are different claims.

    Now build the full experiment:

    1. train a network on masks from this known law and compare its conditionals with the oracle;
    2. substitute the oracle into each production sampler to isolate sampler error;
    3. sweep number of updated coordinates, remasking, and NFE;
    4. report exact TV/KL where supported, pair agreement, entropy, and an external aggregate score;
    5. repeat with a perturbed denoiser to measure interaction between estimator and sampler errors.

    The design scales conceptually to an HMM, where forward–backward inference supplies exact masked posteriors while dynamic programming supplies the target sequence law. It also explains the 2026 sampler-centric result without treating a controlled HMM as a quantitative prediction for every large language model.

    Independent practice and assessment map: capstone research audit¤

    P1 — Research exercises¤

    Complete the tasks as one reproducible notebook or report. For P1.5, predict the schedule ordering before computing mutual information; treat P1.6 as transfer to a sampler not analyzed in the worked study.

    1. P1.1 — Posterior target. Derive the two-state posterior in Chapter 2 directly by Bayes' rule.
    2. P1.2 — Conditional compatibility. Construct two incompatible binary conditionals and show no joint distribution satisfies both.
    3. P1.3 — Policy intervention. Implement a four-token confidence sampler and compare irreversible and remasking variants.
    4. P1.4 — Generator contract. Verify a CTMC generator numerically by checking off-diagonal signs and row sums.
    5. P1.5 — Predict before computing. Predict the schedule ordering, then compare uniform and absorbing corruption at equal mutual information with \(x_0\).
    6. P1.6 — Transfer to guidance. Evaluate a masked generator with confidence computed before versus after guidance.

    Exercise sketches
    1. Enumerate \(x_1\in\{0,1\}\), compute \(q(x_1\mid x_0)q(x_2\mid x_1)\) for each path, and divide by their sum \(q(x_2\mid x_0)\). For the chapter matrix the weights are \(0.09\) and \(0.08\), yielding \((0.529,0.471)\).
    2. Use the Chapter 1 values: the \(X\) conditionals imply joint cross-product ratio 16, while the \(Y\) conditionals imply 4. A positive \(2\times2\) table cannot have two different cross-product ratios.
    3. Hold model logits and random seeds fixed. The irreversible policy cannot revisit an accepted error; the remasking policy can, but may oscillate. Compare exact output histograms, correction counts, NFE, and final quality.
    4. For every row, check \(R(i,j)\ge0\) when \(i\ne j\) and set or verify \(R(i,i)=-\sum_{j\ne i}R(i,j)\). Then \(R\mathbf1=0\), so the master equation conserves total mass.
    5. Compute \(I(X_0;X_t)\) or conditional entropy under each corruption and solve for times with equal values. Equal numeric \(t\) or equal raw corruption probability need not leave equal information.
    6. Evaluate confidence both on base probabilities and on guided probabilities. Because guidance changes ordering as well as token proposals, record commit sets and output distributions, not only final condition accuracy.
    Final retrieval questions
    1. Name the four objects that must remain distinct from data law to deployed sampler.
    2. Why can an exact denoiser still yield an incorrect few-step sample law?
    3. Which error can an exact-HMM oracle remove, and which errors remain?
    4. What qualifiers must accompany a 2026 parallel-sampling theorem?
    5. What is the minimum complete description of a discrete iterative generator?
    Complete retrieval solutions
    1. Data law, training query/path, learned estimator, and sampler; a deployed system additionally composes representation, conditioning, constraints, and decoding.
    2. A denoiser supplies conditional marginals or ratios, while a coarse factorized kernel may sample dependent coordinates independently or approximate time evolution poorly.
    3. It removes learned-estimator error. Reference mismatch, numerical discretization, parallel factorization, adaptive policy, guidance/projection, and representation effects remain.
    4. State the formal model or corruption, target divergence, estimator-error assumption, dimension/vocabulary dependence, schedule knowledge, revision/CoT condition, and whether the result is iteration complexity rather than measured wall clock.
    5. State space and representation; reference/corruption/path; target and objective; architecture/parameterization; transition or update policy; numerical sampler; conditioning/guidance; constraint handling; termination; likelihood semantics; and matched evaluation protocol.

    Source-tracing checklist¤

    Before carrying a result from a paper into a comparison table, record:

    1. venue status and exact paper version;
    2. data, split, tokenizer/codec, context or grid size, and conditional setting;
    3. whether the reported likelihood is exact, a bound, or an external-model score;
    4. corruption/path, target parameterization, loss weighting, and auxiliary losses;
    5. sampler, schedule, NFE definition, changing-token count, correctors, guidance, and clipping;
    6. parameter and active-parameter counts, training tokens/FLOPs, batch, precision, hardware, and wall clock;
    7. entropy/diversity, memorization, constraint, and human-evaluation controls;
    8. whether the result is an author report, independent reproduction, partial reproduction, or unverified claim.

    For a theorem, replace benchmark fields with the state space, target metric, error model, initialization, regularity/support assumptions, hidden constants, and oracle access. A theorem about sequential query complexity and an experiment about accelerator latency belong in adjacent columns, not one undifferentiated “speed” cell.

    Final Synthesis¤

    Discrete iterative generation replaces a fixed left-to-right order with a learned path through corrupted or partially observed states. Masked refinement supplies flexible bidirectional conditionals; D3PMs supply explicit discrete Markov chains and ELBOs; CTMC models supply reverse jump rates from density ratios; discrete flow matching supplies a general probability-path view. Parallelism comes from changing many coordinates per round, but it introduces within-step independence and discretization choices. A complete scientific description must name the state space, corruption/path, target, transition law, sampler, and likelihood semantics.
