    2. Dynamic Programming, Conditioning, and Sampling¤

    Chapter 2 prerequisites and learning objectives¤

    This chapter assumes Chapter 1 and basic dynamic programming. By the end, you should be able to:

    1. O1 — Derive inside inference: construct the lexical base cases and the binary span recurrence for a finite PCFG chart.
    2. O2 — Use outside mass: compute posterior span or rule events and explain why their marginals are not independent tree decisions.
    3. O3 — Select the query algebra: distinguish recognition, marginalization, Viterbi decoding, stable log computation, and expectation calculations.
    4. O4 — Sample the declared law: use chart masses to sample posterior trees and distinguish that procedure from ancestral or best-tree generation.
    5. O5 — Condition globally: augment states for length or formal constraints and state when rejection, masking, and product construction are exact.
    6. O6 — Connect charts to circuits: identify the finite compilation and transfer the recurrence to a new structured query without erasing cycles or complexity assumptions.

    Inside probabilities¤

    Let \(x=x_0\ldots x_{n-1}\). For a binary PCFG, define

    \[ \alpha[A,i,j] =p(A\Rightarrow^*x_i\ldots x_{j-1}). \]

    Lexical rules initialize spans of length one:

    \[ \alpha[A,i,i+1]=\theta_{A\to x_i}. \]

    For a longer span,

    \[ \boxed{ \alpha[A,i,j] =\sum_{A\to BC}\theta_{A\to BC} \sum_{k=i+1}^{j-1} \alpha[B,i,k]\alpha[C,k,j]. } \]

    The marginal likelihood is \(p(x)=\alpha[S,0,n]\). A dense implementation has cubic dependence on sequence length; its dependence on the number of symbols and rules depends on rule sparsity and factorization. Unary and nullable cycles require closure or fixed-point treatment rather than insertion into this acyclic recurrence.

    Outside probabilities and posterior events¤

    The outside value \(\beta[A,i,j]\) collects the mass of every derivation context from the start symbol down to an occurrence of \(A\) spanning \(i{:}j\), excluding the subtree below that occurrence. Inside and outside values give posterior span marginals:

    \[ p(A\text{ spans }i{:}j\mid x) =\frac{\alpha[A,i,j]\beta[A,i,j]}{p(x)}. \]

    Combining the parent outside value, a rule probability, and the sibling inside value gives expected rule-use counts. These expectations power EM, uncertainty visualization, and posterior sampling. They are not independent probabilities: one tree contains mutually constrained spans and rules.

    The classical inside–outside estimator is dynamic-programming EM for PCFGs, not a different model family.

    One recurrence, several algebras¤

    Replacing the chart's addition and multiplication operations changes the query:

    Table GRAM.3. One recurrence, several algebras
    Algebra Chart result
    Boolean OR/AND recognition
    sum/product marginal probability
    max/product with backpointers Viterbi derivation
    log-sum-exp/add stable log probability
    expectation semiring probability plus sufficient statistics
    k-best or forest algebra packed alternatives

    Semiring parsing formalizes this reuse. It does not mean every algebra supports every optimization or that numerical properties are identical.

    Complete worked example: inside versus Viterbi¤

    Return to \(S\to SS\) with probability \(1/3\) and \(S\to a\) with probability \(2/3\). For aaa, the lexical cells are \(2/3\). A length-two cell is

    \[ \alpha[S,0,2]=\frac13\frac23\frac23=\frac4{27}. \]

    The length-three cell has two split points:

    \[ \begin{aligned} \alpha[S,0,3] &=\frac13\left[ \alpha[S,0,1]\alpha[S,1,3] +\alpha[S,0,2]\alpha[S,2,3] \right]\\ &=\frac13\left[ \frac23\frac4{27}+\frac4{27}\frac23 \right] =\frac{16}{243}. \end{aligned} \]

    A max-product chart returns one tree with mass \(8/243\). The sum-product chart returns the string probability \(16/243\). Both are correct answers to different queries.

    Sampling trees, strings, and posterior parses¤

    Unconditional ancestral sampling expands nonterminals according to rule probabilities. It is exact only when the grammar is consistent and the expansion policy does not alter rule probabilities. To sample a parse from \(p(t\mid x)\), start from the root chart item and recursively sample a rule and split with probability proportional to

    \[ \theta_{A\to BC}\alpha[B,i,k]\alpha[C,k,j]. \]

    To condition on length \(n\), build an inside chart indexed by generated length and sample rule decompositions from the resulting conditional masses. Rejecting unwanted lengths is exact in principle but can be arbitrarily inefficient; hard truncation without renormalization is not exact conditioning.

    Constraints by product construction¤

    Suppose a grammar-generated string must also belong to a regular language recognized by a finite automaton. Chart states can be augmented with automaton start and end states. The resulting product dynamic program computes

    \[ P_G(X\in L) \]

    and can sample from \(p_G(x\mid x\in L)\) when this probability is positive. This is global conditioning because the completion mass is retained. It differs from merely masking locally invalid next tokens. More general grammar intersections can be expensive or fall outside context-free closure.

    Work on SCFG probabilities of regular languages studies the numerical fixed-point problem and its assumptions.

    Packed forests and circuit compilation¤

    For a fixed observation, the chart is a directed acyclic sum-product graph: alternative rules and split points are sum nodes, while compatible child items are products. This is an algebraic equivalence to a probabilistic circuit for that instantiated parse forest. The grammar remains a recursive reusable specification across all observations; the chart is one finite compilation.

    Independent practice and assessment map: dynamic programming¤

    Attempt each task before opening its hints. The typical time is a planning aid, not a speed target. Use the success criterion to decide whether to revise before reading the complete solution.

    1. P1.1 — Predict before computing. Type: predict · Difficulty: foundation · Typical time: 8 min. For aaaa under \(S\to SS\) with probability \(1/3\) and \(S\to a\) with probability \(2/3\), predict whether the sum-product chart exceeds the Viterbi chart. Compute both root values.
    2. P1.2 — Recover a posterior rule event. Type: derive · Difficulty: core · Typical time: 10 min. A string ab has two root derivations, \(S\to AB\) with joint mass \(0.36\) and \(S\to BA\) with joint mass \(0.01\). Compute \(p(x)\) and the posterior probability of each root rule. Explain what an outside computation contributes in a larger tree.
    3. P1.3 — Stabilize a chart sum. Type: derive · Difficulty: core · Typical time: 10 min. Compute \(\log(0.36+0.01)\) using \(m+\log[\exp(\log0.36-m)+\exp(\log0.01-m)]\) with \(m=\log0.36\). Compare with \(\log0.37\).
    4. P1.4 — Sample rather than decode. Type: design · Difficulty: core · Typical time: 10 min. The aaa chart has two root split alternatives, each with contribution \(8/243\). Give their posterior sampling probabilities and a correct recursive sampling procedure. Contrast it with a deterministic backpointer.
    5. P1.5 — Condition on a formal event. Type: derive · Difficulty: core · Typical time: 12 min. A finite generator assigns masses \(0.45,0.05,0.25,0.25\) to aa, ab, ba, bb. A product construction admits \(L=\{\texttt{aa},\texttt{ba},\texttt{bb}\}\). Compute \(P(X\in L)\) and the exact conditional law.
    6. P1.6 — Transfer. Type: design · Difficulty: stretch · Typical time: 18 min. Augment a PCFG chart with a two-state DFA that accepts strings containing an even number of b symbols. Specify the augmented item, lexical transition, binary composition, accepting root states, and the condition required for exact conditional sampling.

    Task Evidence of success
    P1.1 One-tree mass, Catalan multiplicity, max, and sum are computed without conflation.

    Complete solutions to P1
    1. P1.1. One ordered binary tree has mass \((1/3)^3(2/3)^4=16/2187\). The Viterbi chart therefore returns \(16/2187\). The sum-product chart adds the \(C_3=5\) bracketings and returns \(80/2187\approx0.03658\), which is strictly larger. Check. Both charts use the same local decompositions; only the aggregation algebra changes.
    2. P1.2. The marginal is \(p(\texttt{ab})=0.37\). The posterior root probabilities are \(0.36/0.37=36/37\) and \(0.01/0.37=1/37\). In a larger tree, outside mass supplies every compatible context above and beside the named item so that its inside mass can become a full-tree posterior event. Check. The two root events are mutually exclusive and sum to one here.
    3. P1.3. With \(m=\log0.36\), the expression is \(\log0.36+\log(1+1/36)=\log0.37\approx-0.994252\). Check. Algebraic equality to the direct result is exact; the advantage is numerical stability for much smaller chart entries.
    4. P1.4. Each root split has probability \((8/243)/(16/243)=1/2\). Sample one split from these normalized contributions, then recursively sample each selected child rule and split in the same way until reaching lexical items. A deterministic Viterbi backpointer chooses one maximum alternative and therefore does not sample \(p(t\mid x)\). Check. Repeated samples should approach a half-half root split frequency.
    5. P1.5. The accepted event has mass \(0.45+0.25+0.25=0.95\). The conditional law is \(p(\texttt{aa}\mid L)=9/19\) and \(p(\texttt{ba}\mid L)=p(\texttt{bb}\mid L)=5/19\). Check. The three values sum to one and retain the original \(0.45{:}0.25{:}0.25\) ratio.
    6. P1.6. Use \([A,i,j,s,t]\) for mass of an \(A\)-yield spanning \(i{:}j\) that takes DFA state \(s\) to \(t\). A lexical a keeps the parity state and b toggles it. For \(A\to BC\), sum over split \(k\) and intermediate DFA state \(u\) the product of \([B,i,k,s,u]\) and \([C,k,j,u,t]\). Start and final parity must both be even. Exact conditional sampling requires positive accepted mass and backward choices proportional to these complete product-chart masses. Rubric. Full credit states all five structural pieces and does not replace completion mass with local viability alone.

    Chapter 2 failure cases and misconception checks¤

    • Underflow: multiply many small probabilities in log space or use scaled charts; a zero caused by floating-point underflow is not grammatical impossibility.
    • Viterbi substitution: a best tree is not a likelihood or a posterior sample.
    • Cycle blindness: nullable and unary cycles can create infinitely many derivations for one yield and require explicit closure conditions.
    • Complexity shorthand: “CKY is cubic” describes sequence-length scaling for a particular normal form; symbol, rule, batch, and hardware costs still matter.
    Retrieval question: what do outside values add?

    Inside values summarize possible subtrees. Outside values summarize the compatible contexts around a subtree. Their product, normalized by the sentence likelihood, gives posterior events and expected rule counts.

    Chapter 2 synthesis¤

    Dynamic programming preserves the sum over ambiguous derivations while sharing repeated subproblems. Changing the algebra changes the query; adding product states changes the conditioning event. Chapter 3 uses posterior rule counts and neural parameterizations to learn grammar distributions from trees or yields.
