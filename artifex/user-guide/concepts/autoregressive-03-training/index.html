    Training Autoregressive Models¤

    Training Autoregressive Models: Prerequisites and learning objectives¤

    You should understand log-likelihood and the shifted prediction convention. By the end of this chapter, you should be able to:

    1. derive token cross-entropy from maximum likelihood;
    2. implement the input/target shift without leaking the answer;
    3. distinguish sequence NLL, mean token NLL, perplexity, and bits per dimension;
    4. mask padding and report the normalization denominator; and
    5. explain what teacher forcing, label smoothing, scheduled sampling, and gradient clipping do—and do not—optimize.

    Maximum Likelihood Training¤

    Autoregressive models are trained via maximum likelihood estimation using teacher forcing:

    Teacher Forcing: During training, use ground truth previous tokens as input (not model's own predictions).

    Training loop:

    def train_step(model, batch, optimizer):
    # batch['sequences']: [batch_size, seq_len] ground truth sequences

    def loss_fn(model):
        # Forward pass with ground truth input
        outputs = model(batch['sequences'])
        logits = outputs['logits']  # [batch_size, seq_len, vocab_size]

        # Shift targets: predict x_i given x_<i
        shifted_logits = logits[:, :-1, :]  # Remove last position
        shifted_targets = batch['sequences'][:, 1:]  # Remove first position
        vocab_size = shifted_logits.shape[-1]

        # Cross-entropy loss
        log_probs = nnx.log_softmax(shifted_logits, axis=-1)
        one_hot_targets = nnx.one_hot(shifted_targets, vocab_size)
        loss = -jnp.mean(jnp.sum(log_probs * one_hot_targets, axis=-1))

        return loss

    loss, grads = nnx.value_and_grad(loss_fn)(model)
    optimizer.update(model, grads)  # NNX 0.11.0+ API

    return loss

    Why teacher forcing?

    • Stable training: Prevents error accumulation from model's mistakes
    • Faster convergence: Model sees correct context
    • Direct pathwise differentiation of the log-likelihood: no sampling estimator is required for ordinary observed discrete targets

    Train–generation context mismatch: at generation time, the model conditions on prefixes sampled from its own distribution rather than only data prefixes. It is often called exposure bias, but that label does not identify a unique error or imply that teacher-forced MLE is statistically inconsistent. Candidate interventions include:

    • Scheduled sampling: Gradually mix model predictions into training prefixes. The resulting objective is generally not maximum likelihood and can be statistically inconsistent; compare it with sequence-level training and stronger MLE baselines rather than treating it as a default fix (Huszár, 2015).
    • Sequence-level or on-policy objectives: Optimise a declared sequence metric or reward, accepting the estimator variance, exploration, and alignment trade-offs.
    • Robust MLE and data coverage: Better models and broader prefix coverage can reduce practical errors without changing the population MLE objective, but scale is not a guarantee against compounding mistakes.

    Loss Functions and Metrics¤

    Primary loss: Negative log-likelihood (NLL) / Cross-entropy:

    \[ \mathcal{L}_{\text{NLL}} = -\frac{1}{N \cdot T} \sum_{n=1}^{N} \sum_{t=1}^{T} \log p_\theta(x_t^{(n)} \mid x_{<t}^{(n)}) \]

    Perplexity: Exponentiated cross-entropy (lower is better):

    \[ \text{PPL} = \exp(\mathcal{L}_{\text{NLL}}) \]

    Bits per dimension (BPD): for an ordered observation with \(D\) scalar dimensions,

    \[ \text{BPD} =-\frac{1}{N D\log 2} \sum_{n=1}^{N}\sum_{i=1}^{D} \log p_\theta(x_i^{(n)}\mid x_{<i}^{(n)}). \]

    If \(\mathcal L_{\mathrm{NLL}}\) has already been averaged over the \(D\) dimensions in nats, then \(\mathrm{BPD}=\mathcal L_{\mathrm{NLL}}/\log 2\); do not divide by \(D\) twice. Discrete-image comparisons must also state preprocessing/dequantisation and the unit of each modeled symbol.

    Accuracy: Token-level prediction accuracy (for discrete data):

    \[ \text{Acc} = \frac{1}{N \cdot T} \sum_{n=1}^{N} \sum_{t=1}^{T} \mathbb{1}[\arg\max p_\theta(x_t \mid x_{<t}) = x_t] \]

    Token accuracy discards the probability assigned to every non-argmax outcome. A model can therefore improve NLL and calibration without changing accuracy, or preserve accuracy while becoming severely overconfident. For variable-length batches, all sums and denominators must exclude padding tokens.

    Worked example: from conditionals to NLL and perplexity¤

    Suppose a three-target sequence receives probabilities \(0.8\), \(0.5\), and \(0.25\) on its observed next tokens. Its sequence probability under the model is

    \[ p_\theta(x_{1:3})=0.8\times0.5\times0.25=0.1. \]

    The sequence NLL is \(-\log 0.1\approx2.303\) nats. The mean token NLL is \(2.303/3\approx0.768\) nats, so token-level perplexity is

    \[ \exp(0.768)\approx2.15. \]

    Exponentiating the sequence NLL would instead give 10. Both numbers are mathematically meaningful but answer different questions; a reported “perplexity” conventionally exponentiates mean token NLL. Comparisons also require a compatible tokenizer because changing the unit called a token changes the denominator.

    Numerical Stability and Best Practices¤

    Log-space computation: Always work in log-space to prevent underflow:

    # WRONG: Can underflow
probs = softmax(logits)
loss = -jnp.mean(jnp.log(probs[targets]))

# CORRECT: Numerically stable
log_probs = nnx.log_softmax(logits)
target_log_probs = jnp.take_along_axis(
    log_probs, targets[..., None], axis=-1
).squeeze(-1)
loss = -jnp.mean(target_log_probs)

    Gradient clipping: Prevent exploding gradients in deep models:

    # Clip the norm of the complete gradient PyTree, preserving its direction.
clipper = optax.clip_by_global_norm(clip_value)
clip_state = clipper.init(grads)
grads, _ = clipper.update(grads, clip_state)

    Element-wise jnp.clip is a different operation: it clips every coordinate independently and can rotate the gradient substantially. Global-norm clipping rescales the whole PyTree only when its joint norm exceeds the threshold.

    Learning rate schedules: Use warmup + decay for Transformers:

    def lr_schedule(step, warmup_steps=4000, d_model=512):
    step = jnp.maximum(step, 1)  # Avoid division by zero
    arg1 = step ** -0.5
    arg2 = step * (warmup_steps ** -1.5)
    return (d_model ** -0.5) * jnp.minimum(arg1, arg2)

    Label smoothing: Reduce overconfidence:

    def label_smoothing(one_hot_labels, smoothing=0.1):
    num_classes = one_hot_labels.shape[-1]
    smooth_labels = one_hot_labels * (1 - smoothing)
    smooth_labels += smoothing / num_classes
    return smooth_labels

    Label smoothing replaces the empirical one-hot target with a mixture distribution. It is therefore not exact MLE for the original empirical distribution. It can regularize training, but its effect on probability calibration, rare tokens, and downstream decoding must be measured rather than inferred from the name.

    Independent practice and assessment map: training¤

    Perform every reduction from unreduced token losses. This makes padding, tokenization, and averaging choices visible instead of hiding them inside a single library scalar.

    P1 — From shifted targets to a comparable likelihood report¤

    Two unpadded training sequences are \((\mathrm{BOS},A,B,\mathrm{EOS})\) and \((\mathrm{BOS},A,A,\mathrm{EOS})\). The model assigns the realized targets probabilities \((0.50,0.25,0.80)\) and \((0.50,0.50,0.80)\), respectively. A batched tensor adds two padding positions after each EOS.

    1. P1.1 — Predict before computing. Predict whether including the four padding targets with probability \(0.99\) raises or lowers the reported mean NLL and perplexity. Explain why either change is a metric bug rather than improved language modeling.
    2. P1.2 — Finite calculation. Calculate each sequence NLL, the corpus token NLL over the six real targets, and perplexity. Then compute the incorrect values after including padding and state both denominators explicitly.
    3. P1.3 — Debugging task. Write the shapes and slices for inputs, targets, logits, and a Boolean target mask. Give one test that catches a one-position shift and one that catches an incorrect per-sequence-then-batch averaging order.
    4. P1.4 — Objective boundary. For teacher forcing, label smoothing, and scheduled sampling, state the distribution supplying the context and the distribution supplying the target. Identify which procedure still optimizes ordinary empirical-data MLE.
    5. P1.5 — Transfer. An image model reports mean NLL in nats per quantized channel. Derive bits per dimension from that value, then explain what changes if the reported NLL was instead summed over all channels and pixels.

    Chapter 3 synthesis¤

    Teacher-forced MLE converts joint modeling into supervised conditional prediction while retaining the exact model likelihood of each complete ordered example. Correct normalization is part of the metric definition: padding, tokenizer, dimensional unit, and averaging order all matter. Exposure bias describes a context-distribution difference, not a proof that MLE is inconsistent; alternative objectives introduce their own target and estimator. Numerical safeguards should preserve the intended objective—stable log-softmax and global-norm clipping do, whereas an accidental target shift changes the problem entirely.

    Retrieval practice: training
    1. If mean token NLL is \(\log 4\), what is perplexity?
    2. Why is perplexity across two tokenizers not automatically comparable?
    3. Does scheduled sampling still optimize the ordinary teacher-forced log-likelihood?
    4. How does global-norm clipping differ from coordinate-wise clipping?

    Solutions. (1) \(\exp(\log4)=4\). (2) The token unit and number of prediction events differ. (3) Generally no; it trains on a mixture of data and model prefixes. (4) Global clipping rescales the complete gradient PyTree by one factor when its joint norm is too large, preserving direction; coordinate clipping truncates entries separately and can rotate it.

    Bridge to Chapter 4. Training defines a conditional distribution, but a deployed system must turn it into an output. Decoding can preserve that distribution, truncate and renormalize it, or search for a high-scoring sequence—three distinct operations with distinct guarantees.
