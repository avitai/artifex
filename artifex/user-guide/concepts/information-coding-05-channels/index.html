    5. Noisy Channels, Bayesian Decoding, and Generative Denoising¤

    Chapter 5 prerequisites and learning objectives¤

    You should understand conditional probability, mutual information, and source coding. By the end of this chapter, you should be able to:

    • define a channel separately from a source and an error-correcting code;
    • compute the capacity of a binary symmetric channel;
    • perform exact Bayesian decoding for a finite repetition code;
    • use Fano's inequality and finite-blocklength qualifications correctly; and
    • explain the limited but important connection to denoising generative models.

    A channel is a conditional law¤

    A communication channel is a probability kernel \(W(y\mid x)\) from transmitted symbol \(X\) to observation \(Y\). The message prior, encoder, channel, and decoder are separate components:

    Source coding removes statistical redundancy to reduce rate. Channel coding deliberately adds structured redundancy to resist corruption. A deployed system may use both, but their objectives point in opposite directions and their rates have different denominators.

    Capacity maximizes reliable information rate¤

    For a discrete memoryless channel,

    \[ C=\max_{p(x)}I(X;Y) \]

    is the asymptotic capacity in bits per channel use when base-two logarithms are used. The maximization is over allowed input distributions and may include cost constraints. Shannon's coding theorem relates rates below capacity to codes whose error probability can vanish with increasing blocklength, and rates above capacity to a converse under its stated error criterion.

    For a binary symmetric channel with flip probability \(\varepsilon\),

    \[ W(y\mid x)= \begin{cases} 1-\varepsilon,&y=x,\\ \varepsilon,&y\neq x, \end{cases} \]

    the uniform input achieves

    \[ C=1-h_2(\varepsilon) \]

    for \(0\leq\varepsilon\leq1/2\). At \(\varepsilon=1/4\), capacity is approximately \(0.1887\) bit per use. This is the same mutual-information value computed in Chapter 2, now maximized over input laws and interpreted as a reliable communication limit.

    Worked repetition-code posterior¤

    Encode a fair message bit \(M\) as \(X^3=(M,M,M)\) and transmit each bit through an independent binary symmetric channel with \(\varepsilon=0.1\). Suppose \(Y^3=(1,0,1)\). Then

    \[ \begin{aligned} p(y\mid M=1)&=0.9^2(0.1)=0.081,\\ p(y\mid M=0)&=0.1^2(0.9)=0.009. \end{aligned} \]

    With equal message priors,

    \[ p(M=1\mid y)=\frac{0.081}{0.081+0.009}=0.9. \]

    Majority vote is the MAP decoder here because the prior is uniform and bit flips are symmetric and independent. Change the prior, introduce burst errors, or make coordinates unequally reliable, and majority vote need not be optimal. The generative channel likelihood makes those assumptions visible.

    The repetition code transmits one message bit using three channel uses, so its nominal rate is \(1/3\). Redundancy improves error probability but does not guarantee correction of every corruption; all three bits can flip.

    Fano bounds error through remaining uncertainty¤

    Let \(M\) take values in an alphabet of size \(K\), let \(\hat M=g(Y)\), and let \(P_e=P(\hat M\neq M)\). Fano's inequality gives

    \[ \begin{aligned} H(M\mid Y) &\leq h_2(P_e)\\ &\quad+P_e\log_2(K-1). \end{aligned} \]

    It converts a decoding-error statement into a conditional-entropy bound. Rearranged with \(I(M;Y)=H(M)-H(M\mid Y)\), it can prove lower bounds on error when observations carry too little information. The alphabet size, message prior, logarithm base, and estimator class must be stated. Fano is generally not an exact formula for the Bayes error.

    Asymptotic capacity does not predict one finite system¤

    At blocklength \(n\), target error probability \(\epsilon\), and a declared channel and cost, the maximal coding rate is a finite-blocklength object. It is below capacity by a term related to channel dispersion in common regular settings, with further lower-order terms. Polyanskiy, Poor, and Verdú (2010) develop achievability and converse bounds for this regime.

    Therefore a capacity number alone does not determine latency, code construction, decoding compute, burst-error robustness, or performance at a chosen \(n\). Reports must attach blocklength, rate, error criterion, channel law, input constraint, and decoder.

    Generative denoising is Bayesian decoding—but not always channel coding¤

    Suppose a clean object \(X\) has prior \(p_\theta(x)\) and a corruption process produces \(Y\sim c(y\mid x)\). Posterior denoising targets

    \[ p_\theta(x\mid y) \propto c(y\mid x)p_\theta(x). \]

    This is the same Bayesian structure as channel decoding. It appears in inverse problems, error correction, masked reconstruction, and diffusion-style corruption. But the operational labels must remain accurate:

    • if no message is encoded and no communication rate is measured, the task is denoising or posterior inference, not automatically channel coding;
    • if the prior is learned from data, posterior quality depends on both prior misspecification and corruption calibration;
    • if a decoder returns one plausible sample, it need not minimize symbol error, sequence error, squared distortion, or another decision loss; and
    • a training corruption schedule need not model the physical observation channel used at deployment.

    Discrete diffusion and masked generation can be interpreted through time-indexed channels \(q_t(x_t\mid x_0)\). Mutual information \(I(X_0;X_t)\) measures retained dependence under a declared data law, but matching that scalar across two corruptions does not make their reverse conditionals, computational costs, or finite-step samplers equivalent.

    Joint source–channel and semantic systems need a declared target¤

    Classical separation theorems justify independent source and channel code design only under asymptotic assumptions. Finite latency, task-specific distortion, unequal error protection, learned representations, and semantic objectives can motivate joint systems. Such a system must still report what is transmitted, which channel is assumed, what the receiver reconstructs or decides, and what errors matter. “Semantic communication” is not permission to replace fidelity with an unspecified realism score.

    Chapter 5 synthesis and retrieval¤

    A channel is a corruption kernel, capacity is an optimized asymptotic rate, and an error-correcting code is a finite construction. Bayesian decoding combines a message prior with the channel likelihood. The same posterior form supports generative denoising, but a denoiser becomes a communication system only when messages, encoding, rate, channel, and error criteria are operationally present.

    Retrieval questions
    1. Why does source coding remove redundancy while channel coding adds it?
    2. What is the capacity of a noiseless binary channel?
    3. When is majority vote the MAP decoder for a repetition code?
    4. Why is a diffusion denoiser not automatically an error-correcting code?

    Solutions. (1) Source coding reduces expected representation length; channel coding spends extra channel uses to make messages distinguishable after noise. (2) One bit per channel use. (3) With a uniform message prior and conditionally independent, symmetric bit flips of equal reliability. (4) It may have no encoded message, rate, physical channel, exact recovery goal, or declared communication error criterion.

    Bridge. Chapters 1–5 measure information in laws and protocols. Chapter 6 uses those measures to choose representations and models, then draws a firm boundary around description-length ideas that are not computable observables.
