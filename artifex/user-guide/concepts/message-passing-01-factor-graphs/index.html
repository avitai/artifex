    1. From a Generative Factorization to an Inference Query¤

    Chapter 1 prerequisites and learning objectives¤

    You should know joint, marginal, and conditional probability and the distinction between normalized and unnormalized models. By the end of this chapter, you should be able to:

    • translate a product of factors into a bipartite factor graph;
    • distinguish a directed generative graph, factor graph, plate diagram, computation graph, and message-passing neural network;
    • name the exact query before selecting an inference algorithm;
    • enumerate one finite model to obtain its normalizer and marginals; and
    • explain why small local factors do not guarantee cheap global inference.

    The probabilistic object comes before the graph¤

    Let each factor \(f_a\) depend only on the subtuple \(x_a\). The nonnegative function

    \[ \gamma(x)=\prod_{a\in\mathcal F}f_a(x_a) \]

    defines a probability law only when

    \[ 0<Z=\sum_x\gamma(x)<\infty, \qquad p(x)=\frac{\gamma(x)}{Z}. \]

    A factor graph is a bipartite representation of this product: variable nodes connect to factor nodes whose scopes contain them. It records which arguments each local function uses. It does not, by itself, normalize the product, select an inference algorithm, or assert that the graph is sparse enough for tractable exact computation.

    Directed and undirected generative models both admit factor-graph representations. A Bayesian network contributes one normalized conditional factor per node. A Markov random field contributes nonnegative potentials and usually a global partition function. After evidence is fixed, either representation becomes a product of factors over the remaining unknown variables.

    The running three-variable model¤

    Let \(A,B,C\in\{0,1\}\). Define unary factors

    \[ u_A=(2,1), \qquad u_C=(1,4), \]

    and pair factors, with rows indexing the left variable and columns the right variable,

    \[ f_{AB}= \begin{bmatrix} 3&1\\ 1&3 \end{bmatrix}, \qquad f_{BC}= \begin{bmatrix} 2&1\\ 1&2 \end{bmatrix}. \]

    The unnormalized joint mass is

    \[ \gamma(a,b,c)=u_A(a)f_{AB}(a,b)f_{BC}(b,c)u_C(c). \]

    Worked example: exact enumeration¤

    For three binary variables there are only eight configurations:

    Table MSG.2. Worked example: exact enumeration
    \(a\) \(b\) \(c\) \(\gamma(a,b,c)\)
    0 0 0 \(2\cdot3\cdot2\cdot1=12\)
    0 0 1 \(2\cdot3\cdot1\cdot4=24\)
    0 1 0 \(2\cdot1\cdot1\cdot1=2\)
    0 1 1 \(2\cdot1\cdot2\cdot4=16\)
    1 0 0 \(1\cdot1\cdot2\cdot1=2\)
    1 0 1 \(1\cdot1\cdot1\cdot4=4\)
    1 1 0 \(1\cdot3\cdot1\cdot1=3\)
    1 1 1 \(1\cdot3\cdot2\cdot4=24\)

    Therefore

    \[ Z=12+24+2+16+2+4+3+24=87. \]

    Collecting rows by one variable gives

    \[ p(A)=\frac1{87}(54,33) =\left(\frac{18}{29},\frac{11}{29}\right), \]
    \[ p(B)=\frac1{87}(42,45) =\left(\frac{14}{29},\frac{15}{29}\right), \qquad p(C)=\frac1{87}(19,68). \]

    Enumeration is the reference computation against which later algorithms can be checked. Its cost grows as \(2^n\) for \(n\) unrestricted binary variables. Factorization becomes useful only when distributivity avoids repeating most of that work.

    One model, several queries¤

    Table MSG.3. One model, several queries
    Query Mathematical object What an answer must contain
    normalizer or evidence \(Z=\sum_x\gamma(x)\) one nonnegative scalar and its numerical accuracy
    marginal \(p(x_Q)=Z^{-1}\sum_{x_{\setminus Q}}\gamma(x)\) a normalized law over the query variables
    conditional \(p(x_Q\mid x_E=e)\) evidence support, a marginal numerator, and nonzero evidence probability
    MPE \(\arg\max_x\gamma(x)\) one or all globally maximizing complete assignments
    marginal MAP \(\arg\max_{x_Q}\sum_{x_R}\gamma(x_Q,x_R,e)\) the order of sum and max operations; they generally do not commute
    exact sample \(X\sim p\) a procedure whose output law is \(p\), not only a high-scoring assignment

    Evidence can be represented by an indicator factor, such as \(I(B=1)\), or by restricting the relevant factor tables. A posterior is not created merely by shading a node in a diagram; conditioning changes the function being summed or normalized.

    Plate notation shows repetition, not computation¤

    Plate notation is valuable when a generative story repeats local variables. In a latent model with global parameter \(\theta\) and \(N\) exchangeable observations,

    \[ p(\theta,z_{1:N},x_{1:N}) =p(\theta)\prod_{n=1}^{N}p(z_n\mid\theta)p(x_n\mid z_n,\theta). \]

    The plate compresses repeated model structure. It does not say whether the posterior over \(\theta\) is conjugate, whether the \(z_n\) can be eliminated independently after conditioning on \(\theta\), or whether an amortized encoder will be used. A factor graph derived after fixing observations makes those inference dependencies explicit.

    Five graphs that answer different questions¤

    Table MSG.4. Five graphs that answer different questions
    Representation Nodes and edges mean What it does not imply
    directed graphical model local normalized conditionals and conditional-independence structure cheap arbitrary posterior inference
    factor graph variables and local factors in one product acyclicity, normalization, or convergence of loopy messages
    plate diagram repeated variables and shared parameters an executable elimination or message schedule
    computation graph primitive numerical operations for one implementation a probabilistic interpretation of every intermediate value
    GNN or MPNN graph learned state exchange under an architecture calibrated beliefs or exact inference in a declared probability law

    This volume assumes that the factor scopes to be queried are already given. Selecting those scopes from repeated multivariate observations is a different model-selection problem developed in Learning Dependency Graphs from Data. Learning a graph and answering a marginal on the fitted graph can be composed, but neither operation performs the other.

    Counterexample: tiny factors can create a hard sum

    A grid of binary variables may use only pairwise \(2\times2\) factors, yet eliminating variables across the grid creates factors whose scopes grow with the grid width. Local table size and global inference complexity are different quantities.

    Chapter 1 synthesis¤

    The product \(\gamma=\prod_a f_a\) defines the object to be queried; a factor graph exposes its local scopes. Enumeration supplies an exact baseline but scales exponentially. Chapter 2 uses distributivity to eliminate variables without repeatedly evaluating every complete configuration.

    Retrieval practice: model, graph, and query
    1. Does drawing a factor graph prove that its product is normalized?
    2. What does a plate communicate that a factor graph need not?
    3. In the running example, what are \(Z\) and \(p(B=1)\)?
    4. Why must marginal MAP state the order of summation and maximization?

    Solutions. (1) No; normalization requires a finite positive \(Z\) or a construction that proves \(Z=1\). (2) Repetition and parameter sharing across indexed variables. (3) \(Z=87\) and \(p(B=1)=45/87=15/29\). (4) Sum and max generally do not commute, so changing their order changes the query.
