    A First Course in Generative Modeling¤

    Learning path

    Required knowledge

    • First Course 1 — Generative Systems and Queries

    Readiness check: can you explain the central object and reproduce one worked example from each required chapter? If not, follow its link before continuing.

    Newly unlocked in the default route

    No immediate successor names this page as its final required prerequisite. Continue through the book navigation or the chapter's cross-links.

    This first course is the short on-ramp for a reader who knows basic Python but has not studied generative models. It does not ask you to complete the full mathematical foundation before meeting a model family. Instead, one finite toy world carries the main ideas from questions to probability, learning, inference, sampling, family comparison, and executable code. Each chapter ends with retrieval and an independently attempted problem. The final diagnostic routes you into the full book according to the particular ideas you need next.

    How to use the first course

    Read the worked example with a pencil or scratch file open. Attempt each Your turn prompt before opening its solution. Say the chapter synthesis in your own words without looking. If that is not yet possible, revisit the worked example rather than merely rereading the summary.

    The shared toy world¤

    Every chapter returns to four kinds of tiles. A tile has two binary attributes:

    • \(X_1=0\) means blue and \(X_1=1\) means orange;
    • \(X_2=0\) means circle and \(X_2=1\) means triangle.

    Twenty observed tiles have counts \(9,1,3,7\) for states \(00,01,10,11\). Their empirical probability mass function is therefore

    \[ \widehat p(00)=0.45,\qquad \widehat p(01)=0.05,\qquad \widehat p(10)=0.15,\qquad \widehat p(11)=0.35. \]

    The world is deliberately small enough that every probability can be checked by hand. Neural generative models address vastly larger objects, but the roles of model, objective, estimator, optimizer, inference procedure, and sampler do not change merely because a neural network replaces this table.

    1. What a Generative System Must Specify¤

    Prerequisites and learning objectives for Chapter 1¤

    No model-family knowledge is assumed. You need only recognize a Python variable and a fraction. After this chapter, you should be able to:

    1. name the object a generator produces and the questions it must answer;
    2. distinguish a model from an objective, estimator, optimizer, and sampler;
    3. tell generation, scoring, completion, and latent inference apart; and
    4. audit an informal system description for missing roles.

    Begin with the query, not the architecture¤

    Suppose someone says, “I built a Transformer generator.” That sentence names an architecture, but it does not yet specify a generative system. What is the generated object: text, an image, a graph, a molecule, or a trajectory? Is the model a normalized probability law, an unnormalized energy, a sampler with no tractable density, or a constrained constructor? Which variables are observed, which are latent, and what is conditioned on? How is a sample produced after training?

    The same architecture can support different probability objects and training objectives. Conversely, recurrent networks, convolutional networks, and Transformers can all parameterize the same autoregressive factorization. Start with the query because the query determines what counts as a correct answer.

    For an observation \(x\) and optional condition \(c\), common queries are:

    Table FIRST.1. Begin with the query, not the architecture
    Query Plain-language question Mathematical form
    Generate What new object could occur under condition \(c\)? \(X\sim p_\theta(\cdot\mid c)\)
    Score How much mass or density does the model assign to \(x\)? \(p_\theta(x\mid c)\) or \(\log p_\theta(x\mid c)\)
    Complete What could the missing part be after observing the rest? \(X_{\mathrm{miss}}\sim p_\theta(\cdot\mid x_{\mathrm{obs}},c)\)
    Infer Which hidden explanation is plausible for this observation? \(p_\theta(z\mid x,c)\)
    Compare Which model or decision performs better for the intended use? a declared evaluation protocol

    A system need not support every query. A GAN generator can produce samples without exposing a normalized density for arbitrary inputs. An energy model can rank states before its normalizing constant is known. A deterministic grammar can construct valid objects without assigning probabilities. Unsupported queries are limitations to state, not defects to hide.

    Seven roles that should not be collapsed¤

    Table FIRST.2. Seven roles that should not be collapsed
    Role What it specifies Tile-world example
    Model A law, energy, transformation, or constructive rule four masses \(q_\theta(00),\ldots,q_\theta(11)\)
    Objective A population or empirical criterion average negative log-likelihood
    Estimator A finite computation for an objective or gradient the mean loss over the 20 observed tiles
    Optimizer A rule that changes parameters one gradient-descent update
    Inference algorithm A method for a conditional or latent query compute \(q_\theta(X_2\mid X_1=1)\)
    Sampler A method that generates a draw map a uniform random number into one of four intervals
    Evaluation protocol Data, metrics, baselines, and uncertainty held-out log score plus coverage of all four states

    The optimizer is not part of the probability law. The sampler may be exact or approximate even when the model is fixed. A low training objective does not by itself establish that the generated samples are useful, diverse, calibrated, or safe.

    Worked example: four different questions about one table¤

    Use the empirical tile law

    \[ \widehat p=(0.45,0.05,0.15,0.35) \]

    in state order \(00,01,10,11\).

    Generate. Draw one state using those four masses. This requires a sampling algorithm in addition to the table.

    Score. The probability mass of an orange triangle is \(\widehat p(11)=0.35\). Because the outcome space is discrete, this is a probability, not a continuous density value.

    Complete. If the color is orange, then

    \[ \widehat p(X_2=1\mid X_1=1) =\frac{\widehat p(11)}{\widehat p(10)+\widehat p(11)} =\frac{0.35}{0.15+0.35}=0.7. \]

    Infer. If a latent variable \(Z\) represented an unobserved manufacturing mode, inference would ask for \(p_\theta(Z\mid X_1,X_2)\). The current four-cell table contains no \(Z\), so that posterior is not defined until a latent-variable model is specified. An algorithm cannot infer a variable the model does not contain.

    Your turn: audit a vague claim¤

    A report says: “Our convolutional generator minimizes mean squared error and therefore learns the data distribution. At deployment we run ten refinement steps.” Identify at least four questions that must be answered before the claim can be evaluated.

    Check your answer

    A defensible audit asks at least: What object or probability law does the convolutional network parameterize? What are the prediction target and conditioning variables? Why does mean squared error identify the intended population object? How is its finite-sample estimator formed? What do the ten refinement steps simulate, and are they an exact or approximate sampler for the declared model? Which held-out protocol tests distributional fit and task utility? Architecture, loss name, and step count alone do not answer these questions.

    Chapter 1 retrieval practice¤

    Without looking back, complete these prompts:

    1. A model states ________; an objective states ________.
    2. Why can two samplers give different finite-compute results for one model?
    3. Name one generator that can sample without exposing a tractable normalized density.
    4. Why is “Transformer” not a complete description of a generative model?

    Chapter 1 synthesis¤

    A generative system begins with an object and a set of supported queries. The model, objective, estimator, optimizer, inference algorithm, sampler, and evaluation protocol answer different questions. Keeping those roles separate is the organizing invariant for the rest of the book.

    2. Probability Objects and Conditional Generation¤

    Prerequisites and learning objectives for Chapter 2¤

    This chapter assumes the role distinctions from Chapter 1. After completing it, you should be able to:

    1. read a finite joint probability table and verify normalization;
    2. compute marginal and conditional probabilities;
    3. distinguish probability, density, likelihood, and log score; and
    4. implement exact inverse-CDF sampling for a finite law.

    Outcomes, events, variables, and laws¤

    An outcome is one possible result of an experiment. In the tile world, the outcome space is

    \[ \mathcal X=\{00,01,10,11\}. \]

    An event is a set of outcomes, such as “orange,” \(\{10,11\}\). A random variable is a measurable function of an outcome. Here \(X_1\) extracts color and \(X_2\) extracts shape. A probability law assigns total mass one to the outcome space and nonnegative mass to every event.

    The joint table is

    Table FIRST.3. Outcomes, events, variables, and laws
    \(X_2=0\) circle \(X_2=1\) triangle color marginal
    \(X_1=0\) blue \(0.45\) \(0.05\) \(0.50\)
    \(X_1=1\) orange \(0.15\) \(0.35\) \(0.50\)
    shape marginal \(0.60\) \(0.40\) \(1.00\)

    The last row and column are marginals: sum over the variable you do not ask about. For example,

    \[ p(X_2=1)=p(01)+p(11)=0.05+0.35=0.40. \]

    A conditional law restricts to an event and renormalizes. Since orange has mass \(0.50\),

    \[ p(X_2=1\mid X_1=1)=\frac{p(11)}{p(X_1=1)}=0.70. \]

    This division is defined only when the conditioning event has positive probability. In continuous spaces, conditioning at a value requires a regular conditional law rather than the informal ratio of two zero-probability events; the full probability chapters develop that distinction.

    Probability mass, density, and likelihood¤

    For a discrete state, \(p(X=x)\) is probability mass. For a continuous variable, a density \(f(x)\) is defined relative to a reference measure and can exceed one; probabilities are integrals over sets. Thus “the probability of the exact real number \(x\)” is generally zero even when \(f(x)>0\).

    Likelihood holds observed data fixed and views the same model expression as a function of its parameter:

    \[ L(\theta;x)=p_\theta(x). \]

    It is not a probability distribution over \(\theta\) unless a separate Bayesian model supplies a prior and normalizes the posterior. The log score \(\log p_\theta(x)\) is often easier to add across independent observations and turns products into sums.

    Factorization is an equality plus an ordering choice¤

    Every joint law of two discrete variables satisfies the chain rule wherever the conditional is defined:

    \[ p(x_1,x_2)=p(x_1)p(x_2\mid x_1). \]

    For the tile law,

    \[ p(X_1=0)=p(X_1=1)=0.5, \]

    and

    \[ p(X_2=1\mid X_1=0)=0.1, \qquad p(X_2=1\mid X_1=1)=0.7. \]

    Multiplication reconstructs all four cells:

    \[ 0.5(0.9,0.1,0.3,0.7)=(0.45,0.05,0.15,0.35), \]

    where the pairs correspond to the two colors. An autoregressive model scales this idea to many ordered components and uses a shared architecture to predict each conditional.

    Worked example: exact categorical sampling¤

    Partition the unit interval using cumulative masses:

    Table FIRST.4. Worked example: exact categorical sampling
    Uniform draw \(u\) Generated state
    \(0\le u<0.45\) \(00\)
    \(0.45\le u<0.50\) \(01\)
    \(0.50\le u<0.65\) \(10\)
    \(0.65\le u<1\) \(11\)

    If \(U\sim\operatorname{Uniform}(0,1)\) and the sampler returns the state whose interval contains \(U\), the output has exactly the desired law. The cumulative intervals are the algorithm; the four masses are the model. Floating-point rounding and pseudorandom-number generation are implementation details that can still matter in a real system.

    Your turn: reverse the conditioning direction¤

    Compute \(p(X_1=1\mid X_2=1)\) and explain why it differs from \(p(X_2=1\mid X_1=1)\).

    Check your answer

    The triangle marginal is \(p(X_2=1)=0.40\), so \(p(X_1=1\mid X_2=1)=0.35/0.40=0.875\). The reverse conditional is \(0.35/0.50=0.70\). Both use the same joint cell in the numerator, but they normalize over different events. In general, reversing a conditional requires the joint law or Bayes' rule; it is not a change in notation alone.

    Chapter 2 retrieval practice¤

    1. What must the four entries of a probability mass function satisfy?
    2. How do you obtain a marginal from a joint table?
    3. Why may a continuous density be greater than one?
    4. Which quantity changes its interpretation when data are held fixed and the parameter varies?
    5. Reconstruct \(p(10)\) from \(p(X_1=1)\) and \(p(X_2=0\mid X_1=1)\).

    Chapter 2 synthesis¤

    Probability supplies normalized statements about what can happen. Marginals sum out variables; conditionals restrict and renormalize; the chain rule turns a joint law into ordered conditionals. A density is not point probability, and a likelihood is the model expression viewed as a function of parameters.

    3. The Minimum Math and Neural-Network Bridge¤

    Prerequisites and learning objectives for Chapter 3¤

    This chapter uses the finite probability table from Chapter 2. After it, you should be able to:

    1. read vectors, matrices, sums, and expectations in later chapters;
    2. interpret a gradient as local sensitivity rather than a magic update;
    3. trace an affine layer, nonlinearity, and output-law parameterization; and
    4. check dimensions and signs in a one-parameter learning step.

    Scalars, vectors, and matrices are typed containers¤

    A scalar is one number. A vector is an ordered list of numbers. A matrix is a rectangular array that often represents a linear map. For the tile law,

    \[ \mathbf p= \begin{bmatrix} 0.45\\0.05\\0.15\\0.35 \end{bmatrix} \]

    is a four-dimensional probability vector. The conditions \(p_i\ge 0\) and \(\sum_i p_i=1\) are part of its meaning, not consequences of its shape.

    If \(\mathbf x\in\mathbb R^d\), a matrix \(W\in\mathbb R^{h\times d}\) and bias \(\mathbf b\in\mathbb R^h\) define the affine map

    \[ \mathbf a=W\mathbf x+\mathbf b\in\mathbb R^h. \]

    A dimension check catches many errors before any calculation: the inner sizes of \(W\mathbf x\) must match, and the result must have the same shape as \(\mathbf b\). A neural layer usually follows the affine map with a nonlinear function \(\sigma\):

    \[ \mathbf h=\sigma(W\mathbf x+\mathbf b). \]

    Stacking such layers creates a flexible function. It does not by itself create a probability law. The final layer must parameterize a declared output object, such as categorical logits, a Gaussian mean and scale, a scalar energy, a score vector, or a transport velocity.

    From logits to a categorical law¤

    Unconstrained network outputs \(a_1,\ldots,a_K\) are often called logits. The softmax transformation produces nonnegative values summing to one:

    \[ q_i=\frac{e^{a_i}}{\sum_{j=1}^K e^{a_j}}. \]

    Adding the same constant to every logit leaves the probabilities unchanged. In numerical code, subtracting \(\max_j a_j\) before exponentiation avoids overflow without changing the result.

    For two outcomes, one logit \(\theta\) can parameterize a Bernoulli probability through the sigmoid

    \[ q_\theta=\sigma(\theta)=\frac{1}{1+e^{-\theta}}. \]

    Expectations are weighted averages under a named law¤

    For a function \(f\) on the four states,

    \[ \mathbb E_{X\sim p}[f(X)] =\sum_{x\in\mathcal X}p(x)f(x). \]

    The subscript matters. Replacing \(p\) with a model \(q_\theta\), an empirical distribution \(\widehat p\), or an approximate posterior changes the quantity. For the indicator \(f(x)=\mathbf 1\{x\text{ is a triangle}\}\), the expectation is the triangle probability, \(0.40\).

    Gradients measure local sensitivity¤

    For a scalar loss \(\ell(\theta)\), the derivative \(d\ell/d\theta\) describes the first-order change in loss for a small parameter change. In many dimensions the gradient collects all partial derivatives:

    \[ \nabla_\theta \ell(\theta) =\begin{bmatrix} \partial \ell/\partial\theta_1&\cdots& \partial \ell/\partial\theta_m \end{bmatrix}^{\!\top}. \]

    Gradient descent proposes

    \[ \theta_{t+1}=\theta_t-\eta\nabla_\theta\ell(\theta_t), \]

    where \(\eta>0\) is a step size. The minus sign follows the local direction of decrease. It does not guarantee a global optimum, a valid probability parameterization, or good generalization. Those require additional structure and evidence.

    Worked example: one Bernoulli update¤

    Let a model predict the probability of a triangle with \(q_\theta=\sigma(\theta)\). For one observed triangle \(y=1\), the negative log-likelihood is

    \[ \ell(\theta)=-\log q_\theta. \]

    Its derivative is \(q_\theta-y\). At \(\theta=0\), \(q_\theta=0.5\) and

    \[ \frac{d\ell}{d\theta}=0.5-1=-0.5. \]

    With step size \(\eta=0.2\),

    \[ \theta_{1}=0-0.2(-0.5)=0.1. \]

    The new probability is \(\sigma(0.1)\approx0.525\), so the update raises the probability assigned to the observed triangle. For a circle, \(y=0\), the gradient would be positive and the same rule would lower that probability. Automatic differentiation computes such derivatives through large programs; it does not decide whether the program expresses the right objective.

    Your turn: perform the opposite update¤

    Starting again at \(\theta=0\), perform one gradient-descent step with \(y=0\) and \(\eta=0.2\). State the sign of the new parameter and whether the triangle probability rises or falls.

    Check your answer

    At \(\theta=0\), \(q_\theta=0.5\), so the derivative is \(q_\theta-y=0.5\). The update gives \(\theta_1=0-0.2(0.5)=-0.1\). Therefore \(\sigma(-0.1)\approx0.475\): the predicted triangle probability falls and the circle probability rises.

    Chapter 3 retrieval practice¤

    1. What additional constraints turn a four-vector into a probability vector?
    2. What are the dimensions of \(W\mathbf x\) when \(W\in\mathbb R^{h\times d}\) and \(\mathbf x\in\mathbb R^d\)?
    3. Why does a neural network's final output need an interpretation contract?
    4. What does the subscript in \(\mathbb E_{X\sim p}\) tell you?
    5. What information does a gradient provide, and what does it not guarantee?

    Chapter 3 synthesis¤

    Vectors and matrices represent typed quantities and transformations. Neural networks are flexible functions whose outputs become generative objects only through a declared parameterization. Expectations are averages under named laws, and gradients are local sensitivities used by—but not equivalent to—an optimizer.

    4. Learning, Inference, and Sampling Are Different Computations¤

    Prerequisites and learning objectives for Chapter 4¤

    This chapter combines the role distinctions, finite law, and gradient bridge from Chapters 1–3. After it, you should be able to:

    1. separate population target, empirical objective, estimator, and optimizer;
    2. distinguish learning parameters from inferring latent variables;
    3. distinguish posterior inference from generation; and
    4. trace a complete train–infer–generate–evaluate workflow.

    The data law, model family, and fitted model¤

    Let \(p_\star\) denote the unknown process that produced the observations. A dataset \(D=(x_1,\ldots,x_N)\) is finite evidence about that process. A model family \(\{p_\theta:\theta\in\Theta\}\) is a set of candidate laws. A learning algorithm maps the data, initialization, randomness, and configuration to a fitted parameter \(\widehat\theta\).

    These objects should not be silently identified:

    \[ p_\star \quad\ne\quad \widehat p_D \quad\ne\quad p_{\widehat\theta} \]

    in general. The empirical law \(\widehat p_D\) records the observed frequencies. The fitted model may smooth, constrain, or generalize beyond them. Even a model that matches the finite dataset exactly need not equal the process that will produce future data.

    Population objective and empirical estimate¤

    Maximum likelihood can be motivated by the population cross-entropy

    \[ \mathcal R(\theta) =\mathbb E_{X\sim p_\star}[-\log p_\theta(X)]. \]

    Because \(p_\star\) is unknown, training uses the empirical average

    \[ \widehat{\mathcal R}_D(\theta) =-\frac1N\sum_{n=1}^N\log p_\theta(x_n). \]

    For a large dataset, a minibatch average is a stochastic estimator of this empirical objective. It is not automatically an unbiased estimator of every other quantity in the model, and lowering it on training data is not the same as lowering population risk.

    An optimizer consumes objective or gradient estimates and proposes parameters. Different optimizers, step sizes, initializations, and stopping rules can reach different fitted models even when the written objective is identical.

    Latent variables create a second direction of reasoning¤

    A latent-variable model introduces an unobserved \(Z\) and specifies a joint law

    \[ p_\theta(x,z)=p_\theta(z)p_\theta(x\mid z). \]

    Generation follows the model direction:

    \[ Z\sim p_\theta(z),\qquad X\sim p_\theta(x\mid Z). \]

    Inference reverses the question after observing \(x\):

    \[ p_\theta(z\mid x) =\frac{p_\theta(z)p_\theta(x\mid z)} {\sum_{z'}p_\theta(z')p_\theta(x\mid z')} \]

    for a finite latent space. The posterior is implied by the joint model, but it may be expensive to compute. Variational inference introduces a tractable approximation \(q_\phi(z\mid x)\); Monte Carlo methods approximate expectations with weighted or correlated samples. Approximate inference changes the computation and may change the training estimator, even when the declared generative model is fixed.

    Worked example: a hidden manufacturing mode¤

    Let \(Z\in\{A,B\}\) be a hidden manufacturing mode with equal prior mass. Mode \(A\) produces only blue tiles and uses shape probabilities

    \[ p(X=00\mid Z=A)=0.9, \qquad p(X=01\mid Z=A)=0.1. \]

    Mode \(B\) produces only orange tiles and uses

    \[ p(X=10\mid Z=B)=0.3, \qquad p(X=11\mid Z=B)=0.7. \]

    Marginalizing \(Z\) gives the same four-state law:

    \[ p(X)=0.5(0.9,0.1,0,0)+0.5(0,0,0.3,0.7) =(0.45,0.05,0.15,0.35). \]

    For an orange triangle, Bayes' rule gives \(p(Z=B\mid X=11)=1\) because mode \(A\) assigns that state zero mass. For a model whose components overlap, posterior mode probabilities would lie strictly between zero and one.

    The latent representation is not uniquely identified by the observed marginal law. One could define other hidden states that integrate to the same four probabilities. Data about \(X\) alone establish the observed law more directly than they establish a preferred hidden explanation.

    One workflow, four checkpoints¤

    1. Fit: use training data to choose parameters.
    2. Validate: measure declared properties on data not used for the same fit.
    3. Infer: answer a posterior or missing-data query for a fixed model.
    4. Generate: sample from the fitted model using a stated algorithm.

    Evaluation can reveal that a fitted model is poor; it does not retroactively turn the metric into the model's training objective. A deployment filter can change the distribution of delivered samples; it must be included when the deployed system, rather than the base model, is evaluated.

    Your turn: classify four operations¤

    5. Six Families Through One Toy Distribution¤

    Prerequisites and learning objectives for Chapter 5¤

    This chapter assumes the four-state law and workflow from earlier chapters. After it, you should be able to:

    1. give a one-sentence generative story for AR, VAE, GAN, EBM, diffusion, and flow models;
    2. identify each family's modeled object, common training signal, and generation procedure;
    3. explain which comparisons are exact in the toy world and which are only structural analogies; and
    4. choose a plausible family based on required queries rather than popularity.

    One target, different interfaces¤

    The target remains

    \[ p_\star=(0.45,0.05,0.15,0.35) \]

    over \(00,01,10,11\). Each family can be related to this target, but it exposes a different interface and makes different approximations at realistic scale.

    The diagram does not claim that the parameterizations or objectives are equivalent. It fixes one target so their structural differences are visible.

    Autoregressive models: multiply normalized conditionals¤

    Choose an order, then factorize:

    \[ p(x_1,x_2)=p(x_1)p(x_2\mid x_1). \]

    The exact tile parameters are

    \[ \begin{aligned} p(X_1=1) &= 0.5, \\ p(X_2=1\mid X_1=0) &= 0.1, \\ p(X_2=1\mid X_1=1) &= 0.7. \end{aligned} \]

    Generation draws color, then shape conditional on color. Scoring multiplies the two normalized terms. In high dimensions, the architecture shares parameters across conditionals and the order affects computation and inductive bias even though the chain rule itself is universal.

    Connection: the inverse-CDF tile sampler and a token-by-token language model are both ancestral samplers. The surface objects differ, but both append one draw conditional on the prefix.

    Variational autoencoders: latent generation plus learned inference¤

    The hidden-mode model from Chapter 4 writes

    \[ p_\theta(x)=\sum_z p_\theta(z)p_\theta(x\mid z). \]

    A VAE uses neural networks for a decoder \(p_\theta(x\mid z)\) and an approximate posterior \(q_\phi(z\mid x)\). It commonly trains with the evidence lower bound

    \[ \mathbb E_{q_\phi(z\mid x)}[\log p_\theta(x\mid z)] -\operatorname{KL}\!\left(q_\phi(z\mid x)\,\|\,p(z)\right) \le \log p_\theta(x). \]

    Generation samples the prior and decodes. Encoding a datum with \(q_\phi(z\mid x)\) is inference, not the unconditional generative direction. The finite tile posterior is exact; “VAE” specifically adds amortized variational inference for settings where exact posterior calculations are not generally available.

    Connection: an autoregressive decoder may appear inside a VAE. “VAE” names the latent-model and inference/training structure, while “autoregressive” names a factorization of an output law; they are composable rather than mutually exclusive labels.

    GANs: match through a learned comparison¤

    Let \(U\sim\operatorname{Uniform}(0,1)\) and let a generator \(G_\theta(U)\) map noise to a tile. A sufficiently flexible generator can allocate intervals of length \(0.45,0.05,0.15,0.35\) to the four states. The pushforward law of \(G_\theta(U)\) then matches the target.

    A GAN does not fit those interval lengths by maximum likelihood. A discriminator or critic learns to distinguish data from generated samples, while the generator changes to improve the adversarial objective. At an ideal population optimum under the stated game, generated and data laws can agree; finite networks, finite samples, and optimization dynamics need not reach that optimum.

    Generation is direct once \(G_\theta\) is trained. A normalized likelihood for an arbitrary tile is not generally exposed by the neural generator.

    Connection: both a GAN and a VAE can use a decoder-shaped network from latent noise to observations. The same architectural arrow does not make their probability semantics or training objectives the same.

    Energy-based models: assign scalar compatibility, then normalize¤

    An EBM writes

    \[ p_\theta(x)=\frac{e^{-E_\theta(x)}}{Z_\theta}, \qquad Z_\theta=\sum_{x'}e^{-E_\theta(x')} \]

    for the finite tile world. Choosing

    \[ E(x)=-\log p_\star(x) \]

    gives \(Z=1\) and exactly recovers the target. Adding the same constant to every energy changes \(Z\) but not the normalized law. For large spaces, calculating \(Z_\theta\) and drawing representative samples can be the hard parts. MCMC is a sampling method for an energy-defined law; the energy function is the model component.

    Connection: the score of a differentiable continuous EBM is \(\nabla_x\log p_\theta(x)=-\nabla_xE_\theta(x)\) because the normalizer does not depend on \(x\). Diffusion models often learn scores without first learning an explicit scalar energy.

    Diffusion models: learn to reverse a corruption process¤

    A discrete corruption kernel can randomly replace a tile with progressively less informative states. If \(q_t(y\mid x)\) is the forward corruption law, then the exact reverse conditional under the data law is

    \[ q_t(x\mid y) =\frac{p_\star(x)q_t(y\mid x)} {\sum_{x'}p_\star(x')q_t(y\mid x')}. \]

    Training examples can be constructed by drawing a real tile and corrupting it. A neural model learns reverse conditionals, noise predictions, scores, or velocity-like targets depending on the formulation. Generation begins from a simple high-noise distribution and applies a numerical reverse process.

    For four states the reverse table can be computed exactly. In image, audio, or continuous-state models it is learned and the finite-step sampler introduces an additional approximation.

    Connection: diffusion generation is sequential like autoregressive generation, but its sequence usually evolves whole noisy states over time rather than appending one observed component at a time.

    Normalizing flows: transport through an invertible map¤

    For continuous variables, an invertible map \(x=f_\theta(z)\) transforms a base density according to

    \[ p_X(x)=p_Z(f_\theta^{-1}(x)) \left|\det J_{f_\theta^{-1}}(x)\right|. \]

    Generation applies the forward map to base noise; scoring uses the inverse and Jacobian determinant. Continuous-time flows replace a finite composition of maps with an ODE and track density change through a divergence.

    The discrete tile world exposes a useful limitation. A bijection of four states merely permutes four base masses. A uniform discrete base therefore remains uniform and cannot become \((0.45,0.05,0.15,0.35)\) through a bijection alone. One needs a nonuniform base, an augmented/discrete construction, or a continuous representation such as dequantized tiles. “Invertible” does not by itself imply universal distribution matching.

    Connection: GAN generators and flows both push base noise through a neural map. The flow constrains the map so that density transformation is available; a typical GAN permits a noninvertible map and uses adversarial comparison instead of change-of-variables likelihood.

    Worked comparison: choose by query¤

    Suppose exact per-tile scoring and fast sampling are both required. In the four-state world, a table or AR factorization provides both directly. An EBM offers energies but may make normalization and sampling expensive. A diffusion model introduces an iterative sampler that is unnecessary for this tiny space. A VAE is useful if a latent representation is part of the query, but its marginal score may require a bound or estimator.

    This is not a universal ranking. Change the query to “learn a useful latent representation” or “generate high-dimensional perceptual data under a fixed latency budget,” and the tradeoffs change.

    6. How to Read Equations and Execute the First Model¤

    Prerequisites and learning objectives for Chapter 6¤

    This chapter assumes the tile law and role map from Chapters 1–5. After it, you should be able to:

    1. translate a compact probability equation into a sequence of operations;
    2. map symbols in an equation to variables in code;
    3. execute and modify a dependency-free finite generative model; and
    4. use normalization, boundary, and simulation checks to catch mistakes.

    Read equations in three passes¤

    Consider the empirical negative log-likelihood

    \[ \widehat{\mathcal R}(q) =-\frac1N\sum_{n=1}^{N}\log q(x_n). \]

    Pass 1: name the output. The left side is one scalar loss computed from a candidate mass function \(q\) and a dataset.

    Pass 2: identify scope and binding. The index \(n\) runs from \(1\) through \(N\). Each \(x_n\) is an observed state. The expression \(q(x_n)\) selects the mass assigned to that state.

    Pass 3: execute the operators. For each observation: look up its mass, take the natural logarithm, add the results, divide by \(N\), and negate. An impossible observed state with \(q(x_n)=0\) gives infinite loss, which is a support failure rather than a floating-point inconvenience.

    This three-pass method—output, scope, operations—works for expectations, integrals, matrix equations, and update rules. Add a fourth pass for assumptions: which law generates the expectation, which variables are fixed, and where denominators or inverses are defined?

    A compact notation decoder¤

    Table FIRST.6. A compact notation decoder
    Symbol Read it as Code-level question
    \(x\sim p\) draw \(x\) according to law \(p\) Which sampler, seed, and approximation?
    \(\mathbb E_{p}[f(X)]\) average \(f\) under \(p\) Exact sum, quadrature, or Monte Carlo?
    \(p(x\mid c)\) law or density of \(x\) given \(c\) How is \(c\) represented and normalized?
    \(\arg\min_\theta L(\theta)\) the parameter set minimizing \(L\) Which optimizer approximates it?
    \(\nabla_\theta L\) local sensitivity to parameters Is it exact, autodifferentiated, or estimated?
    \(\propto\) equal up to a positive factor independent of the displayed variable What normalizer is omitted?
    \(\approx\) approximation Which limit, error source, or diagnostic justifies it?

    Do not translate \(\arg\min\) as “gradient descent.” The former defines a target set; the latter is one algorithm that may or may not reach it. Do not translate \(x\sim p\) as “call this particular library function.” The mathematical draw and its implementation are separate contracts.

    From the tile equations to executable Python¤

    The following program uses only the Python standard library. It estimates the four-state empirical model, verifies the joint and conditional calculations, computes its average negative log-likelihood, and generates reproducible samples by inverse-CDF sampling.

    from bisect import bisect
from itertools import accumulate
from math import isclose, log
from random import Random

states = ((0, 0), (0, 1), (1, 0), (1, 1))
counts = (9, 1, 3, 7)
sample_size = sum(counts)
pmf = tuple(count / sample_size for count in counts)

assert all(probability >= 0.0 for probability in pmf)
assert isclose(sum(pmf), 1.0)

orange_mass = pmf[2] + pmf[3]
orange_triangle_mass = pmf[3]
triangle_given_orange = orange_triangle_mass / orange_mass
assert isclose(triangle_given_orange, 0.7)

average_negative_log_likelihood = -sum(
    count * log(probability) for count, probability in zip(counts, pmf, strict=True)
) / sample_size

cumulative = tuple(accumulate(pmf))


def sample_tile(uniform_draw: float) -> tuple[int, int]:
    """Map one draw in [0, 1) to a tile using cumulative masses."""
    if not 0.0 <= uniform_draw < 1.0:
        raise ValueError("uniform_draw must lie in [0, 1)")
    return states[bisect(cumulative, uniform_draw)]


rng = Random(7)
generated = tuple(sample_tile(rng.random()) for _ in range(8))

print("pmf:", pmf)
print("P(triangle | orange):", triangle_given_orange)
print("average NLL:", round(average_negative_log_likelihood, 6))
print("generated:", generated)

    Expected output under the specified Python pseudorandom generator is:

    pmf: (0.45, 0.05, 0.15, 0.35)
P(triangle | orange): 0.7
average NLL: 1.161121
generated: ((0, 0), (0, 0), (1, 1), (0, 0), (1, 0), (0, 0), (0, 0), (1, 0))

    The assertions are part of the example, not decoration. They encode model invariants and a hand-derived conditional. The fixed seed makes this short run reproducible; it does not prove that eight draws match the distribution. A simulation diagnostic needs many draws and uncertainty-aware tolerances.

    Trace the roles in the program¤

    Table FIRST.7. Trace the roles in the program
    Program element Conceptual role
    states outcome-space representation
    counts observed finite data summary
    pmf fitted empirical model
    average NLL expression empirical objective evaluated at the fitted model
    cumulative and sample_tile exact categorical sampling algorithm
    rng reproducible pseudorandom source, not the probability law itself
    assertions finite correctness checks

    There is no optimizer because the unconstrained categorical maximum-likelihood estimate has a closed form: count each state and divide by \(N\). There is no latent inference because the model contains no latent variable. Adding library machinery would not improve this example's semantics.

    Worked mutation: smoothing an unseen state¤

    Suppose a smaller training set has counts \((4,0,1,3)\). The empirical estimate assigns state \(01\) zero mass, so one future \(01\) observation has infinite log loss. Add-\(\alpha\) smoothing uses

    \[ q_\alpha(x_i)=\frac{n_i+\alpha}{N+K\alpha} \]

    for \(K=4\) states and \(\alpha>0\). With \(\alpha=1\), the masses are

    \[ q_1=(5,1,2,4)/12. \]

    This is a modeling choice, not a numerical trick: it changes the fitted law by allocating mass to unseen outcomes. A Bayesian interpretation is available through a symmetric Dirichlet prior, but the operational consequence can be understood before that derivation.

    Check your answer

    The unsmoothed mass is zero. In the displayed expression Python still evaluates log(0.0) even though its count multiplier is zero, so the code raises a domain error; a robust implementation must omit zero-count terms or handle support explicitly. Add-one smoothing gives mass \(1/12\) to 01, and a sampler built from the smoothed masses can generate it. This exercise separates a mathematical convention (\(0\log0=0\) in entropy-like sums) from the eager evaluation behavior of code.

    Diagnostic solutions
    1. \(E_\theta\) defines the unnormalized model object; score matching is the training objective; Langevin dynamics is the sampler.
    2. An implicit generator can map noise to samples without an invertible map or tractable change-of-variables density.
    3. The circle marginal is \(0.45+0.15=0.60\), so \(p(X_1=0\mid X_2=0)=0.45/0.60=0.75\).
    4. Density is measured relative to a reference measure; probability is an integral over a set. A narrow interval can have density above one while its integrated probability remains at most one.
    5. \(\theta_{t+1}=\theta_t-0.1(-3)=\theta_t+0.3\). This local step does not guarantee a global optimum, generalization, or even a lower loss when the step is too large.
    6. The product lies in \(\mathbb R^5\).
    7. Generation draws \(z\) from its prior and then \(x\mid z\); inference asks which \(z\) values are plausible after observing \(x\), namely \(p_\theta(z\mid x)\).
    8. Marginalization can erase distinctions among joint decompositions; observed \(X\) alone need not identify a unique latent coordinate or causal interpretation.
    9. Both push base noise through a learned map. A density flow constrains the map to provide an inverse and Jacobian density contract; a typical GAN uses adversarial training and exposes no normalized likelihood.
    10. Both produce a sequence of conditional updates. AR usually appends observed components in an order; diffusion iteratively transforms a noisy whole-state representation along a time path.
    11. pmf specifies the four masses; sample_tile is one algorithm that converts uniform randomness into a draw from those masses.
    12. Repeatedly call sample_tile(rng.random()), count each returned state, divide by 10,000, and compare the empirical frequencies with pmf. A useful diagnostic reports deviations together with sampling uncertainty instead of demanding exact equality.
