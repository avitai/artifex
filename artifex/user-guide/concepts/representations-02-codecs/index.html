    2. Discrete, Continuous, Hierarchical, and Temporal Codecs¤

    Chapter 2 prerequisites and learning objectives¤

    This chapter assumes Chapter 1 and latent-variable modeling. By the end, you should be able to compare discrete, continuous, hierarchical, residual, and temporal codecs by rate, distortion, prior, and decoder contracts; diagnose codebook collapse and temporal inconsistency; explain how codec choice changes the downstream generative target; and select metrics that expose perceptual and task-relevant information loss.

    Vector quantization¤

    VQ-VAE maps encoder output to the nearest codebook vector:

    \[ k^*=\arg\min_k\|z_e(x)-e_k\|^2, \qquad z_q=e_{k^*}. \]

    The integer \(k^*\) and vector \(e_{k^*}\) have different roles. A categorical generator models indices; a decoder consumes embeddings. The indices have no intrinsic metric—permuting their names and applying the same permutation to the codebook preserves the codec—whereas distances between embeddings affect quantization and decoding. A prior trained before such a permutation is no longer compatible.

    Nearest-neighbour assignment is discontinuous. A common hard VQ recipe copies the decoder gradient to the encoder with a straight-through surrogate, updates selected embeddings with a codebook loss or moving assignment statistics, and uses a commitment loss to keep encoder outputs near their selected codes. These are different update routes. The VAE estimator chapter derives their stop-gradient equations, gives a complete EMA example, and compares them with exact enumeration, score-function estimators, and Gumbel–Softmax.

    Codec, vocabulary, and prior are three interfaces¤

    Let \(c(x)=(c_1,\ldots,c_T)\) be the encoded index sequence. A two-stage system first learns an encoder–codebook–decoder and then fits \(p_\psi(c_1,\ldots,c_T)\) to frozen encoded data. Its behavior must be audited at three interfaces:

    Table REP.4. Codec, vocabulary, and prior are three interfaces
    Interface Representative checks Failure hidden by the next interface
    Observation \(\leftrightarrow\) codec reconstruction likelihood/distortion, task fidelity, support, latency information discarded before generation
    Encoder \(\leftrightarrow\) vocabulary quantization error, code counts, entropy/perplexity, dead-code duration, assignment stability nominal vocabulary that supplies little effective capacity
    Vocabulary \(\leftrightarrow\) prior held-out token NLL, conditioning, sequence length, sampler cost, checkpoint/index compatibility prior mismatch or expensive code dependencies

    A codebook of size \(K\) permits at most \(K\) symbols per position and a fixed- width identifier needs \(\lceil\log_2K\rceil\) bits. Neither is the source rate. If observed code frequencies are \(\widehat p_k\), their marginal empirical entropy is \(-\sum_k\widehat p_k\log_2\widehat p_k\) bits per token; a contextual entropy model can exploit dependencies and assign a different cross-entropy. Actual compression additionally requires synchronized probability tables and a bitstream implementation.

    Discrete indices support categorical autoregressive, masked, diffusion, and other priors. Quantization can shorten or regularize the state sequence, but it also adds discontinuity and reconstruction loss. Code utilization alone does not settle the trade-off: a tokenizer can use all codes to encode nuisance noise, and a low-entropy source can legitimately need only a subset.

    From latent tensor to bitstream¤

    A learned representation becomes a compression system only after its protocol is specified:

    1. an analysis transform maps \(x\) to latent values;
    2. quantization maps those values to discrete symbols \(\hat z\);
    3. an entropy model assigns synchronized probabilities \(m_\psi(\hat z)\);
    4. an entropy coder and decoder turn the symbols into and out of a bitstream;
    5. a synthesis transform reconstructs \(\hat x\).

    The following measurements answer different questions:

    Table REP.5. From latent tensor to bitstream
    Quantity What it measures What it omits
    Latent shape Number of stored or modeled scalars precision and statistical redundancy
    Nominal index rate tokens \(\times\lceil\log_2K\rceil\) nonuniform and contextual predictability
    Empirical entropy frequency uncertainty in a chosen sample ordering, side information, coder overhead
    Entropy-model cross-entropy ideal expected bits under \(m_\psi\) finite coder and file-format overhead
    Actual bitrate serialized bits per pixel, sample, second, or source symbol reconstruction and perceptual quality
    Generator token count state-space size seen by the prior whether any code is serialized at all

    For a small audit, suppose 16 tokens contain symbols with counts \((8,4,2,2)\). A fixed-width 256-entry vocabulary uses a nominal \(16\times8=128\) bits. The empirical entropy of the four used symbols is \(1.75\) bits per token, or 28 ideal bits for this sequence under its empirical frequencies. Neither number is an actual file size: a decoder must know the probability model, and a real stream has termination, finite-precision, and possibly header costs. A contextual autoregressive entropy model can also assign a rate different from the order-free frequency calculation.

    In latent generation, a compact grid may be chosen primarily to reduce prior compute. Calling a \(16\times\) spatial downsampling factor “\(256\times\) compression” confuses positions with information: channels, precision, entropy, and overhead remain. If no entropy-coded bitstream is produced, report latent shape and generator compute rather than a storage compression ratio.

    Finite scalar and factorized quantization¤

    Finite Scalar Quantization replaces learned vector lookup with bounded scalar levels across dimensions. It can improve code utilization and simplicity, but factorization imposes geometry and does not eliminate the rate–distortion trade-off.

    Residual, product, and lookup-free quantization¤

    One code index need not carry the entire quantization budget. In residual vector quantization, the first codebook approximates \(z\), the next quantizes the remaining error, and so on:

    \[ r_0=z,\qquad e_\ell=Q_\ell(r_{\ell-1}),\qquad r_\ell=r_{\ell-1}-e_\ell,\qquad \widehat z=\sum_{\ell=1}^L e_\ell. \]

    With codebook sizes \(K_1,\ldots,K_L\), a fixed-width representation uses \(\sum_\ell\lceil\log_2K_\ell\rceil\) nominal bits per latent position. The actual entropy can be smaller and depends on correlations within and across levels. Dropping later levels supplies a variable-rate hierarchy only if the decoder was trained for those truncations. SoundStream is a canonical residual-quantization codec with structured quantizer dropout; its reported audio results are protocol-specific rather than a theorem that residual quantization dominates a single codebook.

    Product quantization splits coordinates into groups and combines their indices. It reduces lookup cost but assumes that the chosen grouping exposes useful factorization. Lookup-free or binary schemes encode signs/bits without storing an explicit table of all \(2^d\) combinations. MAGVIT-v2 uses lookup-free quantization in a visual tokenizer. Eliminating the explicit table does not eliminate the exponentially large nominal alphabet, entropy modeling, bit imbalance, or decoder distortion.

    Reconstruction and prior learnability are separate objectives¤

    Let \(C=E(X)\) be a discrete code sequence and let \(m_\psi(C)\) be its learned prior. Codec reconstruction evaluates \(D(X,D(C))\); prior learnability is probed by held-out

    \[ \mathbb E[-\log m_\psi(C)]. \]

    Two lossless representations can have identical ideal information but different sequence interfaces. If \(X=(U,V)\) consists of two independent fair bits, a tokenizer may emit two binary tokens \((U,V)\) or one four-way token with index \(2U+V\). Both use two ideal bits and reconstruct exactly. The first needs two generation positions with binary outputs; the second needs one position with a four-class output. With finite networks, masks, vocabularies, and hardware, their optimization and latency need not match even though neither contains more information.

    Tokenizer--prior co-design can add a proxy for downstream predictability, denoising robustness, or generator loss. It changes the representation target and can sacrifice a reconstruction property not captured by that proxy. Latent Denoising Makes Good Visual Tokenizers, an ICLR 2026 paper, reports gains across its six named downstream generators when tokenizer latents are trained to reconstruct after corruption. This is evidence for downstream-aligned tokenizer training under those protocols, not a task-independent definition of a good latent.

    Retrieval practice: quantizer composition
    1. Does an implicit \(2^d\) lookup-free alphabet have zero rate?
    2. When does truncating residual levels define a supported bitrate?
    3. Can equal reconstruction imply equal prior compute?

    Solutions. (1) No; its bits have entropy and must still be generated or coded. (2) When training and evaluation include the same truncation rule and decoder behavior. (3) No; token count, alphabet, dependencies, and architecture can differ.

    Continuous latents¤

    Continuous codecs avoid quantization and work naturally with diffusion or flows. Scale, covariance, and tail behavior affect noise schedules and conditioning. A learned latent need not be Gaussian merely because the generator starts from Gaussian noise.

    Latent Diffusion separates perceptual compression from generative modeling. This reduces spatial compute but makes decoder distortion and latent scaling part of every result.

    Hierarchical and multiscale tokens¤

    Coarse latents capture layout; fine latents capture texture. Pyramid or residual quantizers can allocate rate adaptively. A generator may model scales sequentially, jointly, or with separate experts. Coarse errors propagate, while independent fine-scale decoding can lose global consistency.

    Video and audio¤

    Temporal codecs compress space and time. Causal encoders support streaming but see less future context; bidirectional encoders improve compression but add latency. Temporal stride can erase fast events, lip–audio alignment, physical contacts, or beats.

    Video reconstruction needs temporal metrics in addition to frame quality. Token count scales with duration, resolution, frame rate, and stride; reporting tokens per clip without these quantities is meaningless.

    Semantic versus reconstruction tokens¤

    Understanding favors invariance to nuisance detail; generation needs enough detail to reconstruct. A single representation can compromise both. Dual codebooks, shared indices with separate features, or decoupled encoders trade architectural unity against task specialization.

    Chapter 2 synthesis¤

    Discrete tokens simplify categorical modeling; continuous latents simplify geometric transport; hierarchies allocate information across scale; temporal codecs determine synchronization and latency. Chapter 3 explains how modalities interact after representation.
