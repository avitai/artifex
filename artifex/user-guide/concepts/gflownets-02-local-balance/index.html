    2. Flow Matching and Detailed Balance¤

    Chapter 2 prerequisites and learning objectives¤

    This chapter assumes Chapter 1 and stable log-domain arithmetic. After completing it, you should be able to:

    1. write flow-matching conservation for terminal-without-stop and explicit-stop graph conventions without double-counting reward;
    2. derive detailed balance from compatible state flow and forward/backward kernels and telescope it along a complete trajectory;
    3. distinguish what FM, DB, and terminal boundary constraints each parameterize and establish at zero residual;
    4. implement positive-flow sums, masks, empty-set handling, and source/terminal constraints stably in log space;
    5. diagnose reachability, co-reachability, reverse-support, and sampled-coverage failures that a low empirical loss can hide; and
    6. transfer local balance to a new construction graph while specifying its parent-enumeration and state-flow costs.

    The original GFlowNet construction converts conservation equations into learnable residuals, analogous to temporal-difference learning.

    Flow Matching¤

    Parameterize edge or state flows. For each noninitial state \(s\), compare total incoming flow with total outgoing flow plus terminal reward when applicable:

    \[ \sum_{p}F_\theta(p\to s) =R(s)\mathbf1_{s\in\mathcal X} +\sum_cF_\theta(s\to c). \]

    A stable loss commonly squares a log-domain residual. Exact form, reward placement, and stop-action representation must be declared. Summing exponentially large parent/child sets may itself require structured parameterization.

    For a graph in which terminal states have no outgoing edges, one common positive-flow residual is

    \[ \Delta_{\mathrm{FM}}(s)= \log\!\left(\sum_{p\in\operatorname{Pa}(s)}F_\theta(p\to s)\right) - \log\!\left(R(s)\mathbf 1_{s\in\mathcal X} +\sum_{c\in\operatorname{Ch}(s)}F_\theta(s\to c)\right). \]

    Training minimizes a sampled average of \(\Delta_{\mathrm{FM}}(s)^2\). Implement both sums with logsumexp, mask invalid edges before normalization, and define what happens when a sum is empty. A source constraint is handled separately; there is no incoming parent flow to match there.

    With an explicit stop edge, an object-bearing state can have both continuation edges and a stop edge. The reward is attached to the stop/terminal boundary rather than added again to ordinary outgoing flow. These two formulations are equivalent only after the state graph and boundary variables are mapped consistently.

    Detailed Balance¤

    With learned state flow \(F_\theta(s)\) and policies, impose for each edge

    \[ F_\theta(s)P_F(s'\mid s) =F_\theta(s')P_B(s\mid s'). \]

    At terminal states set \(F_\theta(x)=R(x)\). This edge-local equation ensures the same flow is seen from both directions. It resembles MCMC detailed balance algebraically, but the interpretation differs: a GFlowNet constructs acyclic trajectories rather than running a stationary reversible chain among terminal objects.

    The scopes differ. Flow matching may require a full parent/child sum at one state; detailed balance checks one sampled edge but carries an extra learned state flow and backward kernel. Multiplying compatible DB equations changes the scope to a path—it does not make the three parameterizations identical.

    Why Detailed Balance Is Sufficient¤

    Multiply detailed-balance ratios along a complete trajectory:

    \[ \prod_{t=0}^{n-1} \frac{P_F(s_{t+1}\mid s_t)}{P_B(s_t\mid s_{t+1})} = \prod_{t=0}^{n-1}\frac{F(s_{t+1})}{F(s_t)} =\frac{R(x)}{F(s_0)}. \]

    The state-flow ratios telescope. Rearranging yields the trajectory-balance identity introduced in Chapter 3. Summing over all backward paths then yields reward-proportional terminal sampling, provided \(P_B\) is a normalized kernel whose paths return to the source and the relevant graph support is covered.

    The converse needs care. A set of full-trajectory TB equations can determine the correct terminal law without uniquely determining every internal state flow. Detailed balance specifies a stronger edge-local factorization when its state-flow variables exist. Thus “same terminal population optimum” is not “identical parameterization or optimization landscape.”

    Worked example: a numerical edge check¤

    If \(F(s)=10\), \(P_F(s'\mid s)=0.3\), and \(F(s')=6\), then balance requires

    \[ P_B(s\mid s')=\frac{10(0.3)}6=0.5. \]

    The remaining backward probability at \(s'\) must be allocated among other parents. If the computed ratio exceeds one, the proposed flows/policies cannot satisfy the local constraint.

    For the Chapter 1 diamond, let \(F(s_0)=4\), \(F(a)=1\), \(F(b)=2\), and \(F(x)=3\). Edge balance on \(a\to x\) gives \(1(1)=3P_B(a\mid x)\), hence \(P_B(a\mid x)=1/3\); the \(b\) edge gives \(2/3\). The backward probabilities normalize because the two incoming edge flows sum to the state flow at \(x\).

    Boundary Conditions Matter¤

    Local edge balance alone does not select the desired target unless terminal flows are anchored to rewards and the source/graph are connected appropriately. Invalid actions, premature stop, duplicate terminal encodings, and unreachable states alter the state graph.

    Three conditions are easy to miss:

    • Reachability: every positive-reward terminal must be reachable from \(s_0\).
    • Co-reachability: a partial state visited by the policy must be able to terminate or route to an explicit failure sink.
    • Kernel support: if \(P_F(s'\mid s)>0\), the reverse edge required by \(P_B(s\mid s')\) must be represented for DB/TB evaluation, and vice versa for the theorem being used.

    An edge residual of zero on one connected component says nothing about a missing positive-reward component. Boundary conditions select the target only over the graph actually modeled.

    Credit Propagation¤

    Local residuals propagate sparse terminal rewards backward over many updates. Long trajectories and large branching factors can make this slow. Replay, backward sampling, reward shaping that preserves the target, and longer-balance objectives address optimization—not the population conservation theorem.

    “Reward shaping that preserves the target” is narrower than ordinary RL shaping. A potential can be incorporated into a reparameterized flow/objective when the terminal boundary remains exactly \(R(x)\); replacing terminal rewards or tempering them changes the target. Every shaping proposal must show the invariance rather than relying on analogy.

    Parent Enumeration and State-Flow Models¤

    Flow matching may require enumerating every parent and child of a sampled state. That is cheap on a small grid but difficult when a graph has combinatorially many inverse actions. Detailed balance evaluates sampled edges and can avoid full parent sums if \(P_F\), \(P_B\), and \(F(s)\) are separately normalized/parameterized. The trade-off is that the model must learn mutually compatible state flow and policies.

    A common parameterization emits log_F(s) and forward/backward action logits. These are distinct outputs: a softmax normalizes action probabilities but does not enforce the scalar flow equation. A successful unit test checks masks, normalization, edge residuals, terminal anchoring, and exact enumeration on a toy graph.

    Counterexample: Small Sampled Loss, Wrong Global Law¤

    Return to two disjoint branches with rewards 1 and 100. If the behavior policy visits only the first branch, every sampled local residual can become zero while the high-reward terminal remains absent. The empirical loss is then a statement about the behavior distribution, not a certificate over the full DAG. This is why exploration belongs to correctness in practice.

    Chapter 2 retrieval Questions¤

    1. What telescopes when detailed-balance identities are multiplied along a path?
    2. Why is GFlowNet detailed balance not an MCMC stationarity result?
    3. Which implementation operation prevents overflow when summing learned flows?
    Complete solutions
    1. The successive state-flow ratios \(F(s_{t+1})/F(s_t)\) telescope to \(F(x)/F(s_0)\).
    2. The forward process is an acyclic source-to-terminal construction, not a recurrent Markov chain over objects at stationarity.
    3. Compute positive-flow sums in log space with masked logsumexp.

    Chapter 2 Synthesis¤

    Flow matching enforces node conservation; detailed balance enforces edge agreement between state flow and forward/backward policies. Both need reward boundary conditions. Chapter 3 propagates reward across whole or partial trajectories.

    Misconception check

    GFlowNet detailed balance does not mean the forward constructive policy is an MCMC chain with stationary distribution over partial states. The shared equation is an analogy with different dynamics.

    Independent practice and assessment map: local balance¤

    1. P1.1 — Predict before computing. An exact DB solution has every state and edge flow multiplied by 7 while \(P_F\) and \(P_B\) are unchanged. Predict which DB residuals, terminal boundary equations, terminal probabilities, and source normalizer remain valid when rewards are (a) unchanged and (b) also multiplied by 7.
    2. P1.2 — Flow-matching calculation. At state \(s\), incoming edge flows are 2 and 6, ordinary outgoing flows are 3 and 1, and \(s\) is also allowed to stop. Determine the stop flow required by conservation. Then show how double-counting the same reward both as stop flow and as an added terminal term breaks the equation.
    3. P1.3 — Detailed-balance derivation. Along \(s_0\to a\to x\), let state flows be \((12,4,3)\) and forward probabilities be \((1/3,3/4)\). Compute the two required backward probabilities, multiply the edge ratios, and recover the complete-path identity.
    4. P1.4 — Log-domain implementation. Given incoming log flows \((1000,999,-\infty)\) with the last edge invalid, write the masked logsumexp expression, reduce it algebraically without exponentiating 1000, and state how an empty valid-parent set differs at the source and at an ordinary reachable state.
    5. P1.5 — Coverage counterexample. Build a two-component graph whose sampled component has reward 1 and exact zero residual while an unvisited reachable terminal has reward 100. Explain why empirical FM/DB loss can be zero and name the additional test that exposes the error.
    6. P1.6 — Transfer. For an ambiguous grammar in which a partial derivation may have many inverse expansions, choose between FM and DB for one implementation. Specify the required outputs, masks, boundary, parent operation, and one exact unit test; justify the computational trade-off rather than claiming one objective is universally better.
    Outcome-to-task alignment: node and edge constraints¤
    Table GFN.8. Outcome-to-task alignment: node and edge constraints
    Learning objective Independently assessed by
    O1 — Write convention-consistent FM equations; evidence is the explicit-stop calculation P1.2
    O2 — Derive and telescope DB; evidence is the two-edge numerical derivation P1.3
    O3 — Separate local parameterizations and boundary consequences; evidence is the flow-rescaling audit P1.1, P1.2
    O4 — Implement stable positive-flow arithmetic; evidence is the masked logsumexp and empty-set contract P1.4
    O5 — Diagnose graph and behavior support failures; evidence is the zero-sampled-loss counterexample P1.5
    O6 — Transfer local balance with an explicit cost model; evidence is the ambiguous-grammar design P1.6
    Complete solutions to P1
    1. DB remains exact after common flow scaling because both sides of every edge equation multiply by 7; policies and terminal probabilities are unchanged. With unchanged rewards, however, \(F'(x)=7R(x)\) violates the terminal boundary and \(F'(s_0)=7Z\) is not the normalizer of the declared reward. If rewards also scale by 7, all boundaries hold, the source normalizer scales by 7, and the normalized terminal law remains unchanged.
    2. Total incoming flow is 8 and ordinary continuation uses 4, so the stop edge must carry 4. In the explicit-stop convention, that stop boundary carries the object's reward once. Writing \(8=3+1+4_{\text{stop}}+4_{\text{reward}}\) demands outflow 12 and counts one terminal mass twice.
    3. On \(s_0\to a\), DB gives \(12(1/3)=4P_B(s_0\mid a)\), hence \(P_B(s_0\mid a)=1\). On \(a\to x\), it gives \(4(3/4)=3P_B(a\mid x)\), hence \(P_B(a\mid x)=1\). The forward product is \(1/4\); multiplying by \(Z=12\) gives 3, equal to \(R(x)\) times the backward product. Equivalently, the state ratios telescope from \(F(x)/F(s_0)=3/12\).
    4. Mask first, then compute \(\operatorname{LSE}(1000,999)=1000+\log(1+e^{-1})\approx1000.3133\); never form \(e^{1000}\). The source legitimately has no parents and uses a separate source-flow constraint. An ordinary state declared reachable but with no valid parents violates the graph/support contract; silently treating its incoming log flow as zero or \(-\infty\) hides that bug.
    5. Let the behavior visit only \(s_0\to a\to x\) with \(R(x)=1\), while another source action reaches \(b\to y\) with \(R(y)=100\) but is never sampled. All evaluated residuals on the first branch can be zero while the learned policy assigns no mass to \(y\). Exact enumeration or a coverage test over every positive-reward terminal/edge reveals the missing component; a minibatch mean cannot.
    6. A DB implementation can avoid summing all inverse grammar expansions by emitting log_F(s), normalized forward logits over valid expansions, and normalized backward logits over represented inverse expansions. It anchors completed derivations to reward and keeps forward/backward masks consistent. The exact unit test enumerates a tiny ambiguous grammar, checks every edge DB residual and terminal frequency after path aggregation, and verifies that duplicate parses do not become duplicate semantic terminals. FM is attractive only if all needed parent and child sums can be computed reliably; that graph-specific cost decides the choice.
