    4. Learning, Inference, and Sampling Are Different Computations¤

    Prerequisites and learning objectives for Chapter 4¤

    This chapter combines the role distinctions, finite law, and gradient bridge from Chapters 1–3. After it, you should be able to:

    1. separate population target, empirical objective, estimator, and optimizer;
    2. distinguish learning parameters from inferring latent variables;
    3. distinguish posterior inference from generation; and
    4. trace a complete train–infer–generate–evaluate workflow.

    The data law, model family, and fitted model¤

    Let \(p_\star\) denote the unknown process that produced the observations. A dataset \(D=(x_1,\ldots,x_N)\) is finite evidence about that process. A model family \(\{p_\theta:\theta\in\Theta\}\) is a set of candidate laws. A learning algorithm maps the data, initialization, randomness, and configuration to a fitted parameter \(\widehat\theta\).

    These objects should not be silently identified:

    \[ p_\star \quad\ne\quad \widehat p_D \quad\ne\quad p_{\widehat\theta} \]

    in general. The empirical law \(\widehat p_D\) records the observed frequencies. The fitted model may smooth, constrain, or generalize beyond them. Even a model that matches the finite dataset exactly need not equal the process that will produce future data.

    Population objective and empirical estimate¤

    Maximum likelihood can be motivated by the population cross-entropy

    \[ \mathcal R(\theta) =\mathbb E_{X\sim p_\star}[-\log p_\theta(X)]. \]

    Because \(p_\star\) is unknown, training uses the empirical average

    \[ \widehat{\mathcal R}_D(\theta) =-\frac1N\sum_{n=1}^N\log p_\theta(x_n). \]

    For a large dataset, a minibatch average is a stochastic estimator of this empirical objective. It is not automatically an unbiased estimator of every other quantity in the model, and lowering it on training data is not the same as lowering population risk.

    An optimizer consumes objective or gradient estimates and proposes parameters. Different optimizers, step sizes, initializations, and stopping rules can reach different fitted models even when the written objective is identical.

    Latent variables create a second direction of reasoning¤

    A latent-variable model introduces an unobserved \(Z\) and specifies a joint law

    \[ p_\theta(x,z)=p_\theta(z)p_\theta(x\mid z). \]

    Generation follows the model direction:

    \[ Z\sim p_\theta(z),\qquad X\sim p_\theta(x\mid Z). \]

    Inference reverses the question after observing \(x\):

    \[ p_\theta(z\mid x) =\frac{p_\theta(z)p_\theta(x\mid z)} {\sum_{z'}p_\theta(z')p_\theta(x\mid z')} \]

    for a finite latent space. The posterior is implied by the joint model, but it may be expensive to compute. Variational inference introduces a tractable approximation \(q_\phi(z\mid x)\); Monte Carlo methods approximate expectations with weighted or correlated samples. Approximate inference changes the computation and may change the training estimator, even when the declared generative model is fixed.

    Worked example: a hidden manufacturing mode¤

    Let \(Z\in\{A,B\}\) be a hidden manufacturing mode with equal prior mass. Mode \(A\) produces only blue tiles and uses shape probabilities

    \[ p(X=00\mid Z=A)=0.9, \qquad p(X=01\mid Z=A)=0.1. \]

    Mode \(B\) produces only orange tiles and uses

    \[ p(X=10\mid Z=B)=0.3, \qquad p(X=11\mid Z=B)=0.7. \]

    Marginalizing \(Z\) gives the same four-state law:

    \[ p(X)=0.5(0.9,0.1,0,0)+0.5(0,0,0.3,0.7) =(0.45,0.05,0.15,0.35). \]

    For an orange triangle, Bayes' rule gives \(p(Z=B\mid X=11)=1\) because mode \(A\) assigns that state zero mass. For a model whose components overlap, posterior mode probabilities would lie strictly between zero and one.

    The latent representation is not uniquely identified by the observed marginal law. One could define other hidden states that integrate to the same four probabilities. Data about \(X\) alone establish the observed law more directly than they establish a preferred hidden explanation.

    One workflow, four checkpoints¤

    1. Fit: use training data to choose parameters.
    2. Validate: measure declared properties on data not used for the same fit.
    3. Infer: answer a posterior or missing-data query for a fixed model.
    4. Generate: sample from the fitted model using a stated algorithm.

    Evaluation can reveal that a fitted model is poor; it does not retroactively turn the metric into the model's training objective. A deployment filter can change the distribution of delivered samples; it must be included when the deployed system, rather than the base model, is evaluated.

    Your turn: classify four operations¤

    Classify each operation as learning, inference, sampling, or evaluation:

    1. updating weights from a minibatch gradient;
    2. estimating \(p(Z\mid X=x)\) after seeing a tile;
    3. drawing \(Z\) from the prior and then drawing \(X\mid Z\);
    4. measuring held-out log score and the frequency of every tile state.
    Check your answer

    The operations are, respectively: learning, inference, sampling, and evaluation. A real system may compose them—for example, VAE learning uses an inference network inside a training estimator—but their conceptual roles remain different.

    Chapter 4 retrieval practice¤

    1. Why are \(p_\star\), \(\widehat p_D\), and \(p_{\widehat\theta}\) different objects?
    2. What is the difference between a population objective and a minibatch estimator?
    3. In which direction does latent-variable generation run?
    4. Which distribution answers the reverse latent question after observing \(x\)?
    5. Why does exact agreement with an observed marginal not identify a unique latent explanation?

    Chapter 4 synthesis¤

    Learning selects parameters from data; inference answers conditional or latent queries under a model; sampling generates draws; evaluation tests declared properties. A population target, empirical objective, stochastic estimator, and optimizer occupy successive but non-equivalent layers.
